MoT Mixture-of-Transformers

Mixture-of-Transformers · 混合 Transformer 架构

📌 TL;DR

MoT 把多个模态（视觉语言 / 动作）分别接入独立的 transformer 路径，但在每层共享同一个 self-attention 操作，让模态间互相"看见"而不互相"干扰"。本质是"同一个注意力，不同的 FFN"的跨模态序列建模。

1定义

标准 Transformer 把所有 token 送入同一套参数（Attention + FFN）。MoT 的变体是：attention 层全局共享，但 FFN 层（或更多层）按模态拆分独立，每个模态走各自的 FFN 分支处理高层特征，最后汇合继续传递。

组件标准 TransformerMoT
Self-Attention所有 token 共享所有 token 共享（跨模态可见）
FFN所有 token 共享按模态独立（视语 FFN ≠ 动作 FFN）
跨模态影响通过 attention 实现通过 attention 实现，FFN 各自为政

2在 LingBot-VLA 里干什么

LingBot-VLA 的 MoT 把understanding expert（Qwen2.5-VL）和action expert分别对应"视语 FFN 路径"和"动作 FFN 路径"：

  • 视觉 + 语言 token 进 VLM 的 FFN；action chunk token 进 action expert 的 FFN。
  • self-attention 层全局共享 → 动作 expert 每层都能"读"到 VLM 全层的语义特征，而非只在输入层做一次 cross-attention。
  • 这避免了"action expert 只在输入看到 VLM 输出"的架构中常见的语义遗忘问题。

3与 π0 / GR00T N1 的对比

模型VLM-动作耦合方式特点
π0VLM + 独立 action expert，blockwise causal attnVLM 和 action expert 也共享 attention，与 MoT 思路相近
GR00T N1DiT action expert + cross-attention 注入 VLM 特征只在 action side 做 cross-attn，不如共享 self-attn 深度耦合
LingBot-VLAMoT（如 BAGEL），共享 self-attn + 独立 FFN每层全深度跨模态信息流，但 FFN 隔离防止干扰

4出处

📜 起源

MoT 这一术语和 BAGEL（Emerging Properties in Unified Multimodal Pretraining，arXiv:2505.14683）同来自字节跳动研究，是统一多模态预训练的框架。LingBot-VLA 直接引用 BAGEL 的 MoT 结构，将其应用到 VLA 动作生成领域。

5类比

把 MoT 想象成一个多学科会议室：每轮讨论（attention）全员参与、互相听取，但写报告（FFN）时工程师、医生、律师各写各专业的分析，不交叉，再汇总。比"全员写同一份报告"（标准 Transformer）更专业，比"完全分房间开会"（两个独立模型）更沟通畅顺。

Flow Matching Action Generation

Flow Matching · 流匹配生成模型

📌 TL;DR

Flow Matching 是一种把噪声→数据的轨迹学成向量场的生成方法。训练时教模型预测"从噪声到真实动作的瞬时速度方向"，推理时沿这个速度场做 ODE 积分，从随机噪声还原出平滑连续动作序列。比 DDPM 类 diffusion 更快推理（步数少）、轨迹更直。

1核心思想

给定真实动作 \(A_t\) 和高斯噪声 \(\epsilon\)，定义线性插值路径：

\[ A_{t,s} = s \cdot A_t + (1-s) \cdot \epsilon, \quad s \in [0,1] \]

当 \(s=0\) 是纯噪声，\(s=1\) 是真实动作。沿这条路径的理想速度场是 \(A_t - \epsilon\)（常数场）。Flow Matching 的训练目标：

\[ \mathcal{L}_{FM} = \mathbb{E}_{s, A_t, \epsilon} \left\| v_\theta(A_{t,s}, O_t, s) - (A_t - \epsilon) \right\|^2 \]

推理时从 \(A_{t,0} = \epsilon \sim \mathcal{N}(0,I)\) 出发，沿学到的向量场 \(v_\theta\) 做 ODE 求解（如欧拉法，通常 10 步即可），得到 \(A_{t,1} \approx A_t\)。

2在 LingBot-VLA 里干什么

action expert \(v_\theta\) 接受：

  • 当前噪声动作块 \(A_{t,s}\)（时间步 \(s\) 的中间状态）
  • 观测条件 \(O_t = [I^1_t, I^2_t, I^3_t, T_t, s_t]\)（三视角图像 + 任务指令 + 本体状态）
  • 流时间 \(s \in [0,1]\)

输出预测速度场，与 MoT 的共享 attention 耦合后，生成长度 \(T=50\) 步的动作 chunk（双臂关节角度序列）。

3与 DDPM Diffusion 的对比

DDPM DiffusionFlow Matching
噪声→数据路径随机/非线性（马尔可夫链）线性（直接插值）
推理步数通常 50–1000 步通常 10–20 步
训练目标预测噪声 \(\epsilon\) 或 \(x_0\)预测速度场 \(v = x_1 - x_0\)

在 VLA 中RoboVLMs、早期工作π0、LingBot-VLA、GR00T N1 等现代 VLA

4出处

📜 起源

Lipman et al. 2022 — "Flow Matching for Generative Modeling"（arXiv:2210.02747）首次提出。在 VLA 领域由 π0（Black et al. 2025）引入并推广，LingBot-VLA 直接沿用这一范式。

5具象例子

推理时机器人需要"把苹果放进碗里"：action expert 从 50 维高斯噪声（代表一个 50 步随机抖动的双臂轨迹）出发，经过 10 步 ODE 积分，逐渐"收敛"出一条平滑的双臂运动轨迹——就像把一团揉皱的纸，通过 10 次"拉伸"操作，变成一张平整的纸。

FSDP2 Distributed Training

Fully Sharded Data Parallel v2 · 全分片数据并行第二代

📌 TL;DR

FSDP2 是 PyTorch 的 ZeRO-3 实现升级版——把模型参数、梯度、优化器状态全部拆碎分布到每个 GPU，每块 GPU 只存自己负责那份，通信时再聚合。LingBot-VLA 的关键创新是给 action expert 建独立 shard group，减少不必要的全局通信。

1定义

ZeRO（Zero Redundancy Optimizer）分三个阶段消除并行训练中的冗余存储：

阶段分片内容显存节省
ZeRO-1优化器状态最小
ZeRO-2优化器状态 + 梯度中等
ZeRO-3（FSDP）优化器状态 + 梯度 + 参数最大（近线性）

FSDP2 是 PyTorch 2.x 对 FSDP 的重构版，API 更简洁、与 torch.compile 兼容性更好、支持精细化的 shard group 配置。

2在 LingBot-VLA 里干什么

LingBot-VLA 的关键工程决策：为 action expert 单独建 shard group，而非与 VLM 共用同一个全局 shard group。

  • VLM（Qwen2.5-VL）参数约数十亿，分片到所有 GPU。
  • action expert 参数量相对小得多；若与 VLM 合并分片，all-gather 的通信量变大（需要在 VLM 每一层反向传播时把 action expert 参数也同步）。
  • 独立 shard group 让 action expert 的同步只在其内部 GPU 子集发生，大幅降低通信开销。

3混合精度策略

计算和通信用 bfloat16（16位，显存省一半），但梯度归约（all-reduce）用 float32——防止 bf16 累加精度损失导致训练不稳定。这是大模型训练的标准做法（又称 mixed-precision with FP32 gradient accumulation）。

4类比

把模型参数想象成一本厚书，DDP 是每个工人都拿一本完整的书（GPU 显存浪费）；FSDP 是把书拆成若干章，每人只存自己的章，要用时向邻居借；FSDP2 的 shard group 则是把书里的"参考文献"（action expert）和"正文"（VLM）分两个小组各自借阅，不用每次借正文都同时通知参考文献组。

FlexAttention PyTorch Op

FlexAttention · 灵活稀疏注意力

📌 TL;DR

FlexAttention 是 PyTorch 2.x 提供的可编程稀疏注意力算子，允许用户通过 Python 函数定义任意 attention mask 模式（如 blockwise causal），编译后高效跳过零块计算，显著减少 FLOPs 和显存访问。

1定义

传统 FlashAttention 对 attention mask 的支持有限（仅 causal/全注意力）。FlexAttention 将 mask 抽象为一个用户自定义的 score modification function：

def my_mask(score, b, h, q_idx, kv_idx):
    # 返回修改后的 attention score 或 -inf（遮蔽）
    return score if q_idx >= kv_idx else float('-inf')

flex_attention(Q, K, V, score_mod=my_mask)

PyTorch 会将这个 Python 函数 JIT 编译成 CUDA kernel，并自动推导哪些 attention block 可以完全跳过（全遮蔽块），实现稀疏计算。

2在 LingBot-VLA 里干什么

LingBot-VLA 的 MoT 采用 blockwise causal attention：序列分为三个功能块 [图像, 状态, 动作]，块间单向（图像←状态←动作），块内双向。这个模式的 attention matrix 有大量零块：

  • 动作 token 不能 attend 到未来动作（causal 方向）。
  • 图像 token 不能 attend 到状态/动作 token（单向）。

FlexAttention 直接跳过这些零块，在长序列（图像 token 数量大）时 FLOPs 节省显著。

3与 FlashAttention 的关系

FlexAttention 底层也用 tiled kernel（类 FlashAttention 思路），但把 mask logic 提升到用户层面，兼顾了 FlashAttention 的 IO 效率和自定义 mask 的灵活性。两者不冲突，可视为"可编程版 FlashAttention"。

GM-100 Benchmark

The Great March 100 · 大行军一百任务基准

📌 TL;DR

GM-100 是一个覆盖 100 个精细双臂操作任务的真实机器人基准，特点是任务覆盖范围广（包含约 50% 测试动词不出现在训练高频词中）、有标准化对象和操作协议，可在多个商业平台上重现。LingBot-VLA 是首个在 GM-100 上进行大规模三平台评测的工作。

1定义与构成

  • 100 个任务：涵盖 pick、place、fold、assemble、pour、peel 等 100+ 原子动作类型，包含高度精细的任务（如"用过滤器筛混合颗粒"、"组装鲁班锁"）。
  • 39,000 条专家演示：按标准协议遥操采集，每任务 130 条（150 原始取 top-130）。
  • 3 平台：AgileX、Agibot G1、Galaxea R1Pro，所有双臂固定底盘。
  • 评估指标：SR（成功率，3 分钟内完成所有子步骤）和 PS（进度分，按完成的子任务数比例）。

2在 LingBot-VLA 里干什么

GM-100 是本文评测的主要 venue——LingBot-VLA 在此基准上做了目前最大规模的 VLA 对比：22,500 trials（100 任务 × 3 平台 × 15 次试验），与 π0.5、GR00T N1.6、WALL-OSS 对比。

3已开源

✅ 开放状态

GM-100 基准数据集已在 Hugging Face 开放：robbyant/lingbot-GM-100，包含任务描述、对象规格和演示数据。评测录像（rosbag 格式）亦计划逐步释出。

4局限性

GM-100 目前仅覆盖固定底盘双臂操作；无移动操作、无人形腿脚任务。最高绝对 SR（约 21%）提示当前 VLA 在精细操作上仍有大量提升空间。

Blockwise Causal Attention Attention Pattern

Blockwise Causal Attention · 块级因果注意力

📌 TL;DR

把序列分成若干功能块（如图像块、状态块、动作块），块内用双向注意力（所有 token 互相 attend），块间用因果注意力（只允许"未来"块 attend 到"过去"块，反向不行）。兼顾了块内信息完整性和块间信息方向性控制。

1定义

设序列被分为三块：图像 token \([I]\)，状态 token \([s]\)，动作 token \([a]\)。Blockwise causal 的 attention mask 规则：

Q 块 \ KV 块图像 [I]状态 [s]动作 [a]
图像 [I]✅ 双向❌ 遮蔽❌ 遮蔽
状态 [s]✅ attend✅ 双向❌ 遮蔽
动作 [a]✅ attend✅ attend✅ 双向（causal within block）

这样动作 token 能看到全部图像和状态信息，但图像 token 不会因为"知道"后续动作而信息泄漏。

2在 LingBot-VLA 里干什么

联合建模序列 \([O_t, A_t] = [I^1_t, I^2_t, I^3_t, T_t, s_t, a_t, ..., a_{t+T-1}]\)，用 blockwise causal 确保：

  • action expert 能利用所有观测信息（图像 + 指令 + 状态）。
  • 图像/语言特征不会被未来动作污染（保持 VLM 的正常 representation）。
  • 动作 chunk 内部 token 间可以双向 attend（允许 action expert 同时预测整个 chunk，而非自回归地逐步生成）。

3出处与同类

同样的机制最早在 π0（Black et al. 2025）中被形式化描述并用于 VLA 训练。LingBot-VLA 明确引用 π0 的这一设计（"Following π0, we implement blockwise causal attention"）。

4类比

想象一个翻译会议：译员（动作 expert）能看所有原文和上下文，但原文作者（VLM）写文章时不需要知道译员会翻成什么——blockwise causal 就是这种"单向知情"的信息流控制。

LingBot-Depth Depth Model

LingBot-Depth · 深度估计辅助模型

📌 TL;DR

LingBot-Depth 是 Robbyant 自研的 RGB→深度图估计模型，通过 Masked Depth Modeling 从单目 RGB 图像预测深度，用于给 LingBot-VLA 提供空间几何先验。推理时不需要额外深度传感器——深度由 RGB 隐式推导。

1定义

Masked Depth Modeling（引用 Robbyant 技术报告）：类比 MAE（Masked Autoencoders），但重建目标是深度值而非 RGB 像素。模型从遮蔽的 RGB patch 重建对应深度，学习几何结构的隐式表征。

2在 LingBot-VLA 里干什么

用于深度蒸馏（depth distillation）分支：

  1. LingBot-Depth 对三视角图像 \(I^1, I^2, I^3\) 输出深度 token \([D^1, D^2, D^3]\)。
  2. VLM 对同样图像用可学习 query \([Q^1, Q^2, Q^3]\) 提取视觉特征。
  3. 通过 cross-attention projection 对齐 \(Q_t\) 和 \(D_t\)，最小化 \(\mathcal{L}_{distill} = \mathbb{E}|Proj(Q_t) - D_t|\)。
  4. 效果：VLM 的视觉特征空间在训练中"靠近"深度模型的几何感知特征，推理时 VLM 就具备了隐式空间理解能力。

3深度 vs 无深度版本

提升幅度稳健但不激进——深度蒸馏是"锦上添花"而非核心突破。

4类比

就像给一个只看 2D 图的人（VLM）配了一本"3D 结构参考手册"（LingBot-Depth）——每次训练时对照这本手册理解空间关系，久而久之即使没有手册，也会形成空间直觉。
