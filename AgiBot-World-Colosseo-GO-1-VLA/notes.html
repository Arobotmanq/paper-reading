ViLLA 框架名称

Vision-Language-Latent-Action · 视觉-语言-潜在动作

📌 TL;DR

ViLLA 是本文提出的总体框架名称。与 VLA（Vision-Language-Action）的本质区别：在"语言"和"动作"之间插入一个"潜在动作"中间层——VLM 不直接预测动作，而是先预测离散的潜在动作 token，再由 Action Expert 将其解码为真实控制信号。

1定义

ViLLA = Vision + Language + Latent + Action 的缩写。相比 VLA，多出一个"Latent"层：

框架信息流VLM 的直接预测目标
VLA（如 OpenVLA、RT-2）图像 + 语言 → 动作直接输出动作 token 或连续动作
ViLLA（本文 GO-1）图像 + 语言 → 潜在动作 token → 动作输出离散的潜在动作 token；Action Expert 再做精控

2本文用法

GO-1 是 ViLLA 框架的具体实现，包含三个组件：

  • LAM（Latent Action Model）：学习潜在动作空间（Stage 1）
  • Latent Planner：VLM 在潜在空间规划（Stage 2）
  • Action Expert：扩散精控低层动作（Stage 3）

三阶段顺序训练，逐步解锁能力。

3为什么要加"Latent"层？

问题ViLLA 如何解决
VLM token 空间 vs 机器人动作空间不对齐潜在动作是 VQ-VAE 学出的紧凑离散 token，维度远小于动作空间，VLM 适配代价低
人类视频/网络视频没有动作标签LAM 无监督从帧差学潜在动作，任何视频都可用
跨本体泛化难LAM 的 codebook 是所有本体共享的语义码本
📜 命名来源

ViLLA 是本文原创框架名称，首次提出。其思路来自 LAPA（Latent Action Pre-training from videos，ICLR 2025）的潜在动作预训练理念，但 ViLLA 进一步将其与 VLM 骨干和 Action Expert 整合成完整的三阶段层级框架。

Latent Action 核心创新

Latent Action Model (LAM) · 潜在动作模型

📌 TL;DR

Latent Action 是从"看两帧画面"中无监督学出的紧凑离散动作语义表示，不需要任何动作标签。任何视频（人类操作视频、跨本体机器人视频）都能用来训练 LAM，解决了"有视频但无动作标签"这一核心瓶颈。

1定义

给定连续两帧图像 \(I_t\) 和 \(I_{t+H}\)，LAM 学习：

\[ z_t = I(I_t, I_{t+H}) \quad \text{(编码器：逆动力学)} \] \[ \hat{I}_{t+H} = F(I_t, z_t) \quad \text{(解码器：正向动力学)} \]

其中 \(z_t = [z_t^0, \ldots, z_t^{k-1}]\)，\(k=4\) 个离散 token，通过 VQ-VAE 量化到码本 \(\mathcal{C}\) 中。

2架构细节

  • 编码器：时空 Transformer，带因果时序 mask（借鉴 Genie 的设计，Bruce et al. ICML 2024）
  • 量化：VQ-VAE 目标，codebook 大小 |C|（论文未披露具体数值）
  • 解码器：空间 Transformer，输入初始帧 \(I_t\) + 量化后的 \(z_t\)，重建 \(I_{t+H}\)
  • 训练数据：Ego4D 人类视频 + 跨本体机器人数据（无需动作标签）

3与 IDM（Inverse Dynamics Model）的区别

Latent Action (LAM)IDM（如 GR00T N1 所用）
输出空间潜在离散 token（与真实动作无直接对应）物理动作（关节角/末端速度等）
训练监督自监督（重建未来帧，不需要 action label）监督（需要带 ground-truth action 的真机数据）
跨本体天然共享同一 codebook需要每个本体单独训练
下游用途作为 Latent Planner 的预测目标作为策略的动作标签
📜 出处与对比

Latent Action 的思路最早见于 Bruce et al. "Genie" (ICML 2024)，用于交互式环境生成。LAPA（Ye et al., ICLR 2025）首次将其引入机器人预训练，但未保留下游任务的潜在规划能力。GO-1 的贡献是将 Latent Action 与 VLM Latent Planner 整合并保留到推理阶段。

Latent Planner Stage 2

潜在动作规划器 · VLM 骨干在潜在空间做长程规划

📌 TL;DR

Latent Planner 是 GO-1 的"大脑"——基于 InternVL2.5-2B，接收多视角图像和语言指令，在 LAM 学出的潜在动作空间里做长程规划，输出潜在动作 token 序列。它充当了 VLM 通用知识与机器人精控之间的桥梁。

1定义与角色

Latent Planner 是 ViLLA 框架的第二组件，解决"如何把 VLM 的通用理解能力迁移到机器人控制"这一问题：

信息流架构
输入: 多视角图像 {I_head, I_left_wrist, I_right_wrist} + 语言指令 l
         ↓ InternViT 编码 → 投影到语言空间
         ↓ 24层 Transformer（全双向注意力，bidirectional attention）
         ↓ VLM 骨干逐层条件化（layer-by-layer conditioning）
输出: 潜在动作 token 序列 z_t = [z_t^0, ..., z_t^{k-1}]
训练目标: Masked Language Modeling（MLM，类 BERT）

2关键设计

  • VLM 骨干：InternVL2.5-2B（InternViT 图像编码器 + InternLM 语言模型）——2B 参数在机器人任务中证明有效，且与 RDT、π0 同量级
  • 双向注意力：与 GPT 系列的单向不同，使用全双向注意力，更适合预测离散序列（类 BERT/masked LM）
  • 层级调制：VLM 主干的每一层输出都作为 Latent Planner 对应层的条件——信息从上到下逐层传递，而不是只用最终输出
  • Stage 2 预训练目标：以 LAM 编码器（Stage 1）产出的 \(z_t\) 为监督信号做 MLM，训练 Latent Planner 学会预测潜在动作

3与 Action Expert 的分工

Latent PlannerAction Expert
角色高层规划（"做什么"）低层精控（"怎么做"）
输出离散潜在动作 token \(z_t\)连续动作 chunk \(A_t\)（H=30 步）
训练目标Masked Language Modeling扩散去噪（Diffusion）
依赖关系依赖 LAM（Stage 1）依赖 Latent Planner（Stage 2）
💡 类比

Latent Planner = 大脑皮层（"我要把杯子移到左边"）；Action Expert = 小脑/脊髓（"具体怎样控制每个关节才能完成"）。两者分工使得高层知识（VLM 的语言/视觉理解）可以直接参与规划而无需关心低层控制细节。

Action Expert Stage 3

动作专家 · 扩散目标回归连续低层动作

📌 TL;DR

Action Expert 是 GO-1 的"精控手"——与 Latent Planner 同架构（24 层 Transformer），但训练目标是扩散去噪（迭代去噪预测连续动作 chunk），由 VLM 主干和 Latent Planner 双重层级调制，实现高频灵巧操作。

1定义

Action Expert 是 ViLLA 框架第三组件，在 Stage 3 与 Latent Planner 联合训练：

\[ A_t = [a_t, a_{t+1}, \ldots, a_{t+H}], \quad H = 30 \] \[ \mathcal{A}(A_t \mid I_t^h, I_t^l, I_t^r, p_t, l) \quad \text{(扩散目标)} \]

其中 \(p_t\) 为本体感受状态（关节/末端位置），三视角图像 + 语言指令 + 本体感受作为条件。

2与 π0 的 action expert 对比

GO-1 Action Expertπ0 action expert
上游规划Latent Planner（潜在动作 token）调制VLM 主干（π0 用 PaliGemma）直接调制
扩散类型扩散目标（论文引用 [34]，GEMS 框架）Flow Matching
动作 chunkH=30 步H 步（具体值因任务）
与规划分离✓ 分两级（Latent + Action Expert）✗ 单级，VLM 直接到动作

3层级调制（Hierarchical Conditioning）

GO-1 的三组件形成严格的信息层级：

调制链推理时
VLM 主干（InternVL2.5-2B）
    → 逐层调制 → Latent Planner
                    → 预测 z_t（潜在动作 token）
                        → 逐层调制 → Action Expert
                                        → 去噪生成 A_t（连续动作 chunk）
📜 技术出处

扩散目标来自 [34] GEMS 框架（Bu et al., 2024, "Towards synergistic, generalized, and efficient dual-system for robotic manipulation"）——该框架提出了将高层语义规划与低层扩散控制分层结合的双系统思路，GO-1 的三阶段架构在其基础上进一步增加了潜在动作中间层。

VLA 基准概念

Vision-Language-Action Model · 视觉-语言-动作模型

📌 TL;DR

VLA 是用预训练 VLM（大视觉语言模型）骨干直接输出机器人动作的框架。GO-1 的 ViLLA 是 VLA 的升级版——在"语言"和"动作"之间插入潜在动作层，避免 VLM 直接面对高维动作空间。

1定义

VLA 模型接收视觉观测 \(I_t\) 和语言指令 \(l\)，直接输出机器人动作：

\[ a_t = \pi(I_t, l) \]

代表性工作：RT-2（Google，2023）——把动作 token 化后接入 PaLI-X；OpenVLA（2024）——开源 VLA 基准；π0（Physical Intelligence，2024）——VLM（PaliGemma）+ Flow Matching action expert。

2VLA vs ViLLA 的本质区别

维度VLAViLLA（GO-1）
预测目标直接预测动作（或动作 token）先预测潜在动作 token，再解码到实际动作
无标签视频利用难——需要动作标签才能训练易——LAM 从帧对无监督学习
VLM 适配代价高——需要将动作空间对齐到 VLM token 空间低——潜在动作空间维度小，语义紧凑
长程规划隐式（通过单次 forward pass）显式分层：Latent Planner 做规划 + Action Expert 精控

3本文用法

论文将 OpenVLA 作为"直接 VLM 适配"方法的代表进行对比：

the latent action space is orders of magnitude smaller than the discretized low-level actions used in OpenVLA

这句话直接说明了 ViLLA 潜在空间比 OpenVLA 动作空间小"数量级"——是选用潜在动作而非直接动作 token 的核心理由。

VQ-VAE 量化工具

Vector Quantized Variational AutoEncoder · 向量量化变分自编码器

📌 TL;DR

VQ-VAE 把连续的 latent vector 强制离散化到一个可学习的"码本（codebook）"里，使 latent 空间变成有限个原型 token 的集合。在 GO-1 中，它被用来量化 LAM 的潜在动作——使不同本体可以共享同一套动作语义码本。

1核心结构

VQ-VAE Pipeline结构
输入帧对 (I_t, I_{t+H}) ──► 时空 Transformer 编码器 ──► z_e (连续向量)
                                    │
                                    ▼ 在 codebook {e_1, ..., e_K} 里找最近邻
                                 z_q = e_{k*}  (离散 token)
                                    │
                                    ▼
                                 空间 Transformer 解码器 ──► 重建 I_{t+H}

2训练目标

\[ \mathcal{L} = \underbrace{\|I_{t+H} - \hat{I}_{t+H}\|^2}_{\text{重建}} + \underbrace{\|\text{sg}[z_e] - e_{k^*}\|^2}_{\text{codebook loss}} + \beta \cdot \underbrace{\|z_e - \text{sg}[e_{k^*}]\|^2}_{\text{commitment loss}} \]

3在 GO-1 中的作用

VQ-VAE 量化的关键好处：

  • 跨本体共享：所有数据（人类视频、单臂、双臂）共享同一码本，强迫模型学到跨本体通用的动作语义原型
  • MLM 友好：离散 token 可直接作为 Latent Planner 的 masked language modeling 预测目标
  • 信息瓶颈：k=4 个 token 的压缩迫使模型捕捉最本质的动作语义，而非细节
📜 出处

Van Den Oord et al. "Neural Discrete Representation Learning" (NeurIPS 2017)，即 VQ-VAE 原始论文。论文中直接引用为 [32]。

Cross-Embodiment 数据策略

跨本体学习 · 不同硬件平台的数据迁移

📌 TL;DR

跨本体学习指让一个模型能从不同硬件形态的机器人（单臂、双臂、不同关节数）甚至人类的数据中学习，使技能迁移而非从头训练。GO-1 通过 LAM 的共享潜在动作码本实现跨本体统一表示。

1问题背景

机器人数据的核心矛盾：不同机器人的观测空间（相机分辨率/位置）和动作空间（关节数/控制频率）完全不同，导致数据无法直接混合训练。OXE 尝试通过格式统一来解决，但各本体间的动作语义差异仍是瓶颈。

2GO-1 的解决方案

ViLLA 的潜在动作空间天然是跨本体共享的：

  • 编码器端：VQ-VAE 只看图像帧差（哪里在动、怎么动），不关心是几关节机器人还是人手
  • 共享 codebook：不同本体的动作都被量化到同一套原型 token，"右手向左移"这个动作语义在人手和机械臂上对应同一个（或相近的）token
  • 预训练数据：Ego4D 人类第一视角视频 + 跨本体机器人数据均参与 Stage 1 训练

3本文的规模

AgiBot World 本身虽然是同构数据集（全部 AgiBot G1 双臂机器人），但 GO-1 预训练还引入了：

  • 网络规模 VL 数据（Stage 2 VLM 预训练）
  • Ego4D 人类操作视频（Stage 1 LAM 训练）
  • 跨本体机器人数据（Stage 1）

三种数据来源在 LAM 的潜在动作空间中统一，再通过 Latent Planner 迁移到 AgiBot G1 的实机控制。

MoE 澄清说明

Mixture of Experts · 专家混合 · 在 GO-1 中的实际含义

📌 TL;DR

GO-1 的"双专家系统"（Latent Planner + Action Expert）不是标准的 MoE（Mixture of Experts，门控路由）架构。它更像是"层级专家分工"——两个专家串行条件化，而非并行路由选择。这里的"Expert"是功能性分工，不是 MoE 的技术含义。

1标准 MoE 是什么

标准 MoE 指：给定输入，通过门控网络选择 Top-K 个"专家子网络"并加权求和其输出。常见于大语言模型（如 Mistral MoE、DeepSeek-V2）以在参数量不变的情况下扩展容量。

2GO-1 的"双专家"实际上是什么

GO-1 的 Latent Planner 和 Action Expert 是层级调制关系，不是 MoE 路由：

特性标准 MoEGO-1 双专家
关系并行，门控选择串行，层级调制
信息流同一输入 → 多个专家 → 加权合并VLM → Latent Planner → Action Expert（单向流）
推理时选 Top-K 激活全部组件都参与，无路由选择
📌 用语澄清

论文中"dual-expert system"的"expert"是描述两个专门化模块的功能角色（规划专家 + 精控专家），不是 MoE 的技术术语。如果读到 GO-1 时看到"MoE"，需要区分语境——本文架构中没有使用标准 MoE 门控路由。

Diffusion Action Head 精控组件

扩散动作头 · 迭代去噪生成连续动作

📌 TL;DR

Diffusion Action Head（或 action expert 的扩散分支）用迭代去噪（DDPM/DDIM 类）生成连续动作 chunk，相比直接回归能更好地捕捉多模态动作分布（同一场景可以有多种合理操作方式）。GO-1 的 Action Expert 使用扩散目标，π0 使用 Flow Matching——两者都是生成式动作头，思路类似。

1为什么需要生成式动作头

机器人操作的动作分布往往是多模态的（抓杯子可以从左抓也可以从右抓），直接 MSE 回归会学到均值，导致"平均姿态"而非任一有效动作。扩散/流匹配通过建模分布解决这一问题：

\[ A_t \sim p_\theta(A_t \mid z_t, I_t, p_t, l) \]

2Diffusion vs Flow Matching

Diffusion (GO-1)Flow Matching (π0)
采样路径非直线（随机噪声 → 数据）接近直线（线性插值）
去噪步数通常多步（10-100）通常少步（1-10）
训练目标预测噪声 \(\epsilon\)预测速度场 \(v\)
推理速度较慢较快

两者在实践效果上接近，GO-1 选择 Diffusion 主要是继承 GEMS [34] 框架的设计；π0 选择 Flow Matching 因其推理效率更高。

3在 GO-1 中的位置

扩散动作头是 Action Expert 的训练目标，与 Latent Planner 的 MLM 目标并列，在 Stage 3 联合训练：

  • Latent Planner：预测 \(z_t\)（MLM，离散）
  • Action Expert：去噪生成 \(A_t\)（Diffusion，连续）
  • 两者在推理时串行执行：先 Latent Planner 输出 \(z_t\)，再 Action Expert 以 \(z_t\) 为条件去噪
