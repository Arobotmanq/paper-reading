Dual-System System 1 / 2

双系统架构 · 受 Kahneman 「快思慢想」认知理论启发

📌 TL;DR

用两个频率不同的 Transformer 模块分别负责「慢速语义推理」（System 2，VLM，10Hz）和「高频精细执行」（System 1，DiT，120Hz），通过 cross-attention 耦合，实现推理深度与控制实时性的兼顾。

1概念起源

Daniel Kahneman 在《Thinking, Fast and Slow》（2011）中提出：人类认知有两套系统——

系统特点类比
System 1（快）自动、直觉、低耗能、高频骑自行车、抓物体的肌肉记忆
System 2（慢）有意识、推理、高耗能、低频解数学题、规划路线

GR00T N1 把这个框架直接映射到机器人控制：

人类认知GR00T N1 实现频率职责
System 2（慢）Eagle-2 VLM10 Hz视觉理解 + 语言推理 → 输出语义 token
System 1（快）Diffusion Transformer (DiT)120 Hzcross-attend 语义 token，生成精细电机动作

2为什么要频率解耦

VLM 推理慢（需要处理图像 tokens + LLM forward pass）但语义稳定——场景语义不会每 8ms 变一次。DiT 需要高频闭环（120Hz 才能抑制抖动、响应碰触）但不需要重新理解任务。频率解耦让两者各做擅长的事：

  • VLM 每 100ms 更新一次语义 token（条件信号）
  • DiT 每 8ms 重新 denoising，基于最新机器人状态生成下一个动作 chunk
  • 两者通过 cross-attention 层异步耦合

3与其他双系统设计的对比

系统System 2System 1耦合方式
GR00T N1Eagle-2 VLM（10Hz）DiT flow-matching（120Hz）Cross-attention，联合训练
π0（Physical Intelligence）PaliGemma VLMAction expert（flow matching）MoE 架构，联合训练
SayCan / RT-2 风格LLM 规划独立低层策略接口调用，分离训练

GR00T N1 与 π0 都采用了端到端联合训练的双系统，区别在于 GR00T N1 用 cross-attention（类 Flamingo 风格），π0 用 MoE token 混合。

🎯 关键工程细节

论文发现使用 VLM 的第 12 层（中间层）而非最后层 embedding 作为 cross-attention 的 key/value，推理更快且下游成功率更高。最后层倾向过度处理成语言分布，中间层更保留视觉-语义融合表征。

DiT + Flow Matching System 1

Diffusion Transformer + Flow Matching Action Head · 扩散 Transformer + 流匹配动作头

📌 TL;DR

DiT 是用 Transformer 替代 U-Net 的扩散模型骨干；Flow Matching 是一种比 DDPM 更高效的「通过学习速度场来去噪」的训练方式，只需 4 步推理就能生成高质量动作 chunk。

1Flow Matching 是什么

经典 DDPM 学的是：给定噪声图，预测「当前噪声」（ε-prediction）。Flow matching（Lipman et al., 2022）则学的是：给定中间状态 \(A^\tau_t\)，预测从噪声到数据的速度场（velocity field）。

GR00T N1 的 flow-matching 目标：

\[\mathcal{L}_{fm}(\theta) = \mathbb{E}_\tau\left[\|V_\theta(\varphi_t, A^\tau_t, q_t) - (\epsilon - A_t)\|^2\right]\]
  • \(A^\tau_t = \tau A_t + (1-\tau)\epsilon\)：在时间步 \(\tau \in [0,1]\) 的插值动作（0=纯噪声，1=真实动作）
  • \(V_\theta\)：DiT 模型，预测去噪方向 \(\epsilon - A_t\)
  • \(\varphi_t\)：VLM 输出的语义 token
  • \(q_t\)：机器人状态

2为什么用 Flow Matching 而非 DDPM

方法推理步数训练稳定性轨迹形状
DDPM50~1000步好随机游走
Flow Matching4~10步好线性直线（效率高）
DDIM (加速版 DDPM)10~50步中确定性路径

Flow matching 的核心优势：噪声→数据的路径被设计为直线（conditional flow = 线性插值），所以少数积分步就能沿轨迹走完，GR00T N1 只需 K=4 步。

3DiT 的结构

DiT (Peebles & Xie, 2023) 把扩散模型的 U-Net 换成 Transformer，每个 block 包含：

  • Self-Attention：作用在动作 token \(A^\tau_t\) 和机器人状态 \(q_t\) 上（让动作内部互相关注）
  • Cross-Attention：以 VLM tokens \(\varphi_t\) 为 key/value，让动作 token 关注语义信息
  • Adaptive LayerNorm：用扩散时间步 \(\tau\) 调制 layernorm 参数（时间步条件化）

最后经过 embodiment-specific Action Decoder MLP 输出 H=16 步的动作 chunk。

4Action Chunking

一次推理生成 H=16 步的动作序列（约 133ms @120Hz），而非逐步预测。好处：

  • 减少推理次数（16:1），降低累积误差
  • 允许多步预测的时序一致性（比如流畅的抓握轨迹）
  • 缺点：执行第1步时第16步已预测好，环境变化需等下一次推理才能反应

Eagle-2 VLM System 2

NVIDIA Eagle-2 Vision-Language Model · NVIDIA 视觉语言模型

📌 TL;DR

Eagle-2 是 NVIDIA 自研的高效 VLM，基于 SmolLM2（轻量 LLM）+ SigLIP-2（视觉编码器），对 Internet 规模数据做了视觉-语言对齐预训练。GR00T N1 用它作为 System 2 推理模块，冻结语言部分，仅 fine-tune 与动作相关的部分。

1架构组成

组件具体模型职责
视觉编码器SigLIP-2 (Google)224×224 图像 → 经 pixel shuffle → 64 个图像 tokens
语言模型SmolLM2 (HuggingFace)轻量 LLM，处理文本+视觉 tokens，输出跨模态表征
对齐训练NVIDIA 内部 VL 数据遵循 Li et al. (2025) 配方进行视觉-语言任务预训练

2在 GR00T N1 中的用法

Eagle-2 在 GR00T-N1-2B 中占 1.34B 参数（总 2.2B 的 61%）。训练时：

  • 冻结语言组件（text backbone），防止 catastrophic forgetting
  • fine-tune 视觉编码器和对齐层
  • 取第 12 层（中间层）的 embedding（形状：batch × seq_len × hidden_dim）作为 DiT 的 cross-attention key/value

图像输入方式：以 VLM chat 格式输入（与 VL 预训练时一致），支持多帧图像输入（多视角相机）。

3中间层 vs 最后层 embedding

为什么用第 12 层而不是最后层？直觉：

  • 最后层受 LM Head 约束，倾向于生成「下一个词的 logits 分布」，语义已过度压缩为文本概率
  • 中间层保留了视觉-语言融合的丰富表征，既有场景语义又保留空间信息
  • 中间层维度更小（不需要 lm_head），推理速度快

Latent Action LAPA / VQ-VAE

Latent Action Pre-training for Any Robot · 任意机器体的潜动作预训练

📌 TL;DR

用 VQ-VAE「看两帧视频，推测帧间发生了什么动作」，把无动作视频自动标上伪动作标签，且所有 embodiment（人类、单臂、双臂、人形）共享同一 codebook，实现跨 embodiment 的动作语义统一。

1问题背景

训练数据金字塔的底层（人类视频）和中层（神经轨迹）都没有动作标签——视频里只有像素，没有关节角。直接用 flow-matching loss 训练需要 ground-truth action。Latent Action 的方案：

  1. 训练一个 VQ-VAE：输入 \((x_t, x_{t+H})\) 帧对，输出「从 \(x_t\) 变成 \(x_{t+H}\) 所需的潜动作 \(z_t\)」
  2. VQ-VAE 自监督训练（重建未来帧），不需要真实动作标签
  3. 训练好后，对所有无动作视频提取 pre-quantization 连续 embedding 作为伪动作标签
  4. 以这些伪标签训练策略，把「LAPA embodiment」视为一种特殊的跨 embodiment

2VQ-VAE 结构

Latent Action VQ-VAE 流程
Encoder(x_t, x_{t+H}) → z_e (连续 embedding)
                            ↓ 量化到 codebook {e_1,...,e_K}
                         z_q = e_{k*}  (量化后的离散 token)
Decoder(x_t, z_q)       → x̂_{t+H} (重建未来帧)

训练 loss：重建 loss + codebook loss + commitment loss

推理时：取 z_e (pre-quantization 连续 embedding) 作为伪动作

3为什么所有 embodiment 共享一个 codebook

论文 Figure 4 展示：同一个 latent action（相同 codebook slot）在人类手、单臂机器人、双臂机器人、GR-1 人形上，都对应着「右臂向左运动」这一语义。共享 codebook 强迫模型在不同 embodiment 之间找到跨域动作语义的最大公约数，是跨 embodiment 迁移的基础。

4LAPA 与 IDM 的对比与互补

LAPA (VQ-VAE)IDM
输出空间抽象潜在动作（与真实关节角无直接对应）真实物理动作（关节角/末端速度）
训练监督自监督（重建未来帧）监督（需要带真实动作的真机轨迹）
跨 embodiment天然共享 codebook，强需每个 embodiment 单独训 IDM
低数据量优（不依赖真机数据规模）弱（标签质量差）
高数据量一般优（标签质量随规模提升）
✅ GR00T N1 的实际选择

两者都用。数据金字塔底层（人类视频）用 LAPA；中层神经轨迹按数据量决定：低数据用 LAPA，高数据用 IDM（论文 §4.4 ablation 验证了这一预期）。真机 GR-1 后训练只用 IDM 动作，因为此时已有足量真机数据。

IDM Inverse Dynamics

Inverse Dynamics Model · 逆动力学模型

📌 TL;DR

IDM 是「看相邻两帧画面，反推这一步执行了什么动作」的神经网络。用真机数据（有 ground-truth action）监督训练好后，可以给神经轨迹（只有视频，无动作）打上伪标签。

1概念

方向输入输出学的是
Forward Dynamics\((s_t, a_t)\)\(s_{t+1}\)"做了这个动作，下一状态是什么"
Inverse Dynamics (IDM)\((s_t, s_{t+1})\)\(a_t\)"从状态 A 到状态 B，中间发生了什么动作"

2在 GR00T N1 中的流程

IDM 伪标签流程
第一步：在真机数据（有真实 action）上训练 IDM：
    loss = || IDM(x_t, x_{t+1}) - a_t ||²

第二步：对神经轨迹视频每对相邻帧做推断：
    â_t = IDM(x_t, x_{t+1})

第三步：用 â_t 作为伪动作标签，训练策略模型

3出处

📜 起源：OpenAI VPT (2022)

Baker et al. 2022 在 7 万小时 Minecraft 视频上训练 IDM，给 YouTube 无标注游戏录像打伪标签，再用这些数据预训练 Minecraft 智能体（VPT，Video PreTraining）。GR00T N1 直接引用了这篇，思路一脉相承：把无监督视频转化为有监督训练数据。

Neural Trajectories Data Aug

神经轨迹 · 用视频生成模型合成的机器人演示视频

📌 TL;DR

在少量真机演示上 fine-tune image-to-video 生成模型，然后用不同的语言提示生成大量「如果机器人被要求做另一件事，它会是什么样」的反事实视频——从 88h 真机数据扩增到 827h（10×）。

1生成流程

  1. Fine-tune 视频生成模型：在 3000 条真机数据上 fine-tune open-source image-to-video 模型（如 Wan、CogVideoX 等），100 epochs
  2. 对象发现 + 反事实提示：用商业 VLM 检测场景中的物体，自动生成 N 种物理可行的「pick up {A} from {B} to {C}」提示
  3. 视频生成 + 过滤：生成视频后，用 VLM 作为 judge 过滤掉不符合语言指令的视频，再重新 caption
  4. 动作标注：用 LAPA 或 IDM 给生成视频打伪动作标签
  5. Co-training：真机轨迹：神经轨迹 = 1:1 采样比

2价值与局限

神经轨迹仿真轨迹（DexMimicGen）
优点真实外观，反事实多样性，适合视觉 OOD有真实动作标签，大规模（78万条）
缺点无法保证物理一致性（视频生成模型的限制）sim-to-real gap，渲染与真实场景差异
用途后训练 co-training，提升 OOD 泛化预训练 + 后训练，补充真机数据
⏱ 生成成本

生成 1 秒视频需 2 分钟（L40 GPU）；全部 827h 神经轨迹共用了约 10.5 万 L40 GPU 小时（约 1.5 天，3600 个 GPU 并行）。相比之下，等效的人工遥操作需要 827 小时人力——成本比约 100x 以上。

DexMimicGen Sim Data

DexMimicGen · 仿真中大规模自动生成灵巧操作轨迹

📌 TL;DR

从少量人类示范出发，将其分解为以物体为中心的子任务片段，然后自动将这些片段变换+重放到新的物体位置和场景配置，从几十条示范扩增到数万条——GR00T N1 用这种方法生成了 78 万条仿真轨迹（≈6500 小时）。

1工作原理

  1. 人类示范收集：用 Leap Motion 遥操作设备收集几十条高质量示范
  2. 物体中心分割：将每条示范切分为以单一物体为中心的子任务片段（如：抓取杯子、移动杯子、放置杯子）
  3. 变换+重放：将片段对齐到新物体位置，调整末端执行器与物体的相对位姿，插值中间运动
  4. 执行+验证：在仿真中执行并验证成功，只保留成功的轨迹

2规模与效率

GR00T N1 的实际效果：从每对（source, target）容器的少量示范出发，生成 10,000 条新示范。54 种容器组合 × 10,000 = 540,000 条预训练仿真轨迹。加上后训练数据，共生成 780,000 条轨迹（6500 小时），仅用 11 小时。

📜 与 MimicGen 的关系

DexMimicGen（Jiang et al., 2024）是 MimicGen（Mandlekar et al., 2023）的扩展版，主要增加了对灵巧手（dexterous hand）和双臂操作的支持。MimicGen 专注于平行夹持器（parallel-jaw gripper）的单臂任务，DexMimicGen 则支持 GR-1 人形机器人的整个手部（多指）控制。

Cross-Embodiment Transfer

跨 Embodiment 学习 · 从多种机器体数据中迁移学习

📌 TL;DR

让一个模型同时学习来自不同「身体」（单臂、双臂、人形、人类手）的数据，利用共同的视觉-语义知识和操作技能实现迁移——GR00T N1 用 embodiment-specific encoder/decoder 统一不同 embodiment 的状态/动作维度，共享 DiT 骨干。

1GR00T N1 支持的 embodiment

  • 真机：Fourier GR-1 人形机器人（灵巧手 + 双臂）；Open X-Embodiment 数据集中的 Franka、Bridge 等单臂机器人；AgiBot-Alpha 数据集
  • 仿真：双臂 Panda（平行夹持器）、双臂 Panda（灵巧手）、GR-1 人形
  • 人类视频：Ego4D、EPIC-KITCHENS 等，作为「LAPA embodiment」
  • 神经轨迹：视频生成模型合成的 GR-1 轨迹

2技术实现

不同 embodiment 的状态/动作维度不同（如单臂 7DoF vs 人形 54DoF），GR00T N1 的解决方案：

  • Embodiment-specific MLP：每个 embodiment 有专用的 State Encoder MLP 和 Action Decoder MLP，负责将任意维度映射到共享 embedding 维度
  • 共享 DiT 骨干：所有 embodiment 共享同一套 Transformer weights
  • Latent action 共享 codebook：所有 embodiment 的无动作视频都标注在同一 latent action 空间

3预训练的数据集组成

数据层来源动作类型
底层（最大量）Ego4D, EPIC-KITCHENS, HOI4D 等人类视频LAPA 潜动作
中层（仿真）DexMimicGen 540k + RoboCasa 多任务真实动作
中层（神经）视频生成模型合成的 GR-1 轨迹（827h）LAPA 或 IDM 伪动作
顶层（真机）GR-1 遥操作（88h）+ OXE + AgiBot真实动作
🔑 迁移的本质

跨 embodiment 学习的本质是：视觉-语义知识不依赖 embodiment（「杯子」在哪儿、该怎么抓，无论是人手还是机械臂都一样），而具体运动执行依赖 embodiment（关节角范围、手指灵活度不同）。共享 backbone 学通用知识，embodiment-specific head 学具体运动。
