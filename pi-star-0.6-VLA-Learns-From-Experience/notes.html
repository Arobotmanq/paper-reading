RECAP RL Framework

RL with Experience and Corrections via Advantage-conditioned Policies

📌 TL;DR

RECAP 是一套让 VLA 通过自己的部署经验持续改进的 RL 框架。它把 demonstrations、自主 rollout（成功与失败）、以及人类在线干预统一成 offline RL 的数据流，用 advantage conditioning 把价值函数信号注入 VLA，无需改变模型结构即可实现端到端 RL 训练。

1为什么需要 RECAP

模仿学习（BC/SFT）的硬上界是示范质量：策略最多学到"平均人类水平"，且会因 distribution shift（累积误差）变得更差。RL 可以突破这个上界——但大型 flow matching VLA 没有可解析的 log-likelihood，PPO/REINFORCE 难以直接应用。RECAP 解决的就是这个矛盾。

2三步流程

1. 数据收集 (Data Collection)
   - 用当前策略部署，给每个 episode 打成功/失败标签
   - 可选：人类专家在自主执行期间进行干预（纠正错误）
   - 全部数据加入 dataset D_ℓ

2. 价值函数训练 (Value Function Training)
   - 用 D_ℓ 中所有数据训练多任务 distributional VF
   - VF 预测"距成功还剩多少步"的分布（B=201 个 bin）
   - 交叉熵 loss，语言条件化，多任务共享

3. Advantage Conditioned Training
   - 用 VF 对 D_ℓ 中每条 (o, a) 估算 advantage
   - 阈值 ε_ℓ（取第 30 百分位值）二值化：A > ε → I=True
   - 用 I 作为文本 token 前缀，以 SL 目标训练整个 VLA

重复步骤 1-3，K 次迭代

3预训练 vs. 微调阶段

阶段数据VF 初始化π 初始化特殊设置
预训练全量多任务多机器人数据从头训从头训步骤 2+3，无在线采数据
任务 SFT目标任务 demonstrations从预训练 VF 微调从预训练 π 微调固定 I=True（等同 SFT）
迭代 RLSFT 数据 + 在线 rollouts + interventions每次从预训练 VF 微调每次从预训练 π 微调完整 RECAP 三步

关键设计：每次迭代从预训练 checkpoint 微调而非接着上次迭代，避免多次更新导致的 drift。

4出处

📜 算法家谱

RECAP 的 advantage conditioning 部分源自 Kumar et al. 2019 "Reward-conditioned policies"（文中 [48]）和 Frans et al. 2025 "CFGRL"（文中 [4]）。价值函数设计参考 Ghasemipour et al. 2025 的 time-to-completion value function。DAgger 风格人类干预来自 Ross et al. 2011。RECAP 的贡献是把这些组件组合成一套适配大型 flow matching VLA 的完整系统。

Advantage Conditioning Policy Extraction

优势条件化策略提取 · Binarized Advantage Indicator as Text Token

📌 TL;DR

把"这个动作比平均水平好还是差"这个二元信息，以自然语言 token（"Advantage: positive/negative"）的形式输入给 VLA。模型同时学条件和无条件策略，推理时强制条件为"positive"即得改进策略。无需可计算 log-likelihood，适配 flow matching 模型。

1理论基础

正则化 RL 的最优解为：

\[ \hat{\pi}(a|o,\ell) \propto \pi_{\text{ref}}(a|o,\ell) \cdot \left(\frac{\pi_{\text{ref}}(a|I,o,\ell)}{\pi_{\text{ref}}(a|o,\ell)}\right)^\beta \]

对 β=1 的特殊情况，改进策略恰好等于给定 I（improvement indicator）条件下的策略——只要我们训练模型同时表示条件和无条件策略，就能在推理时直接得到改进策略，无需显式计算 advantage 的期望或梯度。

2在 π*₀.₆ 中的实现

# 训练时：每个 (o_t, a_t) 都计算 advantage 并二值化
I_t = True  if  A(o_t, a_t, ℓ) > ε_ℓ  else  False

# 文本 prefix 构造：
prefix = "Advantage: positive"  # I_t = True
prefix = "Advantage: negative"  # I_t = False

# 训练目标（所有数据都参与）：
loss = -E[ log π(a_t|o_t,ℓ) + α·log π(a_t|I_t,o_t,ℓ) ]

# 推理时：强制 I = True
action = π(· | "Advantage: positive", o, ℓ)

3与 AWR 和 PPO 的对比

方法数据利用率适配 flow matchingoff-policy 数据实际效果（论文）
Advantage Conditioning全部数据（高/低 advantage 都用）原生支持（SL 目标）天然支持最高吞吐量
AWRexp-weighted，低优数据权重极小（实质丢弃）支持（加权 SL）支持成功率可，速度慢
PPO需 on-policy 数据；off-policy 需极小 trust-region困难（需 log-likelihood）不友好需 η=0.01 稳定，效果差

4类比理解

🎯 直觉类比

想象训练一个学生写代码：AWR 说"只看优秀例子，差例子扔掉"；PPO 说"按当前模型和目标模型的概率比调整"（但 flow matching 没有概率）；Advantage conditioning 说"把'这段代码写得好/差'直接告诉模型，让它自己学习'好'意味着什么"——模型用 CFG 风格同时学"一般写法"和"好的写法"，推理时直接要"好的写法"。

Distributional Value Function Critic

分布式价值函数 · Time-to-Completion 分布预测

📌 TL;DR

不直接回归"期望步数"，而是预测"距离成功还剩 N 步"这个随机变量的离散概率分布（201 个 bin）。用与 VLA 相同架构但更小的 VLM（670M）作为 backbone，语言条件化，多任务共享，输出分布比点估计更稳定，且对稀疏奖励友好。

1奖励定义与价值含义

\[ r_t = \begin{cases} 0 & \text{if } t=T \text{ and success} \\ -C_{\text{fail}} & \text{if } t=T \text{ and failure} \\ -1 & \text{otherwise} \end{cases} \]

这使得价值函数 V(o_t) 等于"距成功还剩步数的负值"（成功轨迹）或大负数（失败轨迹）。实现上归一化到 (-1, 0)，按任务最大长度标准化，使不同长度任务的价值可比。

2训练方式

\[ \min_\phi \mathbb{E}_{\tau \in \mathcal{D}} \left[\sum_{o_t \in \tau} H\left(R^B_t(\tau), p_\phi(V|o_t, \ell)\right)\right] \]

用 Monte Carlo 估计（而非 Bellman backup）：直接把实际 return 离散化成 201 个 bin，然后最小化交叉熵。这避免了 off-policy Q-learning 的 bootstrapping 误差，虽然 on-policy 估计不如 Q-learning 精确，但实践中非常稳定可靠。

3架构细节

  • Backbone：SigLIP 400M + Gemma 3 270M（VLA 用 4B，VF 用 270M 以节省计算）
  • 在线推理 VF 成本极小，可在 VLA 训练时 on-the-fly 估算 advantage
  • 同时用少量多模态 web 数据 co-train VF，防止过拟合
  • VF 初始化：也来自 Gemma 3 预训练权重

4为什么选 distributional 而非点估计

📐 设计理由

点估计 Q-function（如 DQN、SAC）需要 Bellman backup，在 off-policy / offline 设置下存在外推误差（distributional shift）。Distributional VF + MC 估计：
① 无 Bellman backup 误差；② 交叉熵 loss 天然适配 VLM 的 token 预测架构（等价于分类头）；③ 可表示 return 的不确定性，对稀疏奖励/长 horizon 任务更鲁棒。

Flow Matching VLA Architecture

流匹配视觉-语言-动作模型 · 连续动作生成

📌 TL;DR

Flow matching 是一种生成连续动作分布的方法：从噪声出发，用神经网络学习一个"流场"（向量场），沿着流场积分即可把噪声映射到动作分布。相比扩散（DDPM），flow matching 的轨迹更直（ODE 而非 SDE），采样步数更少（推理更快）。π*₀.₆ 用独立的 860M action expert 做 flow matching，与 VLM backbone 共享注意力但 stop gradient 隔离。

1与 Diffusion 的关系

Diffusion (DDPM/DDIM)Flow Matching
过程SDE（随机微分方程），噪声路径弯曲ODE（常微分方程），路径更直
训练目标预测噪声 ε 或 x₀预测向量场 v(x,t) = x₁ - x₀
采样步数通常 20-50 步（DDIM 可少）通常 10 步以内
Log-likelihood下界（ELBO）无解析 log-likelihood，类似 ELBO

在 π*₀.₆ 中，flow matching loss 作为连续 log-likelihood 的下界来近似，使 advantage conditioning 的 SL 目标可以同时优化离散 token（cross-entropy）和连续动作（flow matching loss）。

2为什么 flow matching 对 RL 困难

PPO/REINFORCE 需要 ratio π_θ(a|o) / π_ref(a|o)——这要求可以精确计算当前策略和参考策略在动作 a 上的概率密度。Flow matching 模型没有解析 likelihood，只能通过 ELBO 近似，且近似误差在策略更新中会累积放大。RECAP 通过 advantage conditioning（SL 目标）完全规避了这一问题。

3π*₀.₆ 的 Knowledge Insulation (KI) 训练

VLA 同时预测三类输出：语言子任务 ℓ̂（next token prediction）、离散动作 a^ℓ（FAST tokenizer 离散化，cross-entropy）、连续动作 a（flow matching）。Stop gradient 隔离 action expert 与 VLM backbone，防止流场损失反向影响 VLM 的语言知识。这使预训练在语言/视觉数据上保持稳定。

π₀ 家族 Model Family

Physical Intelligence π₀ / π₀-FAST / π₀.₅ / π₀.₆ / π*₀.₆ 发展脉络

📌 TL;DR

π₀ 家族是 Physical Intelligence 开发的 flow matching VLA 系列。从初代的单任务 flow matching，到 π₀.₅ 的异构数据 co-training 泛化，再到 π*₀.₆ 用 RL 从部署经验中持续改进——每一代解决一个核心问题。

1发展时间线

模型时间核心创新关键组件开源
π₀2024-10Flow matching + VLM 的 VLA 范式SigLIP + PaliGemma + 300M action expert是（openpi）
π₀-FAST2024-10FAST 离散化 token 的动作表示FAST tokenizer 预训练是（openpi）
π₀.₅2025-04异构 co-training → 开放世界泛化同模型两级推理；97.6% 非移动机器人数据是（openpi）
π₀.₆2025-11更大 backbone + 更多数据（工程迭代）Gemma 3 4B + 860M action expert否

2关键设计演进

  • π₀ → π₀.₅：从单纯 flow matching VLA → 异构数据 co-training + 两级推理。核心问题：如何泛化到新环境（数据迁移而非数据扩增）。
  • π₀.₅ → π₀.₆：工程迭代。更大 VLM（Gemma 2B → Gemma 3 4B），更大 action expert（300M → 860M），更多预训练数据。
  • π₀.₆ → π*₀.₆：范式迁移。从"用更好的数据训更好的 BC 策略"到"让策略在部署中通过 RL 不断超越 BC 上界"。核心问题：如何在实际部署中自主提升。
📌 对 Track A 的含义

π₀.₅ 已证明了异构数据迁移的可行性（co-training 使移动操作泛化到新家庭）。π*₀.₆ 进一步证明了 RL 改进的可行性（advantage conditioning 使 flow matching VLA 可 RL 训练）。两篇合起来构成了"怎么让 VLA 既能泛化又能持续改进"的完整路径——对人形操作 VLA 的研究具有直接参考价值。

On-Robot RL Real-World RL

在机器人上做强化学习 · 从真实部署中学习

📌 TL;DR

在真实机器人上运行 RL 面临独特挑战：奖励稀疏且需人工标注、数据采集慢（无法并行仿真）、episode reset 需人工、探索容易损坏设备。π*₀.₆ 通过 offline RL（而非 online RL）、稀疏二元奖励、DAgger 式人类干预来适配这些约束。

1π*₀.₆ 的 on-robot RL 配置

  • 数据规模：T-shirt 洗衣任务，每轮 300 轨迹 × 4 台机器人，共 2 轮；Box assembly，每轮 600 autonomous + 360 intervention 轨迹，共 2 轮。
  • 奖励定义：episode-level 成功/失败（人工判断），episode 结束后标注，无 step-level 奖励信号。
  • 干预策略：人类专家监控自主 rollout，出错时接管（teleoperate 纠正），然后继续让机器人自主执行。
  • 迭代次数：实践中 1-2 次迭代已有显著效果；K=1 足以在洗衣任务上把成功率提升到 90%+。

2与仿真 RL 的关键区别

仿真 RL（e.g. sim2real）On-Robot RL（RECAP）
数据速度可并行 × 1000 环境物理时间限制，慢
奖励自动计算，稠密或稀疏皆可人工标注，必须稀疏
探索随机探索代价低（重置自动）需谨慎（设备安全），RECAP 用策略随机性 + 人类干预
Reset自动 reset人工 reset（π*₀.₆ 的主要人力成本之一）
分布 shift迁移到真实环境有 gap直接在真实分布上训练

3具象例子：洗衣任务的 RECAP 循环

轮次 0（预训练 + SFT）：
  - 用 π0.5 数据预训练 π*₀.₆（offline RL，全数据集）
  - 用洗衣 demonstrations SFT（I=True）→ π⁰_ℓ

轮次 1（第一次 on-robot 迭代）：
  - 4 台机器人各自运行 π⁰_ℓ，收集 300 episodes
  - 人类标注每集成功/失败
  - 用所有数据更新 VF → 更新 π → π¹_ℓ
  - 结果：成功率从 ~60% → ~90%（T-shirt 任务）

轮次 2（第二次 on-robot 迭代）：
  - 用 π¹_ℓ 再收集 300 episodes
  - 重复更新 VF 和 π → π²_ℓ
  - 结果：吞吐量继续提升 ~50%，但成功率已接近上限
