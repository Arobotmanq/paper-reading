Ego3D PE 观测端3D注入

Egocentric 3D Position Encoding · 自我中心3D位置编码

📌 TL;DR

把相机看到的2D图像像素，通过单目深度估计 + 反投影转换为以相机为原点的3D坐标，再编码为位置嵌入加到视觉 token 上。让 VLM 的视觉 token 同时携带"这个像素在3D空间哪个位置"的信息——无需额外深度相机，无需机器人-相机外参标定。

1为什么需要它

标准 VLA（OpenVLA、RT-2 等）将 SigLIP 提取的2D视觉 token 直接送入 LLM，这些 token 只有"语义内容"（这里有一个杯子），但没有"3D位置"（这个杯子在机器人前方 40cm 处，高度 15cm）。对于需要精确空间推理的操控任务——比如"把最近的物体放到左边"——模型只能靠训练数据的统计先验"猜"，而不是真正"知道"。

2技术流程

Ego3D PE Pipeline伪代码
输入：RGB 图像 I (H×W×3)

Step 1: ZoeDepth 估计深度图
  D = ZoeDepth(I)          # D ∈ R^{H×W}，每像素的米制深度

Step 2: 反投影到3D坐标 (自我中心坐标系)
  p_{x,y} = π⁻¹(u, v, D(u,v); K)   # K = 相机内参矩阵
  # 得到每个像素的3D坐标 P ∈ R^{3×H×W}

Step 3: 正弦位置编码 + MLP
  P' = MLP(γ(P))           # γ = 正弦函数，P' ∈ R^{d×H×W}

Step 4: 与 SigLIP 视觉 token 相加
  O3d = X + P'             # X = SigLIP 2D token，O3d 送入 LLM

关键：用的是相机内参（焦距、主点），而不是机器人-相机外参（机械臂基座到相机的变换）。内参通常由厂商提供或可自标定，且一旦固定几乎不变；而外参每次重新安装相机都需要重新标定，是实际部署的痛点。

3在 SpatialVLA 里的具体用法

论文关键公式（式2）：

\[ O_{3d} = X + P' = X + \text{MLP}(\gamma(P)) \]
  • \(X \in \mathbb{R}^{d \times h \times w}\)：SigLIP 视觉编码器提取的2D语义 token（保留 VLM 预训练的视觉-语言对齐）
  • \(P \in \mathbb{R}^{3 \times h \times w}\)：每个 patch 的自我中心3D坐标
  • \(\gamma(\cdot)\)：正弦函数（类似 Transformer 的位置编码，让 MLP 能处理连续3D坐标）
  • MLP 参数在预训练中随机初始化并优化；文本 token embedding 冻结

4和其它3D注入方案的对比

方案硬件需求标定需求跨本体适用性代表工作
RGB-D 相机（深度传感器）专用深度相机RGB-D 内外参低（设备依赖）多数实验室机器人
点云编码器LiDAR / 深度相机点云到机器人基座外参低3D-VLA, PerAct
3D patch（LLaVA-3D）深度相机内外参中LLaVA-3D
Ego3D PE（本文）普通 RGB 相机仅内参高（无需重标定）SpatialVLA
📜 局限

ZoeDepth 是单目深度估计网络，在弱纹理、反光、透明物体上深度估计误差较大；论文未分析 Ego3D PE 对深度估计质量的敏感性。另外，相机内参仍需提供（虽比外参稳定得多）。

Adaptive Action Grids 动作端3D离散化

自适应空间动作格子 · 按数据分布的等概率离散化

📌 TL;DR

把7维连续机器人动作（x,y,z,roll,pitch,yaw,grip）按实际数据的 Gaussian 分布"等概率"切分成离散格子，让高频动作区域格子更细——3个 token 编码一步动作，比 RT-2/OpenVLA 的7个快4倍，且精度更高。

1为什么不用线性等间距离散化

RT-1、RT-2、OpenVLA 都用线性 256-bin 等间距：把每个动作维度均匀切256份。问题是：机器人的实际动作分布是高度非均匀的——大多数操作动作幅度很小（精细调整），极少数幅度很大（大幅移动）。等间距意味着：

  • 大量 bin 落在"几乎从不使用"的大幅动作区域，浪费容量
  • 小幅动作区域 bin 太少，精度不足
  • 消融实验：SpatialVLA 自适应格子 vs 线性256-bin，Google Robot variant aggregation 高 +36.5%

2技术流程

Adaptive Action Grids构建流程
Step 1: 动作分解
  7D 动作 a = {x, y, z, roll, pitch, yaw, grip}
    → ΔT (平移) = 转极坐标 (ϕ, θ, r)   [方向×方向×距离]
    → ΔR (旋转) = (roll, pitch, yaw)
    → grip = {open=0, close=1}           [2 个离散状态]

Step 2: 拟合 Gaussian 分布（在全部预训练数据上统计）
  对 ΔT 的 (ϕ, θ, r) 和 ΔR 的 (roll, pitch, yaw)
  分别拟合 N(μ_a, Σ_a)

Step 3: 等概率切分格子
  在每个归一化变量 [-1, 1] 上，按 1/M 等概率确定切点
  a₂, ..., aₘ = argmin ∫ f(x)dx - 1/M  [式4]
  → 密集区（高频动作）格子细，稀疏区格子粗

Step 4: 构建 token vocabulary
  ΔT → Mtrans 个格子 (Mϕ × Mθ × Mr)
  ΔR → Mrot 个格子 (Mroll × Mpitch × Myaw)
  grip → 2 个 token
  总 V = Mtrans + Mrot + 2 = 8194

Step 5: 学习 spatial token embedding
  Ea = {Etrans, Erot, Egrip}    # 随机初始化，随模型训练
  预测时：autoregressive 预测 3 个 token → de-tokenize 得到动作

3为什么是 3 个 token 而不是 7 个

原来 RT-2/OpenVLA：每个动作维度独立离散化 → 需预测 7 个 token（x,y,z,roll,pitch,yaw,grip 各一个）。

SpatialVLA：把6个连续维度映射到三维网格查表 → 只需预测 3 个 token（ΔT 一个格子编号 + ΔR 一个格子编号 + grip 状态）。每个格子索引覆盖多维，用 embedding 来区分不同格子的语义。这使推理速度从 ~5Hz 提升到 20Hz。

4和其它动作表示的对比

方案token 数/步分辨率分布自适应迁移能力
线性256-bin（RT-2/OpenVLA）7256 per dim否差（均匀切分不匹配新分布）
连续回归（Octo）N/A连续是中

Spatial Embedding Adaption 后训练迁移

空间嵌入自适应 · 新机器人快速迁移机制

📌 TL;DR

对新机器人数据重拟 Gaussian → 生成新的动作格子 → 用三线性插值从预训练格子 embedding 初始化新格子 embedding → LoRA 微调。让新机器人的格子"继承"预训练中学到的3D动作语义，而不是从随机初始化开始——LIBERO 上提升 +4.6~5.4%。

1问题：直接把旧格子用于新机器人

预训练在 OXE+RH20T 的综合动作分布上切格子。新机器人（比如 Franka）的动作分布可能完全不同——比如工作空间更小、旋转范围不同。如果直接用预训练的格子：

  • 新机器人的高频动作落在预训练格子的稀疏区 → 低精度
  • 格子 embedding 的空间几何关系与新机器人不匹配 → 初始化差

2三线性插值初始化

核心思路：新格子的中心点在预训练的3D动作空间里是"有坐标的"，因此可以用周围格子的 embedding 加权平均来初始化：

\[ e^i_{a_{\text{new}}} = \sum_{j=1}^{K} w_j \cdot e^j_a \]

其中 \(w_j\) 按新格子中心点与相邻预训练格子中心点的归一化距离计算（近者权重大）。这相当于在格子空间做双线性/三线性插值，保留了预训练 embedding 中学到的"3D运动方向语义"。

📌 工程意义

这是 SpatialVLA 区别于"只是大模型 LoRA 微调"的关键创新点：通过 embedding 插值初始化，小数据（如 LIBERO 的50条示范/task）也能快速收敛，避免了大量 fine-tuning 步骤带来的遗忘风险。

VLA Vision-Language-Action

视觉-语言-动作模型 · 机器人通用策略的当前范式

📌 TL;DR

把预训练 VLM（视觉+语言）接上一个动作预测头，在机器人演示数据上微调，使模型能根据图像观测和语言指令直接生成机器人动作。核心假设：VLM 的视觉/语言理解能力可以"迁移"到操控任务。

1发展脉络

模型VLM 骨干动作表示特点
RT-2 (2023)PaLI-X (55B)线性256-bin，7 tokens首个大规模 VLA，闭源
OpenVLA (2024)Prismatic (7B)线性256-bin，7 tokens开源，OXE 训练
π0 (2024)PaliGemma+ExpertFlow Matching 连续动作高精度，JAX
SpatialVLA (2025)PaliGemma 2 (3.5B)Adaptive Grids，3 tokens3D感知，开源，快

2通用 VLA 的核心挑战

  • 观测异构：不同机器人的相机位置、分辨率各异 → SpatialVLA 用 Ego3D PE 对齐
  • 动作异构：不同机器人的动作空间（关节/末端/速度）差异巨大 → SpatialVLA 用 Adaptive Grids 统一
  • 数据规模：单机器人数据不足 → 跨本体数据集（OXE、RH20T）
  • 推理延迟：VLM 自回归推理慢 → SpatialVLA 3 tokens/step，20Hz

PaliGemma 2 VLM 骨干

Google PaliGemma 2 · SpatialVLA 的视觉-语言底座

📌 TL;DR

PaliGemma 2 = SigLIP（视觉编码器）+ Gemma 2（语言模型）的组合。SigLIP 提取语义丰富的图像 patch token，Gemma 2 以 LLM 方式自回归生成文本和动作 token。SpatialVLA 在其上添加 Ego3D PE 和 Adaptive Action Grids，动作 token 作为新的词表类别。

1结构

  • SigLIP（Sigmoid 损失的 CLIP）：图像 → patch token，对 contrastive 预训练鲁棒，在机器人任务中通常比 CLIP 更稳定
  • Gemma 2：谷歌开源 LLM，自回归 next-token prediction
  • SpatialVLA 共 3.5B 参数：远小于 RT-2-X 的 55B，但性能更好（空间表示设计功劳）

2为什么选 PaliGemma 2 而非其它 VLM

π0（2024）也用 PaliGemma（初代），表明这个骨干在机器人任务上有良好的预训练基础。SpatialVLA 用 PaliGemma 2（改进版），并冻结文本 token embedding（保留语言理解能力），只训练 Ego3D PE 的 MLP 和 spatial action token embedding。这种"最小改动"策略在论文消融中被证明有益于指令跟随能力。

3D-aware Control 空间感知控制

3D空间感知控制 · 从2D视觉策略到3D空间策略的范式转移

📌 TL;DR

让机器人策略不仅"看到"物体，还"知道"物体在3D空间哪里——使其对视角变化、光照变化、相机位姿变化更鲁棒，并能执行"放到最近的/最高的/左边的"等空间指令。SpatialVLA 是首个在 VLA 框架内同时在输入和输出两端注入3D信息的工作。

1为什么2D策略不够

当相机角度变化时，2D图像中的像素位置完全不同，但3D坐标（相对机器人）几乎不变。2D策略必须靠暴力数据增强"记住"各种视角，而3D感知策略天然具备视角不变性。

论文消融数据（Table IV, row #8 vs #1）：去掉 Ego3D PE 后，variant aggregation（视角/纹理/光照变化测试）下降 12.7%——这正是3D信息提供的不变性在发挥作用。

2SpatialVLA vs 其他3D基础模型方案

方案3D信息来源动作空间是否端到端
3D-LLM点云提取器语言规划否（需低层策略）
3D-VLA3D场景图离散高层动作部分
LEO2D图像 + 点云语言/导航否
SpatialVLA单目深度估计（无额外硬件）连续6DOF+grip是

OXE + RH20T 跨本体数据集

Open X-Embodiment + RH20T · SpatialVLA 预训练数据

📌 TL;DR

SpatialVLA 预训练使用 OXE（来自22种机器人/21个机构）的子集 + RH20T（20万+真实机器人轨迹），共计 1.1M episodes，28个数据集。数据混合权重根据各子集在真实环境的测试性能调优（而非按数据量比例）。

1Open X-Embodiment (OXE)

  • Google DeepMind 联合21个机构构建，覆盖 UR5、Franka、WidowX、Google Robot 等22种机器人
  • 数据质量参差不齐：论文指出 Kuka 数据有明显右倾偏差，RH20T 数据缺清晰 prompt
  • 论文在预训练最后1/3阶段移除 DROID 数据集（按 OpenVLA 经验），改善最终质量

2RH20T

  • 来自上海 AI 实验室（本文同机构）的大规模真实机器人轨迹集
  • 包含多类型机器人、多任务场景，与 OXE 互补

3数据混合比例调优

沿用 OpenVLA 的混合策略，但根据各子集在真实环境测试性能重新调整权重——这是重要的工程细节：直接按数据量混合会让低质量/领域外数据稀释高质量数据的影响。具体权重见论文 Appendix A。

📌 对你的意义

OXE + RH20T 均已公开。如需在 SpatialVLA 基础上微调到新机器人（如人形操控臂），可参考论文的 Spatial Embedding Adaption 后训练方案：只需在新机器人的50条演示上 LoRA 微调，配合格子 re-discretization，即可快速迁移。
