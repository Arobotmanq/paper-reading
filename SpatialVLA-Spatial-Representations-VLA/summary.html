SpatialVLA:
空间表示赋能 VLA

现有 VLA 只"看"2D 图像，缺乏对3D物理世界的真实感知——SpatialVLA 的核心思想是：把 3D 空间信息同时注入观测端（Ego3D Position Encoding）和动作端（Adaptive Action Grids），让机器人策略在预训练时就学会跨本体的空间行动知识，使迁移和泛化变得又快又好。

💡 一句话思想
现有 VLA 把机器人操控当做"看2D图、输数字"的问题，但人操纵物体靠的是对3D空间的直觉感知。SpatialVLA 的答案是：在观测侧用 Ego3D Position Encoding 把深度点云融入视觉 token，在动作侧用 Adaptive Action Grids 把连续运动离散成贴合真实动作分布的空间格子——两端同时"注入3D"，让 VLA 骨干在 1.1M episodes 预训练中就学会空间感知的机器人通用策略。
🔗 逻辑链 · 为什么这个想法成立
  1. 现有 VLA 模型仅接受 2D 观测输入，缺乏对 3D 物理世界的精确感知——而人操控物体天然依赖 3D 空间心理模型。
  2. 跨本体的难点有两个：观测端各机器人相机位姿不统一（无法3D对齐），动作端各机器人运动范围差异巨大（无法统一离散化）。
  3. Ego3D Position Encoding 用自我中心相机坐标系估计深度并反投影，规避外参标定——对任意 embodiment 普适。
  4. Adaptive Action Grids 对整个预训练数据拟合 Gaussian 分布，按等概率原则切分格子——让高频动作区域有更细粒度的离散格子，3 个 token 替代 7 个 token，推理快 4×。
  5. 预训练完成后，格子可以用新数据的 Gaussian 重新离散化（re-discretization），并用三线性插值初始化新 token embedding——快速迁移到新机器人，无需从头微调。
🧩 方法骨架 · 三个关键设计决策
①Ego3D Position Encoding：用 ZoeDepth 估深度 → 反投影得到自我中心3D坐标 P → 正弦编码 γ(P) → MLP → 加到 SigLIP 的 2D patch token 上，得到 O3d = X + MLP(γ(P))。无需外参标定，跨本体通用。
②Adaptive Action Grids：7D 动作 → 分为 ΔT(极坐标 ϕ,θ,r) + ΔR(roll,pitch,yaw) + grip → 各维度拟合 Gaussian 分布 → 等概率切分 M 份 → 学习可训练 spatial token embedding，共 V=8194 个 token，每步只预测 3 个。
③Spatial Embedding Adaption（post-training）：对新机器人数据重拟 Gaussian → 生成新格子 Gnew → 三线性插值从预训练 token 初始化新 token → LoRA 微调。LIBERO 上比 LoRA-only 提升 +4.6~5.4%。
✓ 效果 = 验证（不是重点）

◆核心观点 · 证据

每条 = 中文论点 + 📍页/节 + 🔤逐字原文(Ctrl-F 锚点) + 🀄译文 + 💡解读。点开原文即可最短路径核对。

观点1 真问题：VLA 模型局限于 2D 观测输入，缺乏对 3D 物理世界精确感知和理解的能力，而这正是操控任务的关键。
📍 p.2 · §I Introduction
existing VLA models are primarily confined to 2D observation inputs and lack precise perception and comprehension of the 3D physical world — where humans instinctively construct rich, structured mental representations of space, effortlessly aligning objects within a canonical, intuitive, and even personally tailored workspace for manipulation
现有 VLA 模型主要局限于 2D 观测输入，缺乏对 3D 物理世界的精确感知与理解——而人类本能地在大脑中构建丰富、结构化的空间心理表示，毫不费力地在规范化、直觉性甚至个性化的工作空间中对齐物体进行操控。
这是全文的核心动机：把问题从"2D 感知 → 动作预测"升维为"3D 空间感知 → 空间动作预测"。
观点2 两大挑战：跨本体 VLA 实现3D空间智能面临观测端（相机不对齐）和动作端（动作空间异质）两个根本困难。
📍 p.2 · §I Introduction
the observations from different robot embodiments are not 3D-aligned, because the camera sensors of different robots are various and mounted at different places (e.g. wrist and/or third-person), resulting in non-calibrated 3D observation spaces. Secondly, different robots have different action movement characteristics to accomplish diverse tasks, due to different degrees of freedom, motion controllers, workspace configurations, and task complexity, leading to significant difficulty in learning generalizable spatial actions.
来自不同机器人本体的观测在3D上是未对齐的，因为不同机器人的相机传感器各异且安装位置不同（如腕部和/或第三人称视角），导致3D观测空间未经标定。其次，不同机器人由于自由度、运动控制器、工作空间配置和任务复杂性不同，具有不同的动作运动特性，导致学习可泛化空间动作面临重大困难。
论文将问题切分为"观测端3D对齐"和"动作端3D统一"两个子问题，对应 Ego3D PE 和 Adaptive Action Grids 两个核心创新。
观点3 Ego3D Position Encoding：在自我中心相机坐标系中估深度并反投影，无需外参标定，把3D点云融入 SigLIP 的 2D 视觉 token，跨本体通用。
📍 p.3–4 · §III-A The SpatialVLA Model Architecture
The proposed Ego3D position encoding integrates depth information from the camera frame and image pixels to construct an egocentric 3D coordinate system, which eliminates the need for robot-camera extrinsic calibration and is agnostic to specific robot setups. Specifically, we use ZoeDepth to estimate depth map D and obtain pixel's 3D position p = {x, y, z} in the egocentric 3D coordinate system via back-projection π−1 with camera intrinsic parameters.
所提出的 Ego3D 位置编码将相机帧的深度信息与图像像素相结合，构建自我中心3D坐标系，从而无需机器人-相机外参标定，且与特定机器人设置无关。具体地，我们使用 ZoeDepth 估计深度图 D，并通过相机内参利用反投影 π⁻¹ 获得像素在自我中心3D坐标系中的3D位置 p = {x, y, z}。
关键设计选择：用单目深度估计（ZoeDepth）而非 RGB-D 相机，保留了与现有机器人硬件的兼容性；"自我中心"坐标系规避了依赖机器人底座的外参标定问题。
📍 p.4 · §III-A (Equation 2)
The egocentric 3D spatial representations O3d ∈Rd×h×w are obtained by adding 3D position embedding P′ and 2D path visual tokens X, depicted as follows, O3d = X + P′ = X + MLP(γ(P)).
自我中心3D空间表示 O3d 通过将3D位置嵌入 P′ 与2D图块视觉 token X 相加得到：O3d = X + P′ = X + MLP(γ(P))。
加法融合（而非拼接或交叉注意力）保持了与 PaliGemma 2 预训练格式的兼容——这是工程上的重要考量：改动尽量小，复用 VLM 的预训练知识。
观点4 Adaptive Action Grids：对整个预训练数据拟合 Gaussian 分布，按等概率原则切分动作格子，让高频动作区域自动获得更细粒度——3 个 token 代替 7 个 token，速度快 4×，精度更高。
📍 p.4 · §III-A Adaptive Action Grids
for tokenizing continuous translation and rotation movements, we first normalize each action variable into [−1, 1] for each robot environment and statistic the translation and rotation movements ∆R = {roll, pitch, yaw}, ∆T = {ϕ, θ, r} on the whole dataset mixture, following with a parameterized Gaussian distribution fitting N(µa, Σa). Then, the continuous actions are split into M intervals Gi=1,..,M = {[a1 = −1, a2), ..., [aM-1, aM = 1]} with equal probability 1/M on each normalized action variable
为了对连续的平移和旋转运动进行 tokenize，我们首先将每个机器人环境中的每个动作变量归一化到 [−1,1]，并统计整个数据集混合上的平移和旋转运动（ΔR、ΔT），然后拟合参数化 Gaussian 分布 N(μₐ, Σₐ)。接着，将连续动作按每个归一化动作变量的等概率 1/M 切分为 M 个区间 Gᵢ₌₁,..,M。
"等概率"而非"等间距"是核心：这使高频的小幅动作区域格子更密（精度高），低频的大幅动作区域格子更稀——自适应地将离散化预算分配给信息量最大的区域。
📍 p.4 · §III-A (after Eq.5)
it is worth noting that the model only needs to generate 3 tokens for one-step robot actions rather than 7 tokens as in RT-1, RT-2 and OpenVLA, achieving in fast model inference speed.
值得注意的是，该模型对一步机器人动作只需生成 3 个 token，而不是像 RT-1、RT-2 和 OpenVLA 那样需要 7 个 token，从而实现了更快的模型推理速度。
把 7 维动作 → 3 个 spatial token（ΔT 1个 + ΔR 1个 + grip 1个）是本文效率优势的直接来源：词表虽大（8194），但每步只查询 3 次，推理速度 20Hz vs OpenVLA 的 5Hz。
观点5 Spatial Embedding Adaption（后训练）：对新机器人数据重拟 Gaussian 生成新格子，用三线性插值从预训练 token 初始化新 token，加速收敛并提升小数据微调性能。
📍 p.5 · §III-B Post-training Designs
we fit a new Gaussian distribution N(µnew, Σnew) for each action variable on post-training datasets and create discrete spatial action grids Gnew in translation and rotation movement to construct action grids Gnew and tokens anew, where the embeddings of new spatial action tokens Eanew are initialized by trilinear interpolation with pre-trained action tokens Ea.
我们对后训练数据集中每个动作变量拟合新的 Gaussian 分布 N(μnew, Σnew)，在平移和旋转运动中创建离散空间动作格子 Gnew 以构建动作格子和 token anew，其中新空间动作 token 的嵌入 Eanew 通过与预训练动作 token Ea 的三线性插值初始化。
三线性插值意味着：新机器人的动作格子中心点在预训练的3D动作空间里找 K 个最近邻格子，加权平均其 embedding 作为初始值——这保留了预训练中学到的"3D动作语义"，让新 token 从"有意义的起点"开始优化，而非随机初始化。
观点6 预训练数据：1.1M episodes 来自 OXE 和 RH20T 的 28 个数据集，覆盖多种机器人本体、场景和任务，训练 10 天（64× A100）。
📍 p.5 · §III-B Pre-training Procedure
We train SpatialVLA from Paligemma2 backbone on a cross-robot dataset mixture with 1.1 Million real robot demonstrations {ζ1, ..., ζn}, covering a diverse range of robot embodiments, scenes, and tasks. This pre-training dataset mixture consists of a subset of OXE and the RH20T dataset and we modify the mixture weights from OpenVLA according to the real-word testing performance of individual dataset
我们以 PaliGemma2 骨干为基础，在包含 110 万条真实机器人示范的跨机器人数据集混合上训练 SpatialVLA，覆盖多种机器人本体、场景和任务。该预训练数据集混合由 OXE 的子集和 RH20T 数据集组成，我们根据各数据集在真实环境测试性能修改了来自 OpenVLA 的混合权重。
关键细节：文本 token embedding 在预训练中冻结（保留 VLM 的语言理解）；只有 Ego3D PE 的 MLP 参数和 spatial action token embedding 是随机初始化并优化的——这减少了训练不稳定性。
观点7 主结果：SpatialVLA 在 SimplerEnv 零样本评测超越 RT-2-X（55B → 3.5B，参数量少15倍），并在 WidowX 真实机器人上展现最高平均成功率，证明 3D 空间表示对泛化的决定性作用。
📍 p.6 · §IV-A SimplerEnv Evaluation
Our SpatialVLA model yields 71.9% and 75.1% Visual Matching scores in zero-shot and fine-tuning settings, surpassing the second-best policy, RoboVLM, by +15.6% and +11.7% margins. Notably, our model trained from scratch on OXE mixture with RH20T surpasses the state-of-the-art closed-source model RT-2-X, achieving superior performance in Visual Matching (71.9% vs 60.7%) and Variant Aggregation (68.8% vs 64.3%), while using significantly fewer model parameters (3.5B vs 55B).
我们的 SpatialVLA 模型在零样本和微调设置下分别获得 71.9% 和 75.1% 的 Visual Matching 分数，比第二名 RoboVLM 分别高出 +15.6% 和 +11.7%。值得注意的是，我们从头在 OXE 混合数据集加 RH20T 上训练的模型超越了最先进的闭源模型 RT-2-X，在 Visual Matching（71.9% vs 60.7%）和 Variant Aggregation（68.8% vs 64.3%）上都取得更优性能，同时使用的模型参数量大幅减少（3.5B vs 55B）。
55B 的 RT-2-X 是闭源的谷歌模型，3.5B 的 SpatialVLA 超越它是量级差异上的强有力论据，表明空间表示设计的价值远超单纯的参数规模扩展。
观点8 消融证明：去掉 Ego3D Encoding 导致 variant aggregation 下降 12.7%；Adaptive Action Grids 比线性 256-bin 高 36.5%——两个创新各自独立贡献显著。
📍 p.9 · §IV-D Ablations on Design Decisions
In contrast to the conventional linear 256-bin action space discretization (#1v.s.#2), the proposed adaptive spatial action grids exhibits significant advantages, particularly in the Google Robot task, with the promotion of +36.5% and +42.1% in variant aggregation and visual matching success rates, respectively.
与传统的线性 256-bin 动作空间离散化（#1 vs #2）相比，所提出的自适应空间动作格子展现出显著优势，特别是在 Google Robot 任务上，variant aggregation 和 visual matching 成功率分别提升了 +36.5% 和 +42.1%。
这是论文最关键的消融实验：原有 VLA（OpenVLA 等）用线性 256-bin 离散化，SpatialVLA 的自适应 Gaussian 格子超出幅度之大（+36.5%），表明"如何把连续动作离散化"本身是一个被严重低估的设计变量。
📍 p.9 · §IV-D (Table IV, row #8)
the proposed egocentric 3D position encoding (ego3d), incorporating 3D point cloud features, helps the model overcome varied lighting, color, textures, and camera poses, yielding stronger generalizability in diverse manipulation scenarios. Models w/o ego3d suffer a significant performance drop in variant aggregation, from 81.6% and 79.2% to 68.9% and 66.7%, due to their inability to adapt to scene changes.
所提出的自我中心3D位置编码（ego3d）融合了3D点云特征，帮助模型克服不同的光照、颜色、纹理和相机位姿变化，在多样化操控场景中具有更强的泛化能力。没有 ego3d 的模型在 variant aggregation 上出现显著性能下降，从 81.6% 和 79.2% 降至 68.9% 和 66.7%，原因是无法适应场景变化。
"variant aggregation" 任务专门测试模型对光照/颜色/纹理变化的鲁棒性——正是 Ego3D PE 能提供与外观无关的3D几何信息的场景，故消融差距最大在这里出现，印证了设计动机。

⚖批判 · 评级

✅ 值得读的理由

SpatialVLA 是迄今少数同时解决 VLA 的"观测端3D对齐"和"动作端3D统一"两个跨本体核心难题的工作，且都以极简的方式实现（加法 PE、Gaussian 格子离散化），无需额外的 RGB-D 硬件或相机标定。3 tokens/step 的推理设计使其在 20Hz 运行（实用性强）；代码权重全开源（MIT）；消融实验充分，Adaptive Action Grids vs 线性256-bin 的 +36.5% 差距是关键说服力数据。RSS 2025 接收，是当前 VLA 社区的重要参考基线。

🔍 主要保留与局限

① 单目深度估计的误差：ZoeDepth 的深度估计在弱纹理/反光物体上误差较大，论文未分析 Ego3D PE 对深度估计精度的敏感性。② Gaussian 假设的局限（作者自承）：单轴运动场景下 Gaussian 拟合会导致格子在某轴聚集，丧失其它轴的细粒度。③ 长时序任务困难（作者自承）：LIBERO-Long 成功率 55.5%，明显低于其它子任务，因为模型仅依赖当前帧和历史 token，无显式长程规划机制。④ 评测场景局限：评测全为桌面单臂操控，无移动底盘、双臂或腿足场景，泛化到移动操作 VLA 需验证。⑤ 无与 diffusion-based VLA 的直接对比：π0 的流匹配动作 head 在精细操控上通常更优，但论文仅部分对比 π0（且 π0 无开源权重对齐比较）。

评级：Important  方法创新扎实、开源完整、消融充分，但非颠覆性范式突破——是 VLA 空间表示方向的优质参考论文，尤其对 Track A 移动操作 VLA 有直接工程参考价值。
