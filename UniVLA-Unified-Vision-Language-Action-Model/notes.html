Unified Autoregressive VLA Architecture

统一自回归视觉-语言-动作模型 · UniVLA 的核心范式

📌 TL;DR

把图像、语言、动作全部变成同一本词典里的"单词"（离散 token），然后用一个标准 Transformer 做"下一个词预测"——视觉理解、语言对话、机器人控制，全是同一个 loss，同一套权重。

1背景：现有 VLA 的两条路线

路线代表视觉处理动作处理弱点
纯动作预测RT-2, OpenVLA, π0独立 ViT → 连续 embedding离散 token 或 flow matching视觉是"晚期融合"，丢失时序因果
视觉引导动作SuSIE, GR-1, UniPi预测未来帧 → 解码动作从生成帧中反推生成模型与策略模型分离，无法端到端联合优化
统一自回归（UniVLA）UniVLA（本文）VQ tokenizer → 离散 tokenFAST（DCT）→ 离散 token（待验证）高分辨率场景的空间细节受 VQ 压缩影响

2在 UniVLA 里是怎么实现的

模型基础是 Emu3（BAAI，8.5B 参数的自回归多模态 Transformer）。数据格式：

UniVLA 序列格式策略微调阶段
[lang tokens] [boi] [img tokens] [eoi] [boa] [action tokens] [eoa]
              [boi] [img tokens] [eoi] [boa] [action tokens] [eoa]  ...
                 ↑                             ↑
        VQ tokenizer 离散化               FAST（DCT）离散化
        （8× 空间压缩，256×256 → 32×32 token grid）  （词表大小 1024）

训练目标是 cross-entropy next-token prediction，但不同阶段对不同 token 类型开关 loss mask：

  • 世界模型后训练：只对视觉 token 计算 loss
  • 策略微调：只对动作 token 计算 loss
  • 多模态推断（grounding）：只对文本 token 计算 loss

3类比

🎯 直觉类比

想象一个速记员，用同一套速记符号记录"什么东西"（视觉）、"说了什么"（语言）和"做了什么动作"（动作）。现有 VLA 是三个速记员分别记录再汇总；UniVLA 是同一个速记员用同一本符号手册记录全部内容——所以他能感知"昨天拿过这里"（历史动作）和"刚才桌子上出现了新东西"（视觉变化）之间的因果联系。

4出处

统一多模态 token 序列的想法可追溯到 Gato（DeepMind 2022） 和 Unified-IO；图像离散化（VQ tokenizer）源自 VQVAE（van den Oord 2017） 和 DALL-E（OpenAI 2021）。UniVLA 的新意在于把 机器人动作（通过 FAST/DCT 离散化）也纳入同一框架，并加入世界模型后训练。

World Model Post-Training Training Strategy

世界模型后训练 · UniVLA 三阶段训练的第二阶段

📌 TL;DR

在策略微调之前插一个"中间阶段"：用大量无动作标注的机器人视频，训练模型预测"给了这个指令，接下来视觉会变成什么"——这让模型学到时序因果知识，之后策略微调就像站在巨人肩膀上，数据效率大幅提升。

1什么是世界模型

世界模型（World Model）学习环境的状态转移函数：

\[ P(s_{t+1} \mid s_t, a_t) \]

给定当前状态 \(s_t\) 和动作 \(a_t\)，预测下一个状态 \(s_{t+1}\)。UniVLA 用语言指令代替动作（因为此阶段没有动作标注），训练目标变为：

\[ P(s_{t+1}, s_{t+2}, \ldots, s_T \mid s_1, \text{instruction}) \]

2在 UniVLA 里怎么做

世界模型后训练序列格式Sv
输入序列 Sv = {L¹_t, L¹_v, L²_v, ..., Lᵀ_v}
              ↑语言指令  ↑第1帧   ↑第2帧     ↑第T帧

Loss 只算在 L²_v ... Lᵀ_v（未来视觉 token）上
不需要任何动作标注！

用 622K 机器人视频（RT-1、BridgeV2、DROID、CALVIN 等，见 Table 8），32 × A100 训练约 50k 步（4-5 天）。

3消融：为什么是世界模型，不是其他后训练

后训练策略序列监督LIBERO-Long ↑CALVIN ↑
无后训练（基线）——17.41.46
动作预测T, I, A动作10.6 (−6.8)0.52 (−0.94)
文本到图像T, I视觉（单帧）55.8 (+38.4)3.79 (+2.33)
视频预测I₁, …, Iₜ视觉（多帧）80.8 (+63.4)3.59 (+2.13)
世界模型（本文）T, I₁, …, Iₜ视觉（指令+多帧）89.2 (+71.8)4.61 (+3.15)

关键洞察：仅视频（无文字）学不到"因为这个指令导致了这个变化"；仅文字图像对（无时序）学不到运动动态。必须同时有指令（文字）+ 时序视觉（多帧）。

4类比

🎯 直觉类比

就像外科住院医在进手术室之前要先看大量手术录像——他不需要亲自操刀（动作标注），只需要理解"做了这个操作，组织会怎么变化"（世界模型）。有了这些知识，真正上手时的学习曲线会大幅缩短。

FAST Action Tokenizer

Frequency-space Action Sequence Tokenization · 频域动作序列 tokenizer

📌 TL;DR

FAST 用离散余弦变换（DCT）把连续动作序列压缩成一串离散 token——就像把音频压成 MP3，保留最重要的低频成分，截断高频噪声。压缩后的 token 可以直接插进语言模型词表，让 VLM 用自回归方式生成动作。

1为什么需要 action tokenization

机器人动作是高维连续向量（7 DOF 末端位姿 + 夹爪 = 7 维，每步一个），不能直接进 LM 词表。有两条路：

  • 连续方案：flow matching（π0）、Diffusion Policy——精度高，但与语言模型接口不兼容
  • 离散方案（FAST）：先把 chunk（例如 10 步）的动作序列做 DCT，只保留前 N 个低频系数，再均匀量化成 token ID——与语言词表兼容，next-token prediction 直接可用

2DCT 离散化流程

FAST Tokenizer Pipeline简化示意
# 输入：H 步动作序列，每步 d 维
A_{1:H} = {a_1, a_2, ..., a_H}   (H×d 矩阵)

# Step 1: 按维度做 1D DCT（时域 → 频域）
F = DCT(A_{1:H})    → 保留低频系数，截断高频

# Step 2: 归一化到 [0, 1024) 整数
T_n = quantize(F_n)   → token ID，共 n 个

# 词表替换：FAST 占用语言词表最后 1024 个 ID
# (Emu3 的原语言词表最后 1024 位被 FAST token 替代)

UniVLA 用 action chunk size = 10（CALVIN/LIBERO），每个 chunk 产生可变长度 n 个 token（视频序列中 n 由 DCT 精度决定）。

3与连续动作方案的对比

FAST（离散）Flow Matching（π0）
与 LM 接口天然兼容（共享词表）需要单独 action expert
推理速度一次 forward（自回归）需要多步去噪（通常 10-50 步）
动作精度受量化精度限制连续，高精度
长程一致性DCT 低频优先 = 天然平滑需要设计噪声调度
UniVLA 如何用世界模型后训练 + 策略微调未使用（UniVLA 纯离散）

4出处

📜 来源

FAST 由 Pertsch et al. 2025（Physical Intelligence + Stanford） 提出（arXiv:2501.09747）。π0-FAST 是其在 π0 上的应用，LIBERO 平均成功率 85.5%。UniVLA 把 FAST 移植到 Emu3 架构，不同之处：不用 flow matching head，纯粹依赖 next-token prediction，在 LIBERO 上达到 95.5%。

VQ Vision Tokenizer Image Discretizer

Vector-Quantized Image Tokenizer · 向量量化图像离散化器

📌 TL;DR

VQ tokenizer 把一张图像切成小格（patch），每个 patch 在"视觉词典"里找最近的"视觉单词"——输出的是一串整数 ID，就像把图像"翻译"成视觉语言。这些 ID 和文字 token 拼在一起，统一进 Transformer。

1原理：向量量化

VQ（Vector Quantization）的核心操作：

\[ z_q = e_{k^*}, \quad k^* = \arg\min_k \|z_e - e_k\|_2 \]

编码器输出连续向量 \(z_e\)，在 codebook \(\{e_1, \ldots, e_K\}\) 中找最近邻，替换成对应的"视觉词汇" \(e_{k^*}\)，输出整数 index \(k^*\)。

2在 UniVLA 里的具体参数

  • 基于 MoVQ（Zheng et al. 2022），与 Emu3 相同设计
  • 空间压缩因子 8：256×256 图像 → 32×32 token grid = 1024 个视觉 token
  • codebook 大小 32768（视觉词典有 32768 个"视觉单词"）
  • 世界模型后训练时每次输入 6 帧，序列最大长度 6400 token（含语言+多帧视觉）

3与 ViT patch embedding 的区别

VQ Tokenizer（UniVLA）ViT Patch Embedding（OpenVLA/π0）
输出类型离散整数 ID，可进 LM 词表连续 embedding 向量
与语言统一天然统一（共享词表）需要额外 projector 对齐
信息损失有量化损失（特别是细节/纹理）无量化损失，但语义对齐靠训练
生成能力解码器可重建图像（世界模型输出）不支持图像生成

4为什么 VQ 对世界模型至关重要

世界模型训练的监督目标是"预测未来视觉 token"——这要求视觉输出必须是可预测的离散符号，不能是连续向量（连续向量无法用 cross-entropy 直接做 next-token prediction）。VQ tokenizer 正是把图像变成"视觉语言"的关键一步。

Cross-Embodiment Generalization Transfer

跨本体泛化 · 从多种机器人平台的数据中学习可迁移的策略

📌 TL;DR

用来自多种机器人（单臂、双臂、移动臂等）的数据训练同一个模型，让它学到跨平台通用的视觉-动作知识——UniVLA 通过统一 token 框架自然实现：不同机器人的动作、视觉全部进同一词表，共享 Transformer 权重。

1UniVLA 里的跨本体数据

622K 后训练视频来自 14 个数据集，涵盖：

  • 真实单臂：RT-1（Google）、BridgeV2（伯克利 WidowX）、DROID（多机构 Franka）
  • 真实双臂/工业：Kuka、FMB、Berkeley UR5 等
  • 仿真多任务：CALVIN（Franka 4 环境）、LIBERO、ManiSkill2
  • 人类视频：SSV2（Something-Something v2，220K 视频，无动作标注）

策略微调阶段（ALOHA 真机）用 8 类任务约 4000 条轨迹，平台是 AgileX Cobot Magic V2.0 双臂机器人（14 维动作，3 个 RGB 相机）。

2与 π0.5 跨本体方案的对比

UniVLA（本文）π0.5（Physical Intelligence）
统一方式统一 token 词表，所有模态共享异构 co-training，flow matching 动作头
视觉处理VQ tokenizer（无独立 ViT）SigLIP ViT（400M 参数）
世界模型显式后训练阶段（视觉 token 预测）无专门世界模型阶段
真机评测ALOHA 双臂（定性，8 类任务）移动操作机器人（定量，3 个真实家庭）
优势场景长程任务、数据高效开放世界家庭泛化

3局限性

⚠️ 需要注意

UniVLA 的跨本体泛化目前主要在仿真基准上验证，真机实验（ALOHA）仅定性展示，没有与其他方法的定量对比。不同机器人平台的动作空间差异（关节角 vs 末端位姿，不同自由度）通过 FAST 统一离散化来处理，但高精度操作（毫米级放置）可能受 VQ 图像压缩限制。

4对 Track A（移动操作 VLA）的启示

如果你的机器人平台没有大量自采数据，UniVLA 的路线提供了一个可行配方：

  1. 用已有开源机器人视频（无动作标注）做世界模型后训练，初始化时序理解
  2. 在目标平台上少量微调（10% 数据可超全数据 GR-1）
  3. UniVLA 权重和代码已在 HuggingFace + GitHub 开源，直接 fine-tune 可行

Visual Prediction Loss Fine-tuning Trick

策略微调中的视觉预测辅助 loss · 微调阶段保留视觉 token 的监督信号

📌 TL;DR

在策略微调阶段，除了对动作 token 计算 loss，同时以 0.5 的权重保留对视觉 token 的预测 loss——即使不做完整的世界模型后训练，加这一项辅助 loss 也能显著提升策略性能（CALVIN +0.19 avg）。

1为什么有效

UniVLA 采用视频格式的交错序列微调（而非单帧），每个时间步都有视觉 token 和动作 token 交错出现。在自回归框架下，模型在预测每个动作 token 之前已经"看到"了当前帧的视觉 token——如果同时让模型也预测下一帧的视觉 token，就自然地引入了"这个动作会导致什么视觉变化"的世界模型训练信号，哪怕此时后训练阶段已经完成。
