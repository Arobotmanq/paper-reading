Gradient-Bridged Co-Training 本文核心机制

梯度桥联合训练 · 离散 CE 当桥、多模态 CE 当锚、flow matching 当部署接口

📌 TL;DR

用离散动作 CE 当"梯度桥"，把下游动作信号有力地传回 VLM 骨干（因为它和 VLM 自回归预训练接口同构）；多模态 CE 当"锚"保住接地能力（更新方向与动作优化大致正交）；flow matching 只训练推理用的连续动作生成器。三者单阶段端到端联合优化、不切断梯度。

1它要破解的张力

VLA 训练有个固有矛盾：连续 flow matching 是天然的执行接口（直接建模未量化动作），但对 VLM 骨干的更新很弱；离散动作 token 的 next-token CE 原生于 VLM 接口、能强烈塑形骨干，但解码出的离散动作太粗糙、无法精确控制；而冻结/截断梯度（stop-gradient）虽保护 VLM 先验，却阻止精确动作目标去塑形骨干。梯度桥就是化解这一矛盾的配方。

2三个损失的分工

损失角色对骨干更新的方向 / 份额
离散动作 CE (Lact-CE)梯度桥主导更新（与 VLM 自回归接口同构 → 强）；方向与 flow 正相关
多模态 CE (Lmm-CE)接地锚方向与动作优化大致正交，互补而非竞争
flow matching (Lflow)部署接口越过早期后仅约 5%；但小残差对动作质量仍重要

复合目标（论文 Eq.1）：

\[ \mathcal{L} = \mathcal{L}_{\text{flow}} + \lambda_{\text{act}}\,\mathcal{L}_{\text{act-CE}} + \lambda_{\text{mm}}\,\mathcal{L}_{\text{mm-CE}}, \quad \lambda_{\text{act}}=\lambda_{\text{mm}}=0.01 \]

Lflow 在 Action-Space Supervision 下比 CE 项小约两个数量级，0.01 把两个 CE 拉到与 flow 可比的尺度；动作:多模态数据按 9:1 batch 混合。离散 token 来自 Vision-Aligned RVQ tokenizer。

3与 π₀.₅ 的 stop-gradient 对比

π₀.₅ (stop-gradient / knowledge insulation)Wall-OSS (gradient bridge)

🎯 为什么是"桥"不是"隔离"

离散 CE 的梯度方向与 flow 正相关，所以它替 flow 把骨干往"控制相关特征"方向推，flow 再在已被塑形好的特征上精修动作。stop-gradient 切断了这条路，骨干学不到控制相关表示，故下游更差。

Vision-Aligned RVQ Action Tokenizer

Vision-Aligned Residual Vector Quantization · 视觉对齐的残差向量量化动作 tokenizer

📌 TL;DR

用残差向量量化(RVQ) 把"观察条件化的 delta 动作块"压成多级离散 token；靠三个辅助目标（视觉-动作对齐 / 未来帧预测 / DCT 域重建）让 token 语义化，取代规则化的 FAST tokenizer。这些离散 token 正是 梯度桥 喂给 VLM 的信号。

1RVQ 是什么

普通 VQ（如 VQ-VAE）把一个向量量化到单个码本里的最近码字，量化误差大。残差向量量化用多级码本：第一级量化得到粗近似，把残差送入第二级再量化，逐级逼近——早期码本捕获粗运动结构，后期码本做细残差修正。

RVQ 逐级量化示意
z (连续动作潜变量)
  ├─ q0 = 最近码字(码本0)         # 粗运动
  ├─ r1 = z - q0 ;  q1 = 最近码字(码本1)   # 残差修正
  ├─ r2 = r1 - q1; q2 = 最近码字(码本2)
  └─ ...  → token 序列 [q0, q1, ..., qk]  (Token×N)

2在 Wall-OSS 里的结构与用法

delta-action 空间的 Encoder–RVQ–Decoder（论文 Figure 3）：

  • Encoder：用 temporal cross-attention 压缩"观察条件化的动作块"；Context Encoder（冻结）吃稀疏多视角视频，与 Action Encoder 经对比学习对齐。
  • RVQ：多级码本量化成离散 token。
  • Decoder + Feature Predictor：重建动作块 + 预测未来观察特征。

三个辅助目标（超越单纯重建）：① 视觉-动作对齐（把动作潜变量拉向 VLM 视觉特征）② 未来帧预测（编码动作后果）③ DCT 域重建（抑制高频抖动）。结果是一个可重建、视觉对齐、物理平滑的离散动作表示。

4出处

RVQ / RQ-VAE 源自 Lee et al. 2022（"Autoregressive Image Generation using Residual Quantization"），最早用于图像 tokenization；音频领域的 SoundStream/EnCodec 也用 RVQ。本文把它迁到动作空间并加视觉对齐目标。

Action-Space Supervision flow matching 损失

动作空间监督 · 把 flow-matching 损失定义在恢复出的动作上而非速度场

📌 TL;DR

标准 flow matching 在速度场上算 L2 损失；本文改成在"恢复出的干净动作"上算损失，等价于给速度损失加一个 \((1-\tau)^2\) 权重——强调高噪声步（那里形成动作轨迹的全局形状）。

1两种损失

线性高斯路径：\(A^{\tau} = \tau A + (1-\tau)\epsilon\)（\(\tau{=}0\) 纯噪声，\(\tau{=}1\) 干净动作）。网络 \(f_\theta\) 输出速度预测。

\[ \text{标准(速度空间)：} \ \mathbb{E}\big[\|f_\theta(A^\tau,\tau)-(A-\epsilon)\|^2\big] \]
\[ \text{本文(动作空间)：} \ \hat{A}=A^\tau+(1-\tau)f_\theta(A^\tau,\tau), \quad \mathcal{L}_A=\mathbb{E}\big[\|\hat{A}-A\|^2\big] \]

代入展开可证 \(\mathcal{L}_A = \mathbb{E}\big[(1-\tau)^2\|f_\theta(A^\tau,\tau)-(A-\epsilon)\|^2\big]\)，即速度损失的 \((1-\tau)^2\) 加权版。

2为什么——机器人动作的频谱结构

机器人动作序列低维且平滑，任务相关结构主要在低频轨迹形状而非高频细节（与自然图像不同——图像高低频都含丰富语义）。因此高噪声区（恢复轨迹全局形状处）决定生成质量上限，应重点监督；\((1-\tau)^2\) 权重恰好在 \(\tau\to 0\)（高噪声）时最大。

3与扩散 x-prediction 的关系

形式上类似扩散模型里的 x-prediction（预测干净样本 \(x_0\) 而非噪声/速度），但本文的动机不同：不是出于方差/信噪比考量，而是来自机器人动作的频谱结构（低频主导）。

4效果

LIBERO 消融：action-space loss 在 25k step 达 96.5% 峰值（超 velocity-space 峰值 6.2%），20k step 即 95.8%，而 velocity 跑完 35k 仍未过 90.3%；且 20k 后稳定在 92.5–96.5%。→ 提升收敛速度、峰值、训练稳定性。它也是把 Lflow 压到比 CE 小两个数量级、从而让 梯度桥 的 0.01 权重生效的前提。

Muon / DMuon 优化器

Muon（动量正交化优化器）· DMuon（其分布式实现）

📌 TL;DR

Muon 对 momentum 矩阵做正交化（Newton–Schulz 迭代），产生谱归一化的更新——对梯度幅值不变。这对"VL Expert 与 Action Expert 梯度尺度差异极大"的设定很关键。DMuon 是面向混合并行训练的分布式实现，把 Muon 引入的开销从 ~2× 压到 ~0.02×。

1Muon 原理

对每个 2D 权重矩阵 \(W\)，先算带 momentum 的梯度矩阵 \(M\)，再对 \(M\) 做正交化（用 Newton–Schulz 迭代近似 \(M(M^\top M)^{-1/2}\)），得到奇异值≈1 的更新方向。效果：更新谱归一化、对梯度幅值不变（梯度放大 10× 也不改变更新方向/步长）。这正是它优于 AdamW 之处——AdamW 的逐元素自适应学习率对不同子模块尺度敏感。

2在 Wall-OSS 里干什么

  • Muon 管两个 expert 的 2D 参数；AdamW 管视觉嵌入和 LM head。
  • 为何关键：Action Expert 与 VL Expert 的梯度尺度差异极大（前者从零学动作生成，后者是预训练 VLM）；Muon 的"幅值不变"让两者能用同一套优化稳定共训。

3DMuon — 让它在大规模可用

Muon 的正交化要做矩阵级 Newton–Schulz，朴素实现把 forward+backward 时间拉到 ~2×。DMuon：

  • 用 Longest-Processing-Time-first 负载均衡把每个矩阵参数分给唯一 owner rank；
  • 在 owner 上做一次矩阵级 Newton–Schulz，再 reduce-to-owner + 异步 broadcast（专用 CUDA stream，与下一次 forward 重叠）；
  • owner 本地用对称感知 kernel + 形状自动调优加速。

结果：把额外开销从 ~2× 降到约 0.02×（约 100× 缩减）。

📜 出处

Muon 见 Liu et al. 2025（"Muon is scalable for LLM training"，arXiv:2502.16982）；DMuon 是本文在其上做的分布式工程扩展。配合 梯度桥 的多损失/多专家训练，优化器的尺度不变性尤其有价值。
