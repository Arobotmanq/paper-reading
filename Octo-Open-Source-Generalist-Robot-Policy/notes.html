Generalist Robot Policy GRP

通用机器人策略 · 跨本体/跨任务的统一控制模型

📌 TL;DR

GRP 是一个单模型控制多种机器人完成多种任务的策略，相对于"一个任务 = 一个模型"的传统方式。Octo 是迄今第一个既能做 GRP 又完全开源的操作策略。

1定义

传统机器人学习方法：针对每个机器人、每个任务单独采集数据、从头训练。GRP 的目标是打破这个"任务 → 数据 → 训练"的闭环，训练一个可以零样本或少样本迁移到新机器人和新任务的策略模型。

论文将 GRP 定义为："能跨任务、环境和机器人系统执行低层视觉运动控制的模型"，强调低层控制（直接输出关节/末端执行器动作）而非高层规划。

2在 Octo 里干什么

Octo 是 GRP 的一个具体实现，其"通用性"体现在三个层面：

  • 跨任务：语言或目标图像指令，同一模型处理 pick、place、wipe、open drawer 等不同任务。
  • 跨观测：第三视角相机、手腕相机、力矩传感器——任意组合，缺失的全零 mask。
  • 跨动作空间：端点位置控制、关节位置控制、双臂 14DoF——微调时只换 action head，骨干保留。

4类比

把 GRP 比作 NLP 领域的"预训练语言模型"：就像 BERT/GPT 在大量文本上预训练，再在具体下游任务上微调；GRP 在大量机器人轨迹上预训练，再微调到具体机器人/任务——把"零样本/少样本迁移"的思路从语言/视觉移植到机器人控制。

Readout Token Architecture

读出 Token · 被动汇聚嵌入的模块化输出接口

📌 TL;DR

Readout token 是插入 transformer 序列中的特殊 token：它只读不写——关注前面所有 observation/task token 但不被任何人关注，从而把序列信息压缩成紧凑向量，再送入轻量 action head。新增/移除观测模态时骨干权重完全不受影响。

1概念

标准 transformer 中每个 token 既读（attention 到别人）也写（被别人 attention）。Readout token 打破了这个对称：

  • TR,t 可以 attend 其之前的所有 observation token 和 task token。
  • 所有 observation/task token 不能 attend TR,t。

效果：readout token 汇聚了全局信息，但不会"污染"其他 token 的计算——就像一个只读的全局汇总节点。

2在 Octo 里干什么

Octo 在每个时间步插入一个 readout token TR,t。Transformer 处理完序列后，TR,t 的输出嵌入 eR,t 被送入 diffusion action head，预测从时间 t 开始的 action chunk。

关键好处：微调时添加新观测类型（例如力矩传感器）→ 只需为这个新观测加一个新 tokenizer 和 position embedding；旧的 readout token 和 action head 不需要修改。这是"无痛插拔"的机制基础。

3出处与对比

📜 出处

设计灵感来自 BERT 的 [CLS] token：[CLS] 也是序列开头的特殊 token，用于捕获全局语义信息。Readout token 是这个思想在时序机器人控制场景里的推广——每个时间步一个，因果地汇聚历史信息。

4具象例子

类比"会议记录员"：readout token 是一个只记录不发言的秘书——听完所有人（observation token / task token）的发言后，把内容浓缩成一份摘要（嵌入向量），交给决策者（action head）。加入新的发言人（新模态）不影响秘书的角色或旧决策流程。

Diffusion Action Head Policy Objective

扩散动作头 · 用去噪扩散过程预测连续多模态动作

📌 TL;DR

把 diffusion model 的去噪过程用于动作预测：从 Gaussian 噪声出发，20 步迭代去噪生成 action chunk。既能拟合多峰动作分布（不像 MSE 会均值化），又保留连续动作的精度（不像离散分类）。

1概念

扩散模型（Diffusion Model）的核心思想：训练一个去噪网络 \(\epsilon_\theta\)，学习从噪声逐步还原数据。用于动作预测时：

\[ x_{k-1} = \alpha\bigl(x_k - \gamma\,\epsilon_\theta(x_k, e, k)\bigr) + \mathcal{N}(0, \sigma^2 I) \]

其中 \(x_K \sim \mathcal{N}(0, I)\) 是初始噪声，\(e\) 是 readout token 的嵌入，\(k\) 是去噪步骤索引，\(\alpha, \gamma, \sigma\) 是 cosine noise schedule 的超参数。经过 \(K=20\) 步后得到动作 \(x_0\)。

2在 Octo 里干什么

Octo 的 diffusion head 是一个3 层 MLP（hidden dim=256，含残差连接和 layer norm），接受 readout token 嵌入 \(e\) 作为条件，预测 action chunk（多步连续动作序列）。

设计要点：

  • 每次动作预测只需 transformer 骨干做一次前向传播，20 步去噪完全在小的 diffusion head 内完成——计算高效。
  • 训练用标准 DDPM 目标：对真实动作加噪声，训练去噪网络重建原始动作。
  • 推理时从 Gaussian 噪声开始，20 步 DDPM 采样（可用 DDIM 加速）。

3为什么比 MSE 和离散更好

方法问题Diffusion 如何解决
MSE loss多峰分布 → 平均化 → "对冲"行为，动作迟缓能建模任意多峰分布，采样出确定的一个峰
离散分类（256 bins/dim）精度受限（每个 bin 宽度固定），精细操作失败连续动作空间，精度不受限
Diffusion推理需多次前向（慢）头很小，骨干只跑一次；可用 DDIM 减少步数

4具象例子

机器人抓一个物体：可能的合理抓取姿势有多个（从左边抓、从上方抓、从右边抓）——MSE 会给出"三个方向的平均"（从中间抓，往往失败）；离散分类精度不够（位置差 1mm 就失败的精密插孔任务无法胜任）；diffusion 采样出其中一个合理姿势，精确执行。

5出处

📜 出处

直接来自 Chi et al., "Diffusion Policy" (RSS 2023)。Octo 是第一个在跨本体预训练 GRP 中应用 diffusion action head 的工作，证明其在大规模多样化数据上依然有效。

Block-wise Causal Attention Architecture

分块因果注意力 · 支持模态插拔的注意力掩码设计

📌 TL;DR

Octo 的 transformer 不用标准因果 mask，而是把 token 分成"task block"和"observation block"两类，规定：observation token 只能关注相同或更早时间步的观测以及任意任务 token；不存在的观测 block 被全零 mask 掉而非删除。这让新增/移除模态时 transformer 结构不变。

1概念

标准 decoder-only transformer 的因果 mask：token i 只能看 token 0 到 i。Octo 把这个规则按"模态 block"扩展：

  • 任务 token（语言/目标图像）：所有时间步的 observation token 都可以 attend 它们。
  • 时间步 t 的 observation token：只能 attend 时间步 ≤ t 的观测 token + 所有任务 token。
  • 缺失的模态（无语言标注的数据集 / 无手腕相机）：该 block 的 token 全零 mask，贡献为 0，但模型结构不变。

2在 Octo 里干什么

这个 mask 设计解决了一个核心工程问题：OXE 的 25 个数据集并不是每个都有语言标注（只有 56% 有），也不是每个都有手腕相机（只有 27% 有）。如果用标准方式处理，就需要为每种观测组合训练不同的模型。

Block-wise mask 的解法：永远保留全部 token 位置，只是把缺失的全零掉。这样同一个模型权重可以处理任意模态组合。

微调时添加新模态（例如力矩传感器）：加一个新 observation block，只需学新 tokenizer 的权重和这个 block 的 position embedding；骨干 transformer 权重完全不变。

3与标准方案的对比

对比：标准多模态融合方案

RT-1 等早期工作：把所有模态先 concat/fuse 成固定维度特征，再送入 transformer。优点：简单；缺点：改变输入类型需要重新初始化 fusion 层，大量预训练知识丢失。Octo 的 block-wise mask 避免了这个问题。

4具象例子

类比"会议室"：任务 token 是黑板上的任务说明（所有人都能看，不随时间变化）；每个时间步的 observation token 是那一刻拍的快照（只能参考过去的快照）；readout token 是秘书（只读）。缺少某种相机 = 那个座位空着（贡献为零），但会议室结构不变，其他人正常开会。

Open X-Embodiment Dataset

OXE · 跨本体机器人学习数据集联合体

📌 TL;DR

OXE 是由 30+ 机构联合发布的机器人操作数据集集合，约 1.5M 轨迹，覆盖多种机器人本体、传感器配置和任务类型。Octo 从中精选 25 个数据集共 800k 轨迹进行预训练——是截至发表时最大的公开机器人操作预训练数据集。

1概念

OXE（Open X-Embodiment）是 2023 年由 Google DeepMind 牵头、30+ 机构参与的数据集整合项目。目标是把分散在各研究组的机器人数据集统一格式、开放共享，类比 ImageNet 对计算机视觉的作用。

数据集包含：单臂/双臂机械臂、移动操作、导航等；动作空间有末端执行器控制、关节控制等；传感器有 RGB 相机、手腕相机、力矩传感器等。

2Octo 如何使用 OXE

Octo 的数据筛选标准：

  • 移除没有图像流的数据集。
  • 移除不使用 delta 末端执行器控制的数据集（统一动作空间）。
  • 移除过于重复、图像分辨率低、或任务过于小众的数据集。

剩余 25 个数据集，约 800k 轨迹。采样权重根据数据集大小确定，多样性高的数据集权重翻倍（避免被 Fractal、Kuka 等大数据集主导）。

对比：RT-1-X 使用 OXE 的 350k 子集；Octo 使用 800k——更多、更多样。

3数据处理技巧

  • Hindsight goal relabeling：对没有语言标注的轨迹，从该轨迹的未来帧中随机选一帧作为目标图像。让模型从自监督视觉观测中学习控制，减少对语言标注的依赖。
  • Gripper 动作统一：+1 = 开，0 = 关。各数据集的不同 gripper 约定全对齐。
  • 大 shuffle buffer：500k 帧的 shuffle buffer，并在解码图像前先混洗帧——确保训练 batch 的多样性（小 buffer 严重影响零样本性能）。

4出处

📜 出处

Open X-Embodiment Collaboration et al., "Open X-Embodiment: Robotic Learning Datasets and RT-X Models" (arXiv:2310.08864, 2023)。Octo 团队是该协作的主要贡献者之一。

Action Chunking Policy Design

动作分块 · 预测多步连续动作序列而非单步动作

📌 TL;DR

Action chunking = 策略每次预测未来 H 步的动作序列（而非单步动作），执行时采用 receding horizon control（滚动执行前几步，然后重新规划）。好处：动作更连贯，减少"对齐误差累积"；在 Octo 中与 diffusion head 组合，进一步提升动作连贯性。

1概念

单步动作预测的问题：每步独立推断，前后动作缺乏一致性，在需要连贯运动（如把盖子套上笔）的任务上容易"抖"。

Action chunking 的解法：一次预测接下来 H 步的动作，把它们视为一个整体来优化。实际执行时：

  • 执行前 \(k\) 步（\(k < H\)），然后重新观察/推断（receding horizon control）。
  • 或执行全部 H 步后再重新推断（适合延迟不敏感场景）。

2在 Octo 里干什么

Octo 预测一个 action chunk，diffusion head 的输出维度是 \(H \times d_{action}\)（H 步 × 每步动作维度）。默认用 receding horizon control 而非 temporal ensembling（实验发现 temporal ensembling 无额外增益）。

特别场景：Berkeley Bimanual（ALOHA 双臂）用 chunk size=64，执行 12 步后重规划——长 chunk 有助于双臂协调。

3出处

📜 出处

Action chunking 在 Zhao et al., "ACT: Learning Fine-Grained Bimanual Manipulation with Low-Cost Hardware" (2023) 中被推广用于双臂操作，随后被 Diffusion Policy、Octo 等广泛采用。Octo 论文验证了其在跨本体大规模预训练场景中依然有效。

4具象例子

音乐家演奏 vs 打字：打字是单步动作（一次想一个字母），音乐家是 action chunking（脑子里已经规划好接下来几个小节的手指动作序列，执行时更流畅）。机器人用 action chunking 类似于"把接下来 5 步的动作计划好再开始动"。

Modular Tokenizer Architecture

模块化 Tokenizer · 将不同模态编码为统一 token 序列的轻量编码器

📌 TL;DR

Octo 为每种输入模态设计独立的轻量 tokenizer，将它们编码为统一格式的 token 序列后再拼接。图像走浅层 CNN（patch 化），语言走冻结 T5-base（16 token），缺失模态全零掩码。这让 transformer 骨干无需改变就能处理任意模态组合。

1语言 Tokenizer

使用冻结的 T5-base（111M 参数） 文本编码器，将语言指令编码为 16 个语言嵌入 token。

实验发现：冻结 T5-base 比微调更好，比更大的 T5-large 也更好。原因可能是机器人数据集中语言标注质量参差不齐，微调 T5 反而引入噪声。

2图像 Tokenizer

使用浅层 CNN patch encoder（非预训练大型 ResNet）：

  • 第三视角相机（256×256）→ 256 个 16×16 patch token。
  • 手腕相机（128×128）→ 64 个 16×16 patch token。

这与 ViT 的"transformer-first"哲学一致：把大部分计算预算放在 transformer 骨干，而非视觉编码器。

3为什么不用预训练大型视觉编码器（如 CLIP、VC-1）

实验发现：ImageNet 预训练 ResNet 在零样本评测上无额外增益（可能与 ResNet 架构本身在大数据下表现差有关）。此外，冻结大型视觉编码器会限制模型适应机器人视角图像的能力。Octo 选择训练轻量 patch encoder，让视觉表征从机器人数据中学习，而不是从 ImageNet/网络图像迁移。

4微调时的扩展方式

添加新观测（如力矩传感器）：

  • 为力矩数据写一个新的轻量 tokenizer（例如 MLP）。
  • 为该模态分配新的 position embedding。
  • 骨干 transformer 权重完全不变，只训练新加的 tokenizer 参数。

Hindsight Goal Relabeling Data Augmentation

事后目标重标注 · 将无语言标注的轨迹转化为目标图像条件数据

📌 TL;DR

对于没有语言指令标注的轨迹，从该轨迹未来的某一帧随机选一帧作为"目标图像"——因为轨迹最终到达了那个状态，所以事后来看"到达那个图像的状态"就是那段轨迹的目标。这让 56% 没有语言标注的 OXE 数据也能参与训练，而不需要人工补标注。

1概念

传统条件模仿学习：策略需要任务条件（语言指令或目标图像）才能知道"想干什么"。问题是许多机器人数据集没有语言标注（采集时没有设计语言 annotation pipeline）。

Hindsight relabeling 的解法：轨迹的结果帧就是目标。对轨迹中的每个时间步 t，从 [t+1, T] 中均匀采样一帧作为目标图像。这样即使没有人类语言标注，模型也能学到"观察到 goal image → 执行能到达那个状态的动作"。

2在 Octo 里干什么

Octo 同时支持语言条件和目标图像条件，两者在训练时随机 mask（各有一定概率被 zero-out），让模型学会处理任一条件。对无语言数据集，始终使用目标图像条件（hindsight relabeling 自动生成目标）。

有趣发现：目标图像条件在 WidowX 评测上比语言条件高 25%——目标图像提供了更精确的任务信息。

3出处与局限

📜 出处

思想来自强化学习中的 Hindsight Experience Replay (HER)（Andrychowicz et al., NeurIPS 2017）。在模仿学习中的应用由 Lynch & Sermanet "Language Conditioned Imitation Learning over Unstructured Data" (RSS 2021) 等工作推广。

局限：Hindsight goal image 不能解释"为什么是这条路径"——如果轨迹是绕远路到达目标，模型会学到"绕远路也是 OK 的"。此外，目标图像只有视觉信息，缺乏语义（不知道"把红杯子放到桌上"而只知道"最终状态长什么样"）。这是为什么语言标注仍然有价值的原因。
