SSM State Space Model

State Space Model · 状态空间模型

📌 TL;DR

SSM 是一类用隐状态递推来建模序列的框架：每一步只更新一个固定维度的状态向量，时间复杂度 O(L)，与 Transformer O(L²) 自注意力形成鲜明对比。Deep SSM 是把多层 SSM block 堆叠成神经网络的做法。

1概念与定义

连续时间的经典状态空间方程：

\[ h'(t) = A\,h(t) + B\,x(t), \quad y(t) = C\,h(t) + D\,x(t) \]

其中 \(h(t)\) 是隐状态，\(x(t)\) 是输入，\(y(t)\) 是输出，矩阵 A/B/C/D 是参数。离散化后可以写成递推形式，适合在序列上做神经网络训练。

2S4 → Mamba 的演化

版本关键创新复杂度
S4 (Gu et al. 2022)结构化参数化（对角 + 低秩）处理长程依赖；支持并行卷积训练O(L log L) 训练 / O(L) 推理
Mamba (Gu & Dao 2024)输入依赖选择机制：A/B/C 矩阵从输入动态生成，可选择性传播或遗忘O(L) 训练（硬件感知）+ O(L) 推理

3在 AnoleVLA 里干什么

AnoleVLA 用 Mamba-370M 作为 VLA 的 backbone，替代原有 Transformer LM。多模态 token 序列（本体感知 + 视觉 + 语言）以 O(L) 复杂度处理，实现推理速度 3× 加速和更低 VRAM 占用。

📜 出处

S4: Gu et al., "Efficiently Modeling Long Sequences with Structured State Spaces", ICLR 2022. Mamba: Gu & Dao, "Mamba: Linear-Time Sequence Modeling with Selective State Spaces", CoLM 2024.

Mamba Selective SSM

Mamba · 选择性状态空间模型 (Selective Deep SSM)

📌 TL;DR

Mamba 在 SSM 基础上引入输入依赖的选择机制：A/B/C 矩阵不再是固定参数，而是从当前 token 动态生成——模型可以"决定"保留哪些信息、遗忘哪些。这使它在语言/视觉序列上接近 Transformer 效果，同时保持线性复杂度。

1选择机制

AnoleVLA 中每个 Mamba block 的核心更新（论文 Eq. 3）：

\[ z_{n,i} = A_{n,i}\,z_{n,i-1} + B_{n,i}\,h_{n-1,i}, \quad \tilde{h}_{n,i} = C_{n,i}\,z_{n,i} \]

关键：\(A_{n,i}, B_{n,i}, C_{n,i}\) 是输入依赖的（每个 token 位置 i 都生成不同的矩阵），而非全局固定。这让模型可以根据当前输入动态调节信息流。

2与 Transformer 的核心对比

Transformer Self-AttentionMamba Selective SSM
复杂度O(L²) — 每对 token 都要算注意力O(L) — 只维护固定维度隐状态
上下文访问全局（任意两点可直接关联）因果递归（只能看历史；未来不可见）
推理模式并行全序列，无增量状态可增量推理（每步只更新隐状态）
token 顺序影响位置编码补偿，顺序不影响注意力本质顺序决定信息流（因果性），需精心设计 token 排列

3在 AnoleVLA 里的设计含义

由于 Mamba 因果性，本体感知 token 被故意排在序列最前——确保机器人状态先被编入隐状态，之后视觉和语言信息才叠加进来。这是针对 Mamba 特性的专门设计，Transformer VLA 则无需此考虑。

4已知局限（结合本文失败分析）

位置识别错误（50% 失败来源）可能与 Mamba 缺乏全局注意力有关：Transformer 可以直接用注意力关联「指令中的物体名称」和「图像中的物体位置」，而 Mamba 只能通过因果递归间接传播这种对应关系。

VLA Vision-Language-Action

Vision-Language-Action Model · 视觉-语言-动作模型

📌 TL;DR

VLA 是把预训练视觉-语言模型 (VLM) 在机器人演示数据上微调得到的策略模型：输入图像 + 语言指令，输出机器人动作。核心优势是借用 VLM 的语义理解能力实现语言条件化操作。

1典型架构流程

VLA Pipeline伪结构
视觉编码器 (e.g. SigLIP, CLIP-ViT)
    → 视觉 token
语言编码器 / Tokenizer
    → 语言 token
本体感知状态
    → 状态 token

所有 token 拼接 → Backbone (Transformer 或 Mamba)
    → 上下文化表征

Action Head (离散 token / 连续投影 / diffusion)
    → 动作轨迹

2主要流派

方法Backbone 规模Action 表示典型代表
大规模 VLA7B+离散化 token / flow matchingOpenVLA (7B), π0.5 (3B), RT-2
轻量 VLA<1B离散 token 或连续投影SmolVLA (0.45B), TinyVLA (0.42B), AnoleVLA (0.47B)

3在 AnoleVLA 里的位置

AnoleVLA 属于轻量 VLA 流派，核心差异在于用 Mamba 替代 Transformer backbone，并直接输出连续动作（无离散化步骤）。

📜 出处

RT-2 (Brohan et al. 2023) 和 OpenVLA (Kim et al. 2024) 是 VLA 的奠基性工作；π0/π0.5 (Black et al. 2025) 引入 flow matching action head。

Action Head Action Chunking

Action Head / Action Chunking · 动作头 / 动作分块

📌 TL;DR

Action Head 是 VLA backbone 之后把隐状态映射到机器人动作的模块。AnoleVLA 用最简单的线性投影一次性预测 H=50 步的动作序列（action chunk），无需离散化或迭代去噪。

1三种主要 Action Head 类型

类型原理推理开销典型代表
离散 token动作 bin 化成词表 token，自回归解码高（自回归步数 × 动作维度）OpenVLA, RT-2
Flow Matching / Diffusion迭代去噪生成连续动作中（需 N 步去噪，如 10步）π0, π0.5
线性投影（AnoleVLA）末尾 token 隐状态 → 线性层 → H 步 chunk最低（单次前向）AnoleVLA, ACT

2Action Chunking 的意义

一次性预测 H 步而非逐步预测：① 降低推理频率，每 H 步才调用一次模型；② 更好地捕捉连续动作间的时序依赖（而非 Markov 单步输出）。AnoleVLA 设 H=50 步，采样率 10 Hz → 每 5 秒出一个 chunk。

3在 AnoleVLA 里为什么线性头足够

Mamba 递归隐状态本身是对全序列信息的「压缩摘要」——末尾 token 的隐状态 \(h_{K,L}\) 已经聚合了所有本体感知、视觉和语言信息。因此线性层就能直接读出动作，不需要额外复杂度。

两阶段训练 Velocity + Acceleration Loss

Two-Stage Training with Acceleration Loss · 速度 + 加速度两阶段训练

📌 TL;DR

先训练 200k 步只用速度 loss（学轨迹形状），再训练 200k 步加入加速度 loss（约束速度变化率，使轨迹平滑）。消融证明加速度阶段带来 +4.73 pts 成功率提升，在接触密集任务（开抽屉）上尤为显著。

1数学定义

每个动作 \(y^{(\tau)}\) 是末端执行器的位姿增量（即速度）。连续动作间的差分即加速度：\(\Delta y^{(\tau)} := y^{(\tau)} - y^{(\tau-1)}\)。

第一阶段（速度 loss）：

\[ \mathcal{L}_\text{vel} = \frac{1}{H} \sum_{\tau=1}^{H} \left\| y^{(\tau)} - \hat{y}^{(\tau)} \right\|_1 \]

第二阶段（速度 + 加速度 loss）：

\[ \mathcal{L}_\text{total} = \mathcal{L}_\text{vel} + \lambda_\text{acc} \cdot \frac{1}{H-1} \sum_{\tau=2}^{H} \left\| \Delta y^{(\tau)} - \Delta \hat{y}^{(\tau)} \right\|_1 \]

2为什么分阶段

直接从头加入加速度 loss 会让训练前期梯度方向混乱（模型还未学会基本速度匹配）。先稳定速度 loss 再引入加速度约束，相当于「先学走路，再学走得优雅」——是课程学习 (curriculum learning) 思想的简洁实践。

3物理意义

真实机器人的关节有加速度/力矩限制；轨迹加速度不连续会导致抖动（jerk），引起接触任务失败。加速度 loss 本质上是在训练时就引入物理约束。类比：运动控制里的 jerk 最小化轨迹规划。

Mobile Manipulation 移动操作

Mobile Manipulation · 移动操作 / 移动机器人操作

📌 TL;DR

Mobile manipulation = 机器人底盘移动 + 机械臂操作同时进行，是服务机器人的核心能力。相比固定臂（table-top），移动操作难在视角持续变化、环境动态、以及底盘-臂的协调控制。

1与 Table-Top 操作的关键差异

Table-Top (固定臂)Mobile Manipulation (移动臂)
视角固定外部摄像头 + 固定 wrist 摄像头随底盘/头部运动持续变化
工作空间有限桌面整个家庭/建筑环境
自由度6 DoF 臂底盘 3 DoF + 臂 6 DoF + 头 2 DoF（如 HSR = 11 DoF）
数据难度易收集，视角稳定难收集，需遥操作

2AnoleVLA 的平台

Toyota Human Support Robot (HSR)：11 DoF，移动底座 3 DoF + 机械臂 6 DoF + 头部 2 DoF，配有头部摄像头、末端执行器摄像头和外部摄像头，自 2017 年起是 RoboCup@Home 标准平台。

3为什么 π0.5 在此场景失败

π0.5 的预训练数据主要是固定视角桌面操作，与移动操作的持续变化视角存在严重 domain gap，加上每任务仅 50 episode 难以微调 3B 模型。轻量 AnoleVLA 训练样本效率更高，在小数据集下反而胜出。

Selective Scan 选择性扫描

Selective Scan / Hardware-Aware Algorithm · 选择性扫描 / 硬件感知算法

📌 TL;DR

Selective Scan 是 Mamba 实现 O(L) 并行训练的关键算法：在 GPU 上通过并行前缀扫描高效计算递归，同时配合 Flash Attention 风格的分块 IO 优化。没有它，Mamba 虽然理论 O(L) 但实际训练会很慢。

1问题所在

Mamba 的递归 \(z_i = A_i z_{i-1} + B_i x_i\) 理论上可以并行扫描，但因为 \(A_i\) 是输入依赖的（每步不同），无法像固定 A 的 SSM 那样直接做卷积加速。Mamba 论文提出了硬件感知的并行扫描：

  • 分块处理：把长序列切成块，块间串行、块内并行。
  • 内存优化：用重计算 (recomputation) 代替存储中间激活，类似 Flash Attention。

2推理时的优势

推理时无需并行扫描——每一步只需更新一次隐状态（真正的 O(1) per step），比 Transformer 的 KV cache 机制（O(L) per step）更省内存，对长序列实时推理尤为友好。

3在 AnoleVLA 的体现

AnoleVLA 推理速度 216 ms/chunk（10 Hz 控制），比 SmolVLA (309 ms) 和 π0.5 (578 ms) 更快，部分来源于 Mamba 更高效的推理时隐状态更新机制。

Meta-World 仿真 Benchmark

Meta-World · 多任务机器人操作仿真基准

📌 TL;DR

Meta-World 是一个包含 50 个多样化桌面操作任务的仿真 benchmark（Sawyer 机械臂），从简单的推物体到开关抽屉，分 Easy/Medium/Hard/Very Hard 四个难度。广泛用于 VLA 的标准化评测。

1基准细节

属性值
任务数50 个，覆盖拾放、推拉、旋转、工具使用等
机器人Sawyer 机械臂（6 DoF + 连续 gripper）
动作空间\(d_a = 4\)（3D 末端执行器位置 + 1D gripper）
数据集2500 episode（AnoleVLA 用 2000 训练 / 500 验证）
平均序列长81.9 帧 @ 80 Hz

2作为评测工具的意义与局限

✅ 优点

标准化任务定义 + 可复现仿真环境，便于跨工作公平对比；任务多样性和难度梯度设计合理。SmolVLA 和 TinyVLA 的结果直接引自原文，说明 benchmark 有广泛采用。

⚠️ 局限

固定视角桌面操作，与真实移动操作有 domain gap。论文中 π0.5 在 Meta-World 也训练了（43.80%），说明 benchmark 可以用有限数据训练但不代表真实泛化能力。

3在 AnoleVLA 里

AnoleVLA 在 Meta-World 取得 67.85% 平均成功率，超越 SmolVLA (57.33%) 和 π0.5 (43.80%)。Very Hard 任务 70.00% 尤其突出（SmolVLA 60%），体现了 Mamba 处理复杂动作序列的能力。
