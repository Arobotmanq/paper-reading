Spatial Conditioning Core Idea

Spatial Conditioning on Trajectory · 轨迹空间条件（本文核心创新）

📌 TL;DR

把扩散去噪过程中那条中间末端轨迹投影回图像，沿投影出的像素锚点采样多尺度视觉特征，当作策略的"空间条件"。等于让单相机策略自己生成注意力地图——夹爪要去哪，就让网络盯哪——无需分割、VLM 或点追踪等外部感知模块。

1定义

"空间条件（spatial conditioning）"指：不再把整张图压成一个全局 embedding 喂给策略，而是在图像上选出若干"该看的位置"，只在这些位置抽特征当条件。SCDP 的独到之处在于这些位置从哪来——它们来自当前预测的未来末端执行器（EE）轨迹：把这条 3D 轨迹投影到图像平面，落点就是"空间注意力锚点（spatial attention anchors）"。

关键在于这条轨迹是扩散去噪的中间产物：每去噪一步，动作估计变准一点，投影锚点也跟着挪动、聚焦。于是注意力不是一次性算好的静态地图，而是随去噪迭代演化、逐步收敛到接触区。

2在本文的用法

SCDP 的"空间条件模块（Spatial Conditioning Module，§3.2）"是全文创新核心，分三步（详见 轨迹投影 卡）：① 从中间动作轨迹重建未来 EE 点轨迹 → ② 用相机外参/内参投影成 2D 像素锚点 → ③ 在 多尺度特征 上沿锚点做双线性采样，聚合成一个紧凑的"spatial context 向量"，再经 FiLM 注入去噪 U-Net（替换掉标准 DP 里固定的视觉观测 \(O\)）。

原文点出这套机制的灵魂是"动态"——锚点随去噪步刷新：

📄 原文（p.4 §3.2）

"the subsequent trajectory points \(P_{1:s}\) are iteratively updated throughout the denoising process. This allows the model to dynamically adjust its spatial attention to task-relevant regions."

消融印证了这个设计的价值：重建横长 \(s=0\)（只用一个静态末端点）时成功率 75.8%，\(s=8\) 升到 82.5%，\(s=12\) 达 83.9%——加入未来轨迹点确实提供了更丰富的任务相关上下文。在 Meta-World "Hard" 上，SCDP 单 RGB 拿 82.5%，超过用了多相机的 DP+腕相机（74.2%）。

3对比 · 区别

方案"该看哪"从哪来依赖SCDP 的差异
SKIL分割掩码 → 语义关键点Segment Anything 等大模型 + 多阶段感知管线SCDP 不需要分割
OTTERVLM 找语言相关区域视觉-语言大模型 + 语言指令SCDP 无语言、无 VLM
标准 DP / DP3不显式选区，压成全局 embedding—SCDP 显式空间选区
SCDP（本文）扩散回路内部的中间 EE 轨迹仅需相机外参/内参（手眼标定）感知内化进扩散自身

核心分野：SKIL/OTTER 这派"任务相关视觉条件"强在语义，但靠外部预训练模型搭多阶段管线，早期感知误差会级联到下游；SCDP 把"找区域"内化进扩散循环——中间轨迹既是动作估计，又是空间查询，用同一条轨迹把"动作"和"该看哪"绑死，省掉了会级联误差的外部感知。代价：它假设任务相关区域就在末端轨迹附近，对工具使用/远端接触会失效（§6 局限）。

4出处来源

📜 出处

本文（SCDP）提出的核心机制，见 Kim et al. Spatially Conditioned Diffusion Policy（arXiv:2606.14535, 2026），§3.2 Spatial Conditioning Module。论文将其定位为对 Diffusion Policy 框架的"scene conditioning"新机制——用演化的动作轨迹在任务相关位置采样点特征。思想上承接"任务相关视觉条件"一脉（对照 SKIL [8]、OTTER [10]），但首次用扩散回路内部的中间轨迹本身充当空间查询。

Diffusion Policy DP

Diffusion Policy · 扩散策略（本文搭建其上的基座框架）

📌 TL;DR

Diffusion Policy（DP）把"输出机器人动作"建模成一个条件扩散去噪过程：从高斯噪声出发，在去噪网络引导下迭代细化，最终生成一段未来动作序列（action chunk）。SCDP 直接搭在 DP 上，唯一替换的是条件输入——把固定视觉观测换成随去噪演化的空间上下文。

1定义

传统模仿学习直接回归动作 \(a = \pi(o)\)，难以表达多模态行为（同一观测下有多条合理轨迹）。DP 换个思路：把策略写成条件去噪扩散概率模型（DDPM）。训练时给真动作加噪、学一个网络 \(\epsilon_\theta\) 预测噪声；推理时从纯噪声反复去噪，把它"雕"成一段动作序列。这样能稳定地表达复杂、多峰的动作分布。训练目标是标准的噪声预测 MSE：

\[ \mathcal{L} = \mathbb{E}_{k,A_0,\epsilon}\,\big\| \epsilon - \epsilon_\theta(A_k,\, c,\, k) \big\|^2,\quad A_k = \sqrt{\bar\alpha_k}\,A_0 + \sqrt{1-\bar\alpha_k}\,\epsilon,\ \ \epsilon\sim\mathcal{N}(0,I) \]

其中 \(k\) 是扩散步、\(A_0\) 是真动作序列、\(c\) 是条件（通常为视觉观测）。两个标志性设计：action chunking（一次预测一整段未来动作而非单步，减少误差累积）和receding-horizon 执行（预测一段、只执行前几步、再重预测）。

2在本文的用法

SCDP 明确"搭在 DP 框架之上"，沿用其条件扩散过程、conditional U-Net 主干、DDIM 推理（仿真 16 步 / 真机 10 步），动作头几乎原样照搬。它唯一动的地方是条件输入：

📄 原文（p.4 §3.3）

"We formulate our policy as a conditional diffusion process following the Diffusion Policy (DP) framework [3], where the denoising network is conditioned on the spatial context \(F^k\) instead of the fixed visual observation \(O\)."

注意 \(F^k\) 带上标 \(k\)——它每个去噪步都变（因为轨迹锚点在变），这是与原版 DP 的本质区别：原版 DP 的视觉条件 \(O\) 在整个去噪过程中固定，而 SCDP 的空间条件随去噪步演化，形成"去噪 ↔ 视觉条件"的反馈回路。SCDP 的创新点不在动作头，而在这个反馈回路。

3对比 · 区别

原版 Diffusion PolicySCDP
条件输入固定视觉观测 \(O\)（全局 embedding）随去噪演化的空间上下文 \(F^k\)
条件何时算去噪前算一次，全程不变每个去噪步重算一次
视觉特征单尺度全局特征多尺度、沿轨迹锚点采样
相机配置常配多相机（全局+腕相机）单全局 RGB 即可
动作头两者相同：conditional U-Net + DDIM + action chunking

另需区分：DP3（3D Diffusion Policy）是 DP 的点云变体，把单相机点云压成全局特征；RDT-1B / π0 等用扩散/flow 做动作头的大模型也是同根思路。SCDP 与它们正交——不卷动作头、不卷规模，只换条件输入。

4出处来源

📜 出处

Chi et al. 2023 — "Diffusion Policy: Visuomotor Policy Learning via Action Diffusion"，发表于 Robotics: Science and Systems (RSS) 2023（arXiv:2303.04137，本文参考文献 [3]）。该文首次将视觉运动策略表述为动作空间上的条件去噪扩散过程，并引入 action chunking + receding-horizon 执行，已成为模仿学习的代表性扩散基线。SCDP 的去噪网络、训练 loss、DDIM 推理均直接沿用此框架。

Multi-Scale Feature FPN

Multi-Scale Feature / Feature Pyramid Network 编码 · 多尺度特征金字塔

📌 TL;DR

FPN 式多尺度特征：从 CNN（这里是 ResNet-18）各 stage 抽出从粗到细的多分辨率特征图——低分辨率层含高层语义、高分辨率层含精细细节。SCDP 用全部 5 级 \(F_1\)–\(F_5\)，对 USB 插入这种精密对齐任务至关重要（消融显示比仅用最高层语义 \(F_5\) 高出 +13.9pp）。

1定义

深度 CNN 天然就是一个金字塔：越往深层，空间分辨率越低、通道（语义）越深。FPN 的核心观察是——不同粒度对不同任务都有用：低分辨率特征图擅长"这是什么/在哪个大区域"（语义上下文），高分辨率特征图擅长"边界/纹理/精确对齐"（精细几何）。所以与其只用最深一层，不如把多个 stage 的特征都留下来一起用。原始 FPN 还加了自顶向下通路 + 横向连接，把高层语义回灌到高分辨率层；SCDP 这里取其"多尺度金字塔"思想的简化版。

2在本文的用法

SCDP 的"多尺度图像编码器（§3.1）"以 ResNet-18 为主干，取五个 stage 最后残差块的激活 \(C_1\)–\(C_5\)（相对输入分辨率的有效 stride 为 4/4/8/16/32），各接一个独立 \(3\times3\) 卷积把通道对齐到统一维度 \(d=64\)，得到多尺度特征图集合：

\[ F = \{F_1, \ldots, F_5\},\quad F_i \in \mathbb{R}^{d \times H_i \times W_i},\quad d = 64 \]

空间条件模块沿投影锚点在每一级 \(F_n\) 上做双线性采样，得到 \(f_{i,n}\)；聚合时只沿时间轴平均池化、保留多分辨率拼接，得到 spatial context \(F=[\bar f_1,\ldots,\bar f_5]\in\mathbb{R}^{5d}\)（即 \(5\times64=320\) 维）。原文强调拼接是为了不丢分辨率信息：

📄 原文（p.4 §3.2）

"we apply average pooling only along the temporal axis while preserving multi-resolution information through concatenation of the pooled feature vectors."

消融极有说服力：仅用最高层 \(F_5\) → 68.6%，逐步加层一路上升，用全 \(F_1\)–\(F_5\) 达 82.5%，比仅 \(F_5\) 高 13.9 个百分点；而且连基线 DP 加全尺度也涨（16.7%→48.6%）——说明多尺度对精密动作生成是普适有用的归纳偏置，不止对 SCDP。

3对比 · 区别

单尺度（仅 \(F_5\)）多尺度 \(F_1\)–\(F_5\)（本文）
保留信息高层语义，丢精细细节语义 + 细节兼得
精密对齐弱（看不清接触边界）强（高分辨率层保几何）
Hard 成功率68.6%82.5%（+13.9pp）

与原始 FPN 的差异：FPN 设计用于目标检测，靠自顶向下通路+横向连接构建语义增强的金字塔；SCDP 借的是"多分辨率金字塔捕捉不同粒度"这一核心思想，把它当成采样源（沿轨迹锚点抽点特征），用途完全不同。与 DP3 等点云多尺度方法也不同——后者在 3D 点云上做层级，SCDP 在 2D 图像特征上做。骨干选择上，消融显示 ResNet 的局部性偏置在小数据下优于 ViT-S/16（31.7%）和 CLIP-RN50（16.9%）；ResNet-34 略高（85.0%）但为公平对比选 ResNet-18（82.5%）。

4出处来源

📜 出处

Lin et al. 2017 — "Feature Pyramid Networks for Object Detection"，CVPR 2017（arXiv:1612.03144，本文参考文献 [33]）。FPN 利用深度卷积网络固有的多尺度金字塔层级，以极小代价构建跨尺度特征。SCDP 原文明言"Inspired by the Feature Pyramid Network (FPN) [33]"采用多尺度视觉表征，从单视角捕捉不同粒度信息（p.3 §2 Multi-Scale Visual Representations）。

Trajectory Projection Recon + Proj

Trajectory Reconstruction & Projection · 轨迹重建与投影（空间条件模块前两步）

📌 TL;DR

空间条件模块的前两步，负责"把动作变成图像上的锚点位置"：① 重建——把中间动作（相对位移）累加成未来 EE 的 3D 点轨迹 \(P_i=P_{i-1}+\Delta P_i\)；② 投影——用相机外参 \([R|t]\) 与内参 \(K\) 把每个 3D 点投到 2D 像素 \((u_i,v_i)\)。这两步把"该看哪"从动作空间搬到了图像空间——前提是精确手眼标定。

1定义

这是把"未来动作"翻译成"图像注意力位置"的几何桥梁，两步：

① 轨迹重建（Trajectory Reconstruction）。SCDP 把动作参数化为相对当前状态的位移。给定去噪步 \(k\) 的中间动作序列 \(A^k_t = a_{1:h}\)，取其平移位移 \(\Delta P_{1:s}\)，从当前 EE 位置 \(P_0\) 出发累加重建未来 \(s\) 步 EE 点轨迹（\(s\le h\) 为"重建横长"，本文 \(s=8\)、预测横长 \(h=16\)）：

\[ P_{0:s} = (P_0, P_1, \ldots, P_s),\qquad P_i = P_{i-1} + \Delta P_i,\quad P_i \in \mathbb{R}^3 \]

\(P_0\) 由当前状态定、保持不变；\(P_{1:s}\) 在去噪过程中被反复更新（这正是注意力能"动态聚焦"的原因，见 Spatial Conditioning）。

② 轨迹投影（Trajectory Projection）。这些点在任务空间是 3D 的，要投到图像平面才能当锚点。先用外参 \([R|t]\in SE(3)\) 把机器人基座标系的点变换到相机坐标系，再用内参 \(K\) 投成 2D 像素：

\[ P^{\text{cam}}_i = R P_i + t,\qquad (u_i, v_i) \sim K\, P^{\text{cam}}_i \]

得到的 \((u_i,v_i)\) 就是该点在图像上的注意力锚点位置，供后续在多尺度特征上做双线性采样。

2在本文的用法

这两步是空间条件模块三步流水线（§3.2）的前两步（第三步是点采样，见 多尺度特征）。它们和图像编码器分工不同——图像编码每次推理只跑一次，而轨迹重建+投影每个去噪步都重做一次（因为中间动作在变）。原文给出累加公式与动态更新的判断：

📄 原文（p.4 §3.2）

"The future positions are reconstructed using a cumulative formulation: \(P_i = P_{i-1}+\Delta P_i\). While \(P_0\) remains constant as determined by the current state, the subsequent trajectory points \(P_{1:s}\) are iteratively updated throughout the denoising process."

实践细节：参考点可配置——Meta-World 用夹爪工具中心点（TCP），DexArt 用手掌点（palm point）。整套投影依赖精确的相机外参/内参，即手眼标定的质量直接决定锚点准不准。真机用 Franka + 单 Intel RealSense D435i 验证可行。

3对比 · 区别

  • 相对位移参数化 vs 绝对位姿：用 \(\Delta P_i\) 累加而非直接预测绝对位置，使轨迹重建只需"当前位置 + 动作序列"，与 DP 的动作表示天然兼容。
  • 每步重做 vs 一次算好：与图像编码（每次推理一次）相对——这是 SCDP "注意力随去噪演化"的实现基础；若改成只投影最终去噪结果，就失去了迭代聚焦的好处。
  • 3D→2D 投影 vs 直接 2D 操作：SCDP 在 3D 任务空间重建轨迹再投回 2D，而非直接在图像上预测注意力——好处是几何一致、可利用真实相机标定；代价是必须有准确的外参 \([R|t]\) 和内参 \(K\)，标定误差会直接体现为锚点偏移（这是该方法落地的硬约束之一）。

4出处来源

📜 出处

本文（SCDP）提出，作为空间条件模块的前两步，见 Kim et al. Spatially Conditioned Diffusion Policy（arXiv:2606.14535, 2026），§3.2 Trajectory Reconstruction / Trajectory Projection。其中所用的"外参 \([R|t]\in SE(3)\) + 内参 \(K\) 把 3D 点投到像素"是计算机视觉的标准针孔相机投影模型（非本文原创的几何工具）；本文的贡献在于把扩散中间动作轨迹接进这套投影、当作演化的视觉注意力锚点来用。
