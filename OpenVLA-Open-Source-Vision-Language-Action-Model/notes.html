VLA Vision-Language-Action

Vision-Language-Action Model · 视觉-语言-动作模型

📌 TL;DR

VLA 是把"预训练视觉-语言模型"直接 fine-tune 来输出机器人动作的端到端策略。图像+语言指令进，动作 token 出，把机器人控制建模成语言生成任务，从而直接继承 VLM 的互联网先验知识。

1定义

VLA（Vision-Language-Action Model）是将预训练的视觉-语言模型（VLM）适配为机器人控制策略的一类方法。具体做法：把机器人的连续动作编码为离散 token，拼接到 VLM 的输出词表中，然后用机器人示范数据对整个 VLM 做监督 fine-tune，最终模型直接从图像+语言指令预测动作 token 序列。

2为什么要用 VLM 做基础

传统策略（BC/Diffusion）VLA
从零初始化，泛化依赖训练数据覆盖继承 VLM 互联网先验，天然理解"把红色杯子放到桌子左边"
对新物体/新指令需要重新采数据语言接地能力来自 VLM 预训练，对未见概念有一定零样本能力
小模型（<1B），训练快大模型（7B+），但可用 LoRA 高效适配

3在 OpenVLA 里的用法

OpenVLA 是 VLA 范式的具体实现：以 Prismatic-7B 为 VLM 骨干，在 Open X-Embodiment 970k 示范上做 fine-tune，输出 7D 机器人动作（末端执行器的 Δx, Δy, Δz, Δroll, Δpitch, Δyaw + 夹爪开合）。整个训练就是 next-token prediction，推理就是自回归采样。

4与其他范式对比

范式代表模型核心思路优势
Diffusion PolicyChi et al. 2023去噪扩散预测动作轨迹块高精度、平滑轨迹、双模式分布建模
Transformer BCOcto, RT-1Transformer 序列建模动作多任务、可扩展
VLART-2, OpenVLAVLM fine-tune 输出动作 token互联网先验 + 语言接地 + 通用架构
Flow Matching VLAπ0, OpenVLA-OFTVLM + Flow Matching 动作头连续动作、高频控制、动作 chunking

5出处

📜 起源

VLA 这一术语由 RT-2（Brohan et al. 2023, Google）提出和普及。RT-2 把 PaLI-X（55B）直接 co-fine-tune 预测机器人动作，首次证明 VLM 的语义泛化能直接转移到机器人控制。OpenVLA 是 RT-2 路线的开源实现和延伸。

Prismatic VLM Backbone

Prismatic-7B · 多分辨率视觉语言模型骨干

📌 TL;DR

Prismatic 是 Karamcheti et al. 2024（Stanford）训练的开源 VLM，核心创新是融合 SigLIP（语义）+ DINOv2（空间）两个视觉编码器，再接 Llama 2 7B。OpenVLA 直接用它当骨干，在机器人示范数据上 fine-tune 得到 VLA。

1架构三件套

组件参数量作用
SigLIP + DINOv2 视觉编码器~600M提取图像 patch 特征（通道维拼接）
MLP Projector小（2 层）把视觉 token 投影到语言模型的 embedding 空间
Llama 2 7B 语言模型7B序列建模，输出文字或动作 token

2为什么选 Prismatic 而非 LLaVA/IDEFICS

OpenVLA 在 BridgeData V2 上做了消融对比（见论文 §3.4）：

  • LLaVA vs IDEFICS-1：LLaVA 在多物体语言接地任务上高 35% 绝对成功率；
  • Prismatic vs LLaVA：Prismatic 再高约 10%，归因于 SigLIP+DINOv2 的空间推理增益；
  • Prismatic 的代码库也更模块化，易于机器人任务修改。

3本文用法

OpenVLA 在 Prismatic-7B 权重上继续 fine-tune，包括解冻视觉编码器（这与 VLM 训练时冻结视觉编码器的惯例不同，见 summary 观点2），用 970k 机器人示范数据训练 27 个 epoch，最终 action token accuracy 超过 95%。

4出处

📜 起源

Karamcheti et al. 2024 "Prismatic VLMs: Investigating the Design Space of Visually-Conditioned Language Models"（Stanford）。在 LLaVA-1.5 数据混合上预训练，约 1M 图文对。Prismatic 本身的训练代码和权重也开源（但 SigLIP/DINOv2/Llama 2 的训练数据未公开）。

SigLIP + DINOv2 Vision Encoders

Sigmoid CLIP + Self-Distillation with No Labels v2 · 双视觉编码器融合

📌 TL;DR

SigLIP 用图文对比学习得到高层语义特征（"这是什么"），DINOv2 用自监督学习得到低层空间/结构特征（"在哪里、什么形状"）。两者特征通道维拼接，语义+空间兼得，对精确机器人操作特别有用。

1各自是什么

SigLIP（Zhai et al. 2023, Google）DINOv2（Oquab et al. 2023, Meta）
训练方式有监督对比学习（图像-文本对），用 Sigmoid loss 替代 Softmax（CLIP 式）自监督 SSL（无标签），图像与自身不同裁剪/增强的蒸馏
学到的特征高层语义：与文本 token 对齐，理解"物体类别/关系/场景描述"低层空间：密集的 patch-level 特征，对位置/纹理/形状敏感
适合的任务分类、检索、语言接地分割、深度估计、精细定位

2在 OpenVLA 里怎么融合

两个编码器分别处理同一张 224×224 输入图像，各自输出 patch embedding 序列，然后沿 channel 维拼接（concatenate），得到每个 patch 的融合特征。MLP projector 再把拼接特征投影到 Llama 2 的 embedding 维度。

关键：拼接而非加权平均，保留了两种特征的独立信息，让模型自己学习何时依赖语义、何时依赖空间。

3为什么对机器人控制特别重要

机器人操作需要同时回答：① "要操作什么物体"（语义，SigLIP 强）② "物体在哪里/是什么朝向"（空间，DINOv2 强）。纯 SigLIP 在空间推理上较弱；OpenVLA 用 SigLIP-only 骨干（消融，footnote 4）性能下降，证实了双编码器的价值。

4类比

🎯 类比

SigLIP = 告诉你"这是一个红色苹果" (what)；DINOv2 = 告诉你"它在左下角、呈球形、直径约 8cm" (where + shape)。机器人要抓苹果，两个问题都要答。

Llama 2 LLM Backbone

Llama 2 7B · Meta 开源大语言模型

📌 TL;DR

Llama 2 是 Meta 在 2023 年发布的开源大语言模型，7B 版本是 OpenVLA 的语言骨干。它提供了语言理解/生成能力和 动作 token 的自回归预测框架。OpenVLA 覆写其词表末尾 256 个 token 为动作 token。

1在 OpenVLA 中的角色

Llama 2 7B 接收 MLP projector 输出的视觉 token 序列 + 语言指令 token 序列，进行因果自注意力建模，最终输出动作 token（每个维度一个 token，共 7 个 token 对应 7D 动作）。

核心优势：Llama 2 具有大规模互联网语言预训练的语义理解，使 VLA 能够理解复杂语言指令（如"把红色积木放到绿色积木的左边"），而不仅仅是执行固定模板指令。

2Tokenizer 改造

Llama tokenizer 只预留 100 个特殊 token 位，不够 256 个动作 bin 用。OpenVLA 的方案：直接覆写词表中使用频率最低的末尾 256 个 token（这些词在自然语言里极少出现），赋予它们新含义（动作 bin 0-255）。简单粗暴但有效，不需要扩展词表。

3训练细节

  • 学习率：固定 2e-5（与 VLM 预训练时相同），不用 warmup；
  • 训练 27 个 epoch，直到 action token accuracy 超过 95%；
  • 批大小 2048，64×A100，14 天。

4出处

📜 起源

Touvron et al. 2023（Meta AI）。Llama 2 在约 2 万亿 token 互联网文本上预训练，并用 RLHF 对齐。权重公开，但训练数据和代码未全部公开。OpenVLA 在 Llama 2 基础上的适配全部开源。

动作离散化 Action Binning

Action Discretization / Binning · 连续动作的 token 化

📌 TL;DR

把机器人的连续动作空间（如 Δx ∈ [-0.05, +0.05] m）切成 256 个等宽 bin，每个 bin 对应一个语言模型 token，这样机器人动作预测就变成了标准的 next-token classification 问题，可以直接用交叉熵 loss 训练。

1为什么要离散化

语言模型本质上是分类模型（预测下一个 token 的类别），无法直接输出实值。把连续动作离散化后：

  • 动作预测 = 词表分类，可以直接用 LLM 的 cross-entropy 训练；
  • 可以复用整套 LLM 训练基础设施，无需设计新的动作头；
  • 256 bins 通常足以表示机器人的有效动作精度（位置控制精度约 0.1mm-1mm 量级）。

2分位数档宽 vs 最小最大档宽

方案档宽计算问题本文选择
RT-2 方案（min-max）均匀分 [min, max]异常值（如演示中意外大动作）会把区间拉到很大，正常动作被压缩到少数几个 bin，有效精度低否
OpenVLA 方案（分位数）均匀分 [P1, P99]忽略 1% 尾部异常值；99% 正常动作获得均匀精度是

3推理时的还原

模型输出 7 个动作 token（对应 7D 动作的每一维），每个 token 对应一个 bin 索引 \(k \in [0, 255]\)，通过 Action De-Tokenizer 还原为连续值：

\[ a = P1 + \frac{k + 0.5}{256} \times (P99 - P1) \]

取每个 bin 的中点值。

4主要局限

⚠️ 局限

① 没有动作 chunking：每步只预测当前动作，无法利用未来规划的平滑性，容易抖动；② 没有历史帧输入，无法利用运动估计；③ 推理需要逐 token 自回归，7D 动作 = 7 次 forward pass，限制了推理频率。这些都是 OpenVLA-OFT 和 π0 等后续工作改进的方向。

Open X-Embodiment OXE Dataset

Open X-Embodiment · 开放跨形态机器人数据集

📌 TL;DR

OXE 是 2023 年由 50+ 机构联合发布的大规模机器人操作数据集，包含 70+ 子数据集、200万+ 轨迹，跨越多种机器人形态。OpenVLA 从中筛选出 970k 高质量示范作为训练数据。

1数据集规模与构成

OXE 的设计目标是为机器人 foundation model 提供类似 ImageNet 的大规模训练数据：

  • 70+ 子数据集，跨越 WidowX、Franka、UR5、Google Robot 等众多机器人；
  • 200万+ 轨迹，但质量参差不齐；
  • 统一了数据格式（RLDS），方便大规模训练。

2OpenVLA 的数据处理

不是直接用全部 OXE，而是做了数据策展：

  1. 筛选条件：只保留有至少一个第三人称摄像头、使用单臂末端执行器控制的操作数据集；
  2. 数据混合权重：沿用 Octo 的混合权重（对多样性高的数据集上调权重，对低多样性大量重复的下调）；
  3. 清洗：过滤 BridgeData V2 中的全零动作帧（数据采集故障）；
  4. 最终 970k 轨迹，来自约 7 种机器人形态。

3为何多形态数据重要

多形态数据迫使模型学习"动作的语义意图"而非"特定机器人的关节运动"，从而获得对不同机器人和新场景的泛化能力——这也是为什么 OpenVLA 能开箱即用控制 WidowX 和 Google Robot 两种不同机器人。

4出处

📜 起源

Open X-Embodiment Collaboration（2023），来自全球 50+ 机构联合整理，以 Google DeepMind 为主导。数据全部公开在 robotics-transformer-x.github.io，格式为 RLDS (TensorFlow Datasets)。

LoRA PEFT

Low-Rank Adaptation · 低秩适配

📌 TL;DR

LoRA 冻结预训练权重 \(W_0\)，在每个线性层旁边插入两个小矩阵 \(A \in \mathbb{R}^{r \times d}\) 和 \(B \in \mathbb{R}^{d \times r}\)（r 远小于 d），只训练这两个矩阵。适配量 \(\Delta W = BA\) 是低秩的。对 VLA 而言，rank=32 只需训练 1.4% 参数，却能匹配全参数微调性能。

1数学原理

原权重矩阵 \(W_0 \in \mathbb{R}^{d \times k}\)，LoRA 做的是：

\[ h = W_0 x + \frac{\alpha}{r} \cdot B A x \]

其中 \(A\) 用随机高斯初始化，\(B\) 初始化为零（保证训练开始时 \(\Delta W = 0\)，不破坏预训练初始化）。\(\alpha\) 是缩放因子（通常等于 r）。训练时只更新 \(A, B\)，\(W_0\) 被冻结。

2应用于哪些层

OpenVLA 的 LoRA 应用于所有线性层（包括 Attention 的 Q/K/V/O 投影和 FFN 的所有线性层）。这与只适配 Attention 的原始 LoRA 论文有所不同，但对 VLA 效果更好。

3在 OpenVLA 里的效果

策略成功率训练参数显存（batch 16）
全参数微调69.7%7,188M（100%）163 GB（需 2 卡）
LoRA rank=3268.2%97.6M（1.4%）59.7 GB（单卡）
LoRA rank=6468.2%195.2M60.5 GB
只微调最后一层30.3%465.1M51.4 GB
夹心微调62.1%914.2M64.0 GB

rank=32 和 rank=64 效果完全一样，说明 VLA 微调所需的参数空间维度低，r=32 是推荐默认值。

4类比

🎯 类比

LoRA 就像在一张大地图（预训练权重）上贴一个小便利贴（低秩矩阵）来标注机器人任务专用信息，地图本身不动，任务完成后撕掉便利贴权重又回来了。可以同时存多个不同任务的便利贴，按需切换。

5出处

📜 起源

Hu et al. 2021 "LoRA: Low-Rank Adaptation of Large Language Models"（Microsoft）。最初为 GPT-3 微调设计，后成为 LLM/VLM 生态的标准 PEFT 方法。OpenVLA 是首批在 VLA 上系统验证 LoRA 有效性的工作。

量化推理 int4 / int8

Model Quantization · 模型权重低精度量化

📌 TL;DR

量化把模型权重从 bfloat16（16 位浮点）压缩为 int8（8 位整数）或 int4（4 位整数）存储，以节省 GPU 显存。对 OpenVLA 而言：int4 无损（7.0 GB，成功率 71.9%），int8 反而掉性能（推理慢到 1.2Hz 导致）——原因是量化粒度和硬件 matmul 效率的非线性关系。

1为什么量化对 VLA 特别重要

OpenVLA 7B 在 bfloat16 下需要 16.8 GB 显存，超出消费级 GPU（RTX 3090/4090，约 24 GB，但需要留空间给系统和 batch）的舒适区，对实验室级部署造成门槛。int4 量化把显存压到 7 GB，大多数现代消费级 GPU 都能跑。

2int8 为什么反而更差

这是 OpenVLA 一个反直觉的发现：

  • int8 量化需要在推理时做额外的反量化操作（int8 → float16），这个操作在大多数 GPU 上比减少的内存传输更慢；
  • 结果：int8 下 A5000 只能跑 1.2 Hz（vs bfloat16 的 ~6 Hz）；
  • 1.2 Hz 推理频率改变了机器人控制的系统动态，导致策略失效（不是量化精度不够）。

3int4 为什么可以

int4 量化减少内存传输量更大（相比 bfloat16 减 75%），且 Ada Lovelace 架构（RTX 4090, H100）有专用 int4 矩阵乘法硬件，内存带宽收益超过了量化开销，最终吞吐量接近或高于 bfloat16。

4工具

OpenVLA 使用 bitsandbytes（Dettmers et al.）做 int4/int8 量化，与 HuggingFace transformers 无缝集成，一行代码即可启用：load_in_4bit=True。

5类比

🎯 类比

就像把高清照片压缩为 JPEG：4 位量化类似"高压缩但保留主要内容的 JPEG"，8 位量化类似"解压时比读原图还慢的某种格式"——收益取决于具体的编解码器效率。
