统一联合分布视角 Predictive-Control Joint

Policy / Passive WM / Controllable WM / IDM = 同一分布的不同查询

📌 TL;DR

本综述最核心的"思想"：先写下一个未来观测与未来动作的联合分布 \(p(o_{t+1:t+k}, a_{t+1:t+k}\mid o_t, l)\)，再把策略、被动世界模型、可控世界模型、逆动力学模型都解释成"对它做不同 marginal / conditional"。四个看似独立的范式，其实是一棵分布树上的不同枝——这就解释了为什么 world model 与 policy 能天然耦合。

1四种查询

令 \(o_t\) 当前观测、\(a\) 动作、\(l\) 指令。从联合分布出发（§3.1 式 4–8）：

范式表达式怎么从联合分布来
Policy\(p(a_{t+1:t+k}\mid o_t, l)\)对未来观测 \(o\) 积分掉
Passive WM（视频生成）\(p(o_{t+1:t+k}\mid o_t, l)\)对动作 \(a\) 积分掉
Controllable WM\(p(o_{t+1:t+k}\mid o_t, a_{t+1:t+k})\)以动作为条件（不积分）
IDM\(p(a_{t+1:t+k}\mid o_{t:t+k})\)给定首尾观测反推动作

2为什么这个视角重要

它把"策略到底要不要内含世界模型"这个争论降维成"你查询联合分布的哪一面"：

  • 策略可以把"世界模型生成的未来观测"当作一个中间潜变量，再用 IDM-style 解码器从预测的未来里恢复可执行动作（解耦流水线就是这么干的）。
  • 而"统一 VLA / 单骨干"则直接学整个联合分布、同一生成过程里出 \(o\) 和 \(a\)。
  • 关键子类是 action-conditioned world model：因为"视觉合理但对动作不一致"的未来（passive WM 容易产生）对闭环控制几乎无用——综述反复强调这点。

3一句话记忆

🎯 直觉

同一张"世界如何在动作下演化"的概率图：挡住动作看未来 = 视频生成；挡住未来看动作 = 策略；两头都给看中间 = 逆动力学；以动作为条件看未来 = 可控世界模型。四者不是四个模型，是四种提问方式。

IDM-style vs 单骨干 vs MoT Policy Architectures

"世界模型作策略"的三种耦合方式（Figure 3）

📌 TL;DR

同样是"用视频世界模型的预测先验帮策略"，三种架构的差别在于预测与动作生成耦合得多紧：IDM-style 完全解耦（先想象未来视频、再反推动作）；单骨干把观测/动作 token 塞进同一个 backbone 联合建模；MoT 保留视频专家与动作专家、靠 joint attention 深度交互。耦合越紧，越能端到端传播时空约束，但训练/推理也越重。

1三种架构对照

IDM-style（解耦）Single-backbone（单骨干）MoE/MoT（专家）
结构WM 先预测未来观测 → 独立 IDM/策略头反推动作observation+action token 共享一个生成 backbone视频专家 + 动作专家各自保留，逐层 joint/cross-attention 融合
耦合Decoupled（WM 常先预训练再冻结）参数全共享、联合去噪/生成路由分解：参数不全共享，但梯度跨专家流动
代表UniPi, VidMan, Vidar, VPP, Gen2ActUVA, UWA, VideoVLA, Cosmos Policy, DreamZeroGE-Act, Motus, LingBot-VA, BagelVLA, LDA-1B
优点模块化、可复用视频先验、未来可解释时空先验直接注入控制、接口最简兼顾"共享先验"与"模态特化"（视频/动作频率、尺度不同）
痛点受生成保真度上限制约；视觉合理≠动作一致→误差累积训练/推理重；是否优于同规模 VLM 仍开放架构复杂；好处可能更多来自训练期共训而非推理期想象

2为什么会从解耦走向耦合

解耦路线（predict-then-act）最早、最有影响，但有个本质问题：世界模型只对"像不像"负责、不对"动作一不一致"负责，于是常生成"看着对、执行起来错"的未来，误差在闭环里累积。后续工作就是不断把"预测"和"动作"拉近：单骨干让二者共享一个生成过程；MoT 在保留专家特化的同时，用注意力反复把视频分支的 foresight 注入动作分支。

⚖️ 综述的克制

它明确说："视频预训练 backbone 是否一致优于同规模 VLM backbone，仍是开放经验问题"。所以这套 taxonomy 是组织地图，不是优劣排名——这也是它好用的地方。

JEPA / Latent World Modeling Predict in Embedding

Joint-Embedding Predictive Architecture · 表征空间里的世界建模

📌 TL;DR

第四条"世界模型作策略"的路线：完全不生成未来图像/视频，而是在表征（embedding）空间里预测"未来会变成什么样"。既保留了"给动作生成注入预测结构"的核心收益，又省掉了像素级生成解码的算力与冗余。思想源自 LeCun 力推的 JEPA 家族（在嵌入空间而非像素空间做预测）。

1核心动机

像素空间预测有两个老问题：① 算力贵（扩散迭代去噪、长视频）；② 预测目标里大量是与控制无关的视觉细节（光照、纹理），模型容易"抄近路"靠像素变化拟合而学不到动作相关的状态转移。JEPA 路线把预测目标换成"未来观测的 latent 表征"，强迫模型学动作相关的状态演化。

2这一族的代表与差异

方法关键设计
FLAREFuture Latent Representation Alignment：把动作去噪网络的隐特征与"未来观测的 latent"对齐，让策略隐式预判未来而不显式生成
VLA-JEPAJEPA 式预训练 + leakage-free 状态预测：未来帧只用来产生 latent 监督目标，避免靠像素变化抄近路
JEPA-VLA反过来——直接把 V-JEPA 2 学到的预测式 embedding 当更强的 VLA backbone（而非加预测头）
WoG / DIAL把世界建模搬进动作生成的"条件空间"：只预测对精确控制最有用的那部分紧凑未来条件

3和像素路线的关系

🎯 不是对立，是光谱

从"显式渲染整段未来视频"（IDM-style）→"训练时生成、推理时跳过"（Fast-WAM）→"只在 latent 里预测"（JEPA）是一条由重到轻的连续光谱。综述把 latent/symbolic 列为"非像素路线"，强调：控制相关的预测可以用紧凑的 latent 或抽象变量表达，往往是更直接的动作接口。也连接到效率挑战（§8.2）与符号结构（§8.5）。

WM 作模拟器的 RL Learned Simulator

在世界模型里做强化学习 + 世界模型–策略 co-evolution

📌 TL;DR

把学到的世界模型当环境本身：VLA 策略在"想象"里 rollout、拿 reward、用 GRPO 这类目标更新——替代又慢又贵又不安全的真机交互。综述指出一个关键升级：simulator 本身不可靠，所以最新工作让"世界模型"和"策略"交替互相改进（co-evolution），而不是把世界模型当一个固定的完美环境。

1基本配方

世界模型 \(p_\phi\) 产生想象的 transition、可选 reward 与终止信号：

\[(\hat o_{t+1}, \hat r_t, \hat d_t) \sim p_\phi(\cdot \mid o_{\le t}, a_{\le t}, l)\]

策略在想象 rollout 上最大化回报，实践中常用 GRPO 式的 clipped 目标（适配 chunk / flow-based 动作头）。代表：UniSim、World-Env、VLA-RFT、DiWA、WMPO、RISE、GigaBrain。

2两层认识

层级核心主张代表
第一层世界模型 = learned simulator，在里面做 RL。差异主要在 reward 设计、rollout 表示、优化兼容性World-Env, VLA-RFT, WMPO, World-Gymnast
第二层承认 simulator 不完美 → 必须和策略一起改进：策略失败 rollout 回头修世界模型，更好的世界模型再产更好的想象数据World-VLA-Loop, VLAW, WoVR

WoVR 的 co-evolution 写得最直白：

\[\phi_{k+1} \leftarrow \text{UpdateWM}(\phi_k,\, D_{\text{real}} \cup D_{\text{policy}}(\pi_{\theta_k})),\quad \theta_{k+1} \leftarrow \text{UpdatePolicy}(\theta_k,\, \hat D(\phi_{k+1}))\]

3同一能力的另一面：evaluator

🔁 一体两用

能 rollout 动作条件未来的世界模型，既能训练时当模拟器，也能决策时当评估器（给候选动作打分、排序、拒绝，甚至离线评估不同策略/checkpoint，如 WorldEval）。综述强调：评估场景下，realism 不够——rollout 必须可靠到能支撑排序与选择，否则幻觉会直接污染评估信号本身（WoVR 的警告）。
