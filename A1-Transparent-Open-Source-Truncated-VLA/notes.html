VLA Vision-Language-Action

Vision-Language-Action Model · 视觉-语言-动作模型

📌 TL;DR

VLA 把感知（视觉）、理解（语言）、控制（动作）统一进一个端到端可训练模型。通常以大型 VLM 为骨干，接上一个动作解码头，直接输出机器人的连续电机指令。

1定义

VLA 是一类机器人策略模型，以视觉观察（RGB 图像）+ 语言指令为输入，直接输出连续动作向量（末端执行器位姿、关节角、底盘速度等）。其核心思路是把 Transformer 在多模态预训练上的泛化能力"嫁接"到机器人控制上。

2典型结构

组件作用A1 中的选择
视觉编码器将 RGB 帧压缩为 tokenCLIP（含在 Molmo 内）
语言骨干 (VLM)跨模态理解与意图推理Molmo-7B (Qwen2-7B + CLIP)
动作头从 VLM 隐状态解码连续动作Flow Matching (Qwen3-400M) 或 MLP

3在 A1 中的用法

A1 是标准 VLA 框架，但做了两项关键改动：① 训练时对每一层的隐状态都接 action head 并监督，而不仅是最后一层；② 推理时动态决定在哪一层退出，而非固定跑完全部层。这两点合起来是论文的核心贡献。

4对比 / 出处

📜 VLA 谱系

最早大规模 VLA 包括 RT-2（Google, 2023）、OpenVLA（2024）、π0（Physical Intelligence, 2024）。A1 定位于"效率导向"分支，与 DeeR-VLA、SmolVLA、EfficientVLA 同类，但技术路线不同（A1 单共享头 vs. DeeR-VLA 多独立头）。

5类比

想象语言翻译：VLM 是理解原文的翻译官大脑，action head 是把"想法"落实为"手写汉字"的手。VLA 就是把大脑和手连在一起，直接从"看到场景 + 听到指令"到"执行动作"，中间不再有规则系统或单独的规划器。

Truncated VLA Architecture

截断式 VLA 架构 · 中间层 action 监督 + 动态退出

📌 TL;DR

"截断"指的是在推理时不必跑完 VLM 的全部 L 层——只要中间某层的 action 预测已经稳定，就在那里退出，舍弃后面的计算。为了使这一点可行，训练时必须对每层的 action 都进行监督。

1核心思想

标准 VLA 训练：只监督最后一层的 action loss。A1 的改动：随机采样层 i ∼ U(0, L)，对第 i 层隐状态预测的 action 计算 loss。等价于"多退出点的知识蒸馏自己蒸自己"。

\[ \mathcal{L} = \sum_{i \in E} \mathcal{L}^{(i)}(\theta), \quad \text{其中} \quad E = \{i_1, i_2, \ldots, i_K\} \subseteq \{1, \ldots, L\} \]

2与 DeeR-VLA 的关键区别

A1DeeR-VLA
Action head单个共享头，挂在任意层每个退出层一个独立头
参数量节省（共享参数）较多（多个头）
退出层兼容性理论上任意层均可预设的固定几个退出点

3类比

相当于接力赛：运动员（VLM 层）一棒接一棒往前跑，裁判员（action 一致性检测）站在赛道旁，只要速度/方向已经足够稳定就鸣枪结束——不必等所有棒次都跑完。但每个运动员都必须事先练过"被中途叫停时怎么把棒传下去"（即每层都要被训练能产出合理 action）。

4具象例子

在 LIBERO-Long 的"打开灶台，把摩卡壶放上去"任务里，A1 实测：机械臂简单移动帧在第 3 层就退出（共 28 层），而关键的"开灶"帧要到第 17 层、"抓锅"帧要到第 25 层才退出。这说明模型自动把算力集中在关键动作上。

Early Exit Adaptive Inference

动态提前退出 · 自适应推理终止

📌 TL;DR

在推理时，逐层计算 action，并和上一层的 action 作差比较。差值小于阈值 η_i 则认为"action 已稳定"，立即停止、返回当前 action，跳过后续所有层的计算。

1触发机制

\[ \Delta_t^i = d\!\left(A_t^{(i)},\, A_t^{(i-1)}\right) < \eta_i \;\Rightarrow\; \text{Early Exit at layer } i \]

差异度量 d(·,·) 可选余弦相似度、L2 距离或均值绝对偏差（MAD）。

2阈值 η_i 如何设定

不是手调的。用训练集跑一遍前向，收集每层的实际差异值分布，根据目标"退出预算"c 的指数分布 p_k ∝ ρ^k 自动设定各层分位数阈值。c 越小 → 退出越早 → 节省越多计算，但精度也有代价。

3本文实验结果

  • c=1.0：省 15.6% 算力，精度不降（甚至提升——多层监督的正则化效果）
  • c=0.7：省 39.1%，精度降 0.3%
  • c=0.4：省 58.5%，精度降 2.3%
  • c=0.1：省 76.6%，精度降 1.7%

4出处

📜 Early Exit 的来源

早期 exit 在 NLP 中已有大量研究（BranchyNet 2016、DeeBERT 2020 等）。A1 引用 DeeR-VLA（2024）将这一思路引入 VLA 领域，并在其基础上做了关键改动（共享单头 + 与 Inter-Layer Truncated FM 联动）。

5类比

就像坐地铁：你要去某站，但在中途某站已经觉得"方向对了，这就是目的地了"——立刻下车，不再等终点站。终点站通常更准，但多坐几站的时间浪费不值得。

ILTFM Inter-Layer Truncated FM

Inter-Layer Truncated Flow Matching · 层间截断流匹配

📌 TL;DR

在 early-exit 推理中，每层都要执行 flow matching 去噪——如果每层都从随机噪声跑 10 步，反而比不退出还慢。ILTFM 的解法：每层只跑 δ=2 步，并把上一层的去噪结果作为下一层的初始条件（热启动），整个 episode 的去噪进度在层间累积传递。

1问题根源

设 FM 每次去噪需要 δ=10 步，VLM 有 28 层，early-exit 在第 k 层退出：需要跑 k×10 步去噪（比无 early-exit 的 10 步还多）。这让 A1-FM 在 c=1.0 时延迟反而从 1.15s 增至 4.44s（Table 3 数据）。

2解决方案

\[ A_t^{0(i+1)} = A_t^{1(i)} \]

即：第 i 层去噪结束后得到的 \(A_t^{1(i)}\)，直接作为第 i+1 层去噪的起点 \(A_t^{0(i+1)}\)，而不是从随机噪声 \(\mathcal{N}(0, I)\) 重新开始。同时将每层去噪步数压缩到 δ=2。

ILTFM 伪代码
# 标准 FM（每次从噪声重启）:
for each layer i until exit:
    A = random_noise()
    for step in range(10):  # 10步去噪
        A = denoise(A, layer_i_features)
    if consistent(A, A_prev): break

# ILTFM（层间热启动，每层仅2步）:
A = random_noise()
for each layer i until exit:
    for step in range(2):  # 仅2步！
        A = denoise(A, layer_i_features)
    if consistent(A, A_prev): break
    # A 直接传给下一层，不重置！

3效果

在 LIBERO 上：episode 推理时延从 40.9s → 27.5s（去掉热启动）→ 10.5s（加热启动）。更惊喜的是，热启动还把成功率从 95.4% 提升到了 96.4%——热启动提供了更好的去噪初始化。

4直觉类比

就像数学考试做草稿：标准做法是每道题都从空白纸开始草稿；ILTFM 的做法是把上一道题的草稿直接带到下一道——如果题型相关，这个"接力草稿"让你快得多，而不是重新乱写一通。

Flow Matching Action Head

Flow Matching · 流匹配生成模型

📌 TL;DR

Flow matching 是一种生成式动作解码方法：训练时学习一个"向量场"，推理时从随机噪声出发，沿该向量场积分几步得到最终动作。比传统扩散模型（DDPM）更简洁，通常 10–20 步就够，可表示多峰分布。

1训练目标

\[ \mathcal{L}_\tau(\theta) = \mathbb{E}\left[\left\| v_\theta(A_t^\tau, o_t) - u(A_t^\tau | A_t) \right\|^2\right] \]

其中 \(A_t^\tau = \tau A_t + (1-\tau)\epsilon\)（noisy action），\(u(A_t^\tau|A_t) = \epsilon - A_t\) 是目标向量场，\(\tau\) 从 beta 分布采样（强调低 τ，即更嘈杂的端）。

2推理：Euler 积分

\[ A_t^{\tau+\delta} = A_t^\tau + \delta \cdot v_\theta(A_t^\tau, o_t), \quad A_t^0 \sim \mathcal{N}(0, I) \]

从随机噪声 A⁰ 出发，按步长 δ 积分多步，最终在 τ=1 时得到预测 action。

3在 A1 中的实现

A1 用约 400M 参数的 Qwen3 作为 FM action expert，通过 KV-conditioned self-attention（把 VLM 骨干的 prefix 输出注入为 past K/V）与骨干连接——这样 action expert 可以在不修改 VLM 的情况下独立缩放。

4与 ILTFM 的关系

ILTFM 是 Flow Matching 在 early-exit 场景下的适配：通过层间热启动，把本来需要的 10 步去噪压缩到 2 步，同时不显著损失最终精度。这是 A1 论文中最有新意的技术创新点。

5对比 MLP head

Flow Matching (A1-FM)MLP (A1-MLP)
表达能力多峰分布，细粒度连续动作单峰拟合，适合简单精确任务
推理成本高（多步去噪），需 ILTFM低（单次前向）
噪声抑制一般L1 loss 天然抑制噪声
层间 action 相似度高（利于早退出）较低（阈值更难统一）

Action Head Policy Decoder

动作解码头 · 从 VLM 隐状态到连续电机指令

📌 TL;DR

Action head 是 VLA 中负责把 VLM 的"理解结果"转化为机器人可执行动作序列的模块。A1 提供两种实现：FM action expert（精细、多峰）和 MLP head（快速、低噪）。两者共享同一套骨干表示，通过不同接口连接。

1FM Action Expert

采用约 400M 参数的 Qwen3 解码器。通过 KV-conditioned self-attention 接入骨干：骨干产生的 prefix context 被注入为 K/V，action token 在 suffix 位置同时 attend 骨干 prefix 和 action 自身。输出为预测动作块 \(A_t = [a_t, a_{t+1}, \ldots, a_{t+H}] \in \mathbb{R}^{H \times D}\)（action chunk，H 步预测，D 维）。

2MLP Head

在输入 token 序列中插入额外的"动作 query token" S，从这些 token 对应的隐状态直接通过 MLP 回归连续动作：

\[ H_t = f_\phi([o_t, \ell, S])_S, \quad \hat{A}_t = g_\psi(H_t), \quad \mathcal{L}_{MLP} = \mathbb{E}\|\hat{A}_t - A_t\|_1 \]

用 L1 loss 而非 L2，抑制异常值，适合抓取等需要精确但稳定的任务。

3共享头设计的重要性

A1 在所有退出层共享同一个 action head 参数（FM 或 MLP）。这与 DeeR-VLA 的"每个退出层独立一个小 head"形成对比。A1 的设计：① 参数量更少；② 训练更简洁；③ 但对 head 的泛化能力要求更高（需要在不同深度的特征质量上都能工作）。

4类比

骨干 VLM 是"眼睛+大脑"，action head 是"手"。KV-conditioned attention 就像手通过神经网络读取大脑发出的指令——手并不直接"看"场景，而是接收大脑预先消化好的指令。

4对"Fully Transparent"的理解

论文把 §、数据来源、超参数、消融设置都公开得很详细；code repo 文档质量较好；使用的均为可公开数据（自采数据也说明了来源平台）。但"Fully Transparent"目前更准确的描述是"意图和文档层面的透明度高，实际制品层面尚未完全兑现"。

Molmo VLM Backbone

Molmo · A1 使用的视觉语言模型骨干（来自 Allen AI）

📌 TL;DR

Molmo 是 Allen Institute for AI (Ai2) 于 2024 年发布的开源多模态大模型，基于 CLIP 视觉编码器 + Qwen2-7B 语言模型。A1 选择 Molmo 的原因是它在视觉-语义理解和"可供性先验（affordance priors）"上有较强表现，且完全开源。

1结构

  • 视觉编码器：CLIP / SigLIP（多视图支持）
  • 语言骨干：Qwen2-7B（28 层 Transformer），总推理 GFLOPs = 11,074（序列长 352）
  • CLIP 占 2,013 GFLOPs，每个 LLM 层占 323.6 GFLOPs

2为什么选 Molmo（而非 PaliGemma/LLaVA 等）

论文说 Molmo"从大规模多模态预训练中学到了强视觉语义理解以及隐式可供性先验（implicit affordance priors）"。这意味着 Molmo 的 token 表示天然包含"哪里可以抓握"、"物体的功能部件"等对操作有用的空间语义——无需额外的 affordance 标注。

3训练时的处理

A1 训练时冻结 ViT backbone（CLIP 部分），只训练 VLM（Qwen2-7B 部分）和 action head。VLM 使用学习率 5×10⁻⁵，action head 用更高的 5×10⁻⁴，前 1000 步仅训练 VLM 骨干冻结状态下做 warm-up。

4类比

Molmo 像一个已经看过大量"人类如何使用各种物品"视频的助手——他知道"杯子的把手是用来抓的"、"抽屉是从正面拉的"。A1 在此基础上再教他"当你看到这个场景、听到这个指令时，你的手应该做什么动作"。
