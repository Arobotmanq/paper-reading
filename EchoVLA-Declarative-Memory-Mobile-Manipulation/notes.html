Declarative Memory 神经科学

Declarative Memory · 陈述性记忆（显性记忆）

📌 TL;DR

陈述性记忆是人类大脑中可以被意识提取、用语言描述的记忆系统，分为语义记忆（知道"世界是什么"）和情节记忆（知道"自己经历了什么"）。EchoVLA 用它类比机器人记忆：scene memory ≈ 语义记忆，episodic memory ≈ 情节记忆。

1定义

心理学将记忆分为两大类：

类型子类型内容例子
陈述性记忆（Declarative）语义记忆（Semantic）关于世界的事实与概念"巴黎是法国首都"
情节记忆（Episodic）个人经历与时序事件"我昨天在厨房开了抽屉"
程序性记忆（Procedural）技能与习惯（无需意识）骑车、打字

陈述性记忆依赖海马体（Hippocampus）和海马旁回（Parahippocampal Cortex, PHC）：PHC 负责编码空间和场景语义，海马体整合时序情节。

2在 EchoVLA 中的用法

EchoVLA 将这一神经科学框架映射到机器人记忆设计：

  • PHC → Scene Memory：维护 voxel 3D 特征地图，代表"这个房间的空间结构和物体布局"（语义/空间记忆）。
  • Hippocampus → Episodic Memory：维护时间索引 token buffer，代表"我刚才做了什么、拿了什么"（情节/时序记忆）。

两种记忆互补而非冗余：场景记忆提供慢变的空间背景，情节记忆提供快变的任务进度。这种分离正是人脑应对长程任务的机制。

3为什么叫"协同"（Synergistic）

论文标题中的 "Synergistic" 指两种记忆不是简单拼接，而是通过粗细双级交叉注意力协同融合：scene memory 先用粗粒度 attention 贡献全局空间约束，episodic memory 再用细粒度 attention 贡献精确时序线索，两者的融合表示 \(\mathbf{H}_t\) 才驱动策略。

Scene Memory 空间语义地图

Scene Memory · 场景记忆 / Spatial-Semantic Map · 空间语义地图

📌 TL;DR

EchoVLA 的场景记忆是一个增量累积的 voxel 3D 特征地图，跨 episode 持续更新；用差异驱动规则只在场景真正变化时写入，并在推理时在线自适应。它给策略提供"这个房间长什么样"的稳定空间背景。

1数据结构

场景记忆是一个 voxelized 3D feature volume：

\[ \mathbf{V}^{3D}_t \in \mathbb{R}^{X \times Y \times Z \times C} \]

其中 \(X \times Y \times Z\) 是 voxel 网格维度，\(C\) 是特征维度。每个 voxel 存储由 PointAttn 网络（可学习 3D 编码器）从 depth 观测提取的几何特征，包含自由空间、支撑面、物体边界等信息。

2差异驱动更新（Discrepancy-Driven Rule）

关键工程细节：不是每帧都全量写入，而是只在重建误差超过阈值 τ 的区域更新：

当前点云 → PointAttn → V3D_t（新特征）
比较：recon_error = ||V3D_t - reconstruct(M_scene)||
if recon_error[voxel] > τ:
    M_scene[voxel] = V3D_t[voxel]  # 更新
else:
    keep M_scene[voxel]             # 保留旧值

这设计有两个好处：(1) 避免冗余写入，记忆稳定；(2) 允许在线自适应，推理时遇到未见过的环境也能逐步建图。超参 τ=0.5 经消融验证为最优（τ=0.1 不稳定，τ=0.7 "ghosting"）。

3检索方式

用当前 3D 特征 \(\mathbf{V}^{3D}_t\) 作为 query，通过 cosine similarity 在记忆库中选 top-k 匹配，再做粗粒度 cross-attention：

\[ \mathbf{Z}^{\text{scene}}_t = \text{CrossAttn}(\text{q}=\mathbf{V}^{3D}_t,\ \text{k/v}=\mathcal{M}^{\text{scene}}_{\text{sel}}) \]

4局限

论文诚实承认：若环境几何快速变化（如冰箱门打开），显式 3D 记忆会出现"ghosting"（残影），导致性能下降。未来计划整合 visual-SLAM 或 loop-closure 缓解。

Episodic Memory 情节记忆

Episodic Memory · 情节记忆 / Task Progress Buffer

📌 TL;DR

EchoVLA 的情节记忆是一个时间索引 FIFO token buffer，存储最近 k 步的完整多模态 token（不压缩为摘要），通过细粒度 cross-attention 为策略提供"我刚才做了什么"的精确时序线索，解决非马尔可夫歧义。

1形式定义

\[ \mathcal{M}^{\text{epi}} = \{(\mathbf{S}_{t-k}, t-k),\ \ldots,\ (\mathbf{S}_{t-1}, t-1)\} \]

其中 \(\mathbf{S}_t = [\mathbf{L}, \mathbf{V}_t, \mathbf{P}_t, \mathbf{R}_t]\) 是当前时刻的多模态 token 序列（语言 + 多视角图像 + 点云 + 本体感知）。窗口大小 k=8 经消融验证为最优。

2为什么不压缩为摘要

关键设计选择：存储原始 token 而非抽象摘要。摘要会丢失区分"正要抓"和"已抓"所必需的精确位置、末端执行器构型等细节。完整 token 虽占用更多空间，但对解决非马尔可夫歧义至关重要。

3检索方式

用当前多模态 token \(\mathbf{S}_t\) 作为 query，通过 cosine similarity 选最相关子集，再做细粒度 cross-attention：

\[ \mathbf{Z}^{\text{epi}}_t = \text{CrossAttn}(\text{q}=\mathbf{S}_t,\ \text{k/v}=\mathcal{M}^{\text{epi}}_{\text{sel}}) \]

4与场景记忆的对比

Scene MemoryEpisodic Memory
对应人脑区域海马旁回（PHC）海马体（Hippocampus）
存储内容环境空间结构（voxel 3D features）任务历史 token 序列
变化速度慢（跨 episode 稳定）快（每步更新）
注意力粒度粗粒度（coarse）细粒度（fine）
解决的问题空间定位、物体布局非马尔可夫歧义、任务进度

Long-Horizon Mobile Manipulation 任务定义

长程移动操作 · 协调导航与精细抓取的序列任务

📌 TL;DR

长程移动操作 = 机器人需要跨空间导航（底盘移动）+ 精细操作（机械臂抓取/放置）+ 多步序列（涉及数百帧），且两帧视觉相似但任务进度可能完全不同。这是当前 VLA 的主要未攻克难题。

1为什么难

  • 空间跨度大：需要在 7m×7m 甚至更大场地内导航，单帧观测无法看到全局。
  • 非马尔可夫性强："已开冰箱门"和"将开冰箱门"视觉相似，但所需动作截然不同；策略必须记住历史。
  • 底盘-机械臂协调：底盘运动和臂精细操作的动力学差异大（一个是低频粗粒度，一个是高频精细），统一建模困难。
  • 数据稀缺：收集多步骤长程移动操作数据代价高，现有基准规模有限。

2EchoVLA 中的具体任务

仿真任务（RoboCasa）：PickPlaceCounterToStove、PickPlaceSinkToCounter、TurnOnSinkFaucet、TurnOnStove，以及增加了初始位置偏移和后续导航的 Mobile Manipulation 变体。

真机任务（TidyBot++）：Open Drawer (OD)、Close Microwave (CM)、Place Cup Into Sink (PCIS)、Open Refrigerator (OR)、Rotate Knob (RK)、Enter room and Place pears (EnP)。其中 EnP 是最具挑战性的跨房间任务（平均 362 帧）。

3类比

类比"帮人整理厨房"：先要走过去（导航），记住哪些已经整理了（情节记忆），知道柜子在哪里（场景记忆），然后精确拿取放置（操作）。这正是 EchoVLA 设计目标的最直观描述。

VLA Vision-Language-Action

Vision-Language-Action Model · 视觉-语言-动作模型

📌 TL;DR

VLA 是将视觉感知、自然语言指令理解和机器人动作生成统一在同一个模型中的端到端框架。它通常基于大型视觉-语言模型（VLM）预训练，再接动作生成头（自回归 token 或 diffusion）。EchoVLA 在 VLA 基础上附加了双记忆模块。

1核心思路

VLA 的关键洞见：机器人控制可以看作"以图像和语言为条件的序列预测"，与 LLM 的 next-token prediction 统一。这允许借助大规模视觉-语言预训练来获得强泛化能力，然后 fine-tune 到机器人控制。

代表模型动作表示特点
RT-2离散 action token首次大规模 VLA，利用 web 预训练
OpenVLA离散 action token开源，7B 参数
π0 / π0.5diffusion（连续）per-part 底盘+臂分离，移动操作专用
EchoVLAper-part diffusion + 双记忆为长程非马尔可夫任务增加记忆机制

2当前局限（EchoVLA 的切入点）

现有 VLA 大多是马尔可夫策略：每步决策只看当前帧，不维护历史。这在桌面短程操作中够用，但在移动操作中会因"历史遗忘"失败。EchoVLA 的贡献正是打破这个限制。

3技术细节（EchoVLA 的 backbone）

EchoVLA 使用冻结 SigLIP 作为语言和图像编码器（保留跨模态对齐），可学习 PointAttn 处理 3D 点云，小 MLP 处理本体感知，最终形成统一 token 序列 \(\mathbf{S}_t = [\mathbf{L}, \mathbf{V}_t, \mathbf{P}_t, \mathbf{R}_t]\) 驱动记忆检索和 diffusion policy。

Per-Part Diffusion Policy 动作生成

Per-Part Diffusion · 分部件扩散策略

📌 TL;DR

Per-part diffusion 是为底盘（base）和机械臂（arm）分别设置独立的去噪扩散网络，但共享同一个记忆增强条件 \(\mathbf{H}_t\)。这样底盘运动和精细操作可以独立优化，同时保持协调。

1为什么要分离

底盘和机械臂的动力学特性差异显著：

  • 底盘（base）：低精度、大范围移动，全向驱动，误差容忍度较高。
  • 机械臂（arm）：高精度、小范围操作，7-DoF，毫米级对齐要求高。

统一用单一扩散网络建模两者会导致"精度-范围"的权衡困难。分离允许各自的损失函数和网络容量针对性优化。

2形式定义

对每个部件 \(p \in \{\text{base}, \text{arm}\}\)，独立去噪器预测噪声：

\[ \epsilon_\theta^{(p)} = \text{Denoiser}_p(\mathbf{z}_t, \mathbf{H}_t, t),\quad p \in \{\text{base}, \text{arm}\} \]

训练目标为两部件去噪损失之和：

\[ \mathcal{L} = \sum_{p \in \{\text{base}, \text{arm}\}} \mathbb{E}_{t, \mathbf{z}_t}\Big[\|\epsilon - \epsilon_\theta^{(p)}(\mathbf{z}_t, \mathbf{H}_t, t)\|^2\Big] \]

3与 π0.5 的关系

π0.5 已引入 per-part decomposition 的思路。EchoVLA 在此基础上的贡献是：两个 per-part 去噪器都以双记忆增强表示 \(\mathbf{H}_t\)（scene memory + episodic memory 的融合）为条件，使历史感知能同时影响底盘规划和臂操作精度。

4具体实现

每个 Denoiser 是一个 DiT（Diffusion Transformer）块序列，接收噪声动作和条件表示（\(\mathbf{H}_t\)），输出预测噪声。底盘和臂各有独立的 MLP 投影层将 \(\mathbf{H}_t\) 适配到各自的动作空间。

MoMani 基准数据集

MoMani Benchmark · Mobile Manipulation Benchmark · 移动操作基准

📌 TL;DR

MoMani 是 EchoVLA 提出的移动操作数据集，特点是仿真自动生成（7889 episodes）+ 真机示范（1200 episodes），通过 MLLM 引导规划和质量门控生成专家级"导航+操作"协同轨迹，是目前唯一同时支持全自动 Co-Gen 和真机数据的移动操作基准。

1生成流水线

Stage I：在线候选生成——MLLM prompter 处理任务规格，执行 L1（Target-Aligned Departure Sampling）→ L2（Safety-Aware Multi-Detour Navigation）→ L3（Continuous Nav-Manip Stitching）。候选必须通过硬质量门：零碰撞、位置偏差 <0.05m、角度偏差 <5°、100% 任务成功。

Stage II：离线选择与审核——对候选池按路径长度、步骤数、规划代价做字典序排名，选出 Top-K 专家级轨迹集，最终经场景-相机审核确保训练就绪。

2数据组成

子集规模任务

3与现有基准的区别

MoMani 是唯一同时满足：(1) 支持 Co-Gen 自动生成导航+操作协同轨迹，(2) 同时提供仿真和真机数据，(3) 基于全向移动平台（TidyBot++）的基准。RoboCasa 仅有仿真，RoboTwin 2.0 不含导航，HomeRobot 无自动生成。
