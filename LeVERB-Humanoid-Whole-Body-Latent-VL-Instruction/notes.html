Latent Verb Core Interface

Latent Action Vocabulary · 潜在动词 / 潜在动作词汇

📌 TL;DR

"Latent Verb"是高层 VLA（System 2）输出给低层 WBC 控制器（System 1）的唯一接口——不是手工设计的速度/末端位姿，而是一个从运动数据中自监督学出来的连续向量 z，携带运动语义信息，低层策略把它解码成关节命令。

1定义与直觉

传统做法：VLA → 输出"向前走 0.5 m/s"（root velocity）→ 低层 PD 跟踪。

LeVERB 做法：VLA → 输出 latent z（256维连续向量）→ 低层 Transformer 解码 → 关节位置动作。

z 的意义：它是从真实运动轨迹（坐下、走、转身…）中用 CVAE 压缩出来的，所以它隐式编码了"接下来几秒应该做什么动作"这个信息，而不是一个简单的离散类别或标量速度。

2在 LeVERB 中的用法

  • LeVERB-VL（System 2）在每个时刻输出一个 Gaussian 分布 \(\mathcal{N}(\mu_\rho, \sigma_\rho^2)\)，代表"根据当前视觉和语言，接下来的运动应该在 latent space 的哪个区域"。
  • LeVERB-A（System 1）在训练时从这个分布中采样 \(z_t \sim \mathcal{N}(\mu_\rho, \sigma_\rho^2)\)（不是用均值！），保持固定 H 步后重新采样（即 10 Hz 接口）。
  • 在推理/部署时，LeVERB-A 用预测均值 \(\mu_\rho\) 作为条件（确定性推理）。
🔑 为什么训练时采样、推理时用均值

训练时采样是为了让低层策略学会处理 latent 分布中的整个区域（多模态性），而不只是单一点。推理时用均值是为了稳定性。消融实验（NLS variant）证实：训练时不采样、只用均值，整体成功率从 58.5% 降到 6.5%。

4类比

可以把 latent verb 类比为人说话时的"语调+语气向量"——你听不懂一句话的具体词，但语调告诉你"这是命令还是请求"，然后你的身体自动调整响应方式。同理，z 告诉低层"这是坐下还是转身"，但低层不需要知道对应的具体视觉场景。

Residual CVAE LeVERB-VL Core

Conditional Variational Autoencoder (Residual Variant) · 条件变分自编码器（残差变体）

📌 TL;DR

CVAE 是 VAE 的条件化版本——用视觉语言输入作为"条件"，学一个将运动轨迹压缩到 latent space 的编码-解码框架。LeVERB 用残差 CVAE：VLA prior（视觉语言）+ 运动学编码器（轨迹细节）= 残差叠加的完整 latent，让 VLA 专注语义、编码器补运动细节。

1标准 CVAE 结构

标准 VAE 的 encoder 把输入 x 压缩成 latent z（均值+方差），decoder 从 z 重建 x，KL loss 约束 z 接近标准正态分布。

CVAE 加入条件 c（本文中 c = 视觉+语言）：encoder 接受 (x, c) → z，decoder 接受 (z, c) → x。这样 z 只需要编码 c 无法解释的"剩余信息"。

2LeVERB 的 Residual CVAE 设计

训练时有两个 encoder：

  • VLA prior ρ_θVL：只看视觉+语言，输出均值 μ_ρ 和方差 σ_ρ。这是推理时能用的部分。
  • Kinematics Encoder E_ψ：看未来运动学轨迹 s_{t+1}...s_{t+M}，输出均值 μ_E 和方差 σ_E。这是训练时才有的"特权信息"，部署时丢弃。

残差合并：\(\mu = \mu_\rho + \mu_E\)，\(\sigma = \sigma_E\)。KL loss 约束后验 q(z) 接近 prior p(z)，强制 kinematics encoder 只补"视觉语言推断不到的信息"。

🔑 残差设计的精妙之处

如果不用残差，kinematics encoder 可能"偷懒"——直接记住整条轨迹而无视 VLA prior 的输出，导致 VLA prior 学不好。残差约束了 encoder 只能"补差量"，VLA prior 被迫认真学语义。这与 motion VAE（Motion CVAE，Ling et al. 2020）的设计思路一致，作者在 §4.2 明确引用。

3训练目标

\[ \mathcal{L} = \underbrace{\mathbb{E}[\|D_\psi(s_t, z) - s_{t+1:t+H}\|^2]}_{\text{轨迹重建}} + \beta_1 \underbrace{D_{KL}(q(z_t) \| p(z_t))}_{\text{分布对齐}} + \beta_2 \underbrace{\mathcal{L}_{disc}(z_t)}_{\text{对抗分类}} \]

其中 β₁=0.1，β₂=5×10⁻⁴。前两项是标准 CVAE loss，第三项是对抗判别器（见下方 Discriminator 概念）。

Dual-Process VLA System 1 + 2

Dual-Process Vision-Language-Action · 双进程视觉语言动作架构

📌 TL;DR

受人类认知"快思考（System 1）+ 慢思考（System 2）"启发的机器人控制架构：高层 VLA（慢，10 Hz）负责视觉语言理解和语义规划，低层 WBC（快，50 Hz）负责反应性动力学控制，两者通过 latent verb 单向通信。

1认知科学背景

Daniel Kahneman 的双进程理论：System 1（快速、直觉、无意识）处理感知-运动反应；System 2（慢速、分析、有意识）处理高层推理和规划。

LeVERB 直接套用这个框架：LeVERB-VL = System 2（视觉语言推理），LeVERB-A = System 1（本体感受反馈控制）。论文 §1 引用 Jensen 2006 的"心智计时"作为引述来源。

2在人形控制中的必要性

人形机器人全身控制为什么必须分层？

  • 频率需求矛盾：视觉语言推理（大模型）10-30 Hz 已是极限；平衡控制需要 ≥50 Hz，复杂动力学需要 ≥200 Hz。
  • 计算资源分配：大型 VLA 模型（GPU）无法在板载嵌入式计算上满足实时需求；低层控制器（小型 MLP/Transformer）可以在 CPU 上 50 Hz 推理。
  • 训练解耦：端到端学习需要在渲染环境中跑 RL，成本极高；分层后 System 2 单独训练（监督学习），System 1 单独训练（RL + DAgger），各自针对性优化。

3部署架构

  • LeVERB-VL 运行在外部 PC（NVIDIA RTX 4090），10 Hz，读取头部相机和外部相机各 1080×720，输出 latent z 发布到 ROS2 topic。
  • LeVERB-A 运行在 Unitree G1 板载 CPU，ONNX C++ runtime，50 Hz，读取关节编码器+IMU（500 Hz 状态估计），输出关节位置期望值。

WBC with RL Track B 核心

Whole-Body Control via Reinforcement Learning · 强化学习全身控制

📌 TL;DR

LeVERB-A 的训练分三步：(1) 用 PPO 训练多个视觉无关的"运动跟踪教师"策略（有特权观测）；(2) 冻结 LeVERB-VL 的 latent space，从中采样 z 作为条件；(3) 用 DAgger 蒸馏成轻量 Transformer 学生策略，全程域随机化保证 sim-to-real。

1PPO 教师策略训练

教师策略（Teacher T_ξ）有访问特权信息的权限：完整本体感受状态 + 未来参考运动轨迹（即"下一帧应该在哪里"）。

奖励函数（DeepMimic 风格）：

  • Global Torso Position（权重 0.5）：exp(-‖p_motion - p_robot‖²/σ²)
  • Global Torso Orientation（权重 0.3）：exp(-quat_error²/σ²)
  • Global Body Position（权重 0.5）：exp(-‖x_motion - x_robot‖²/σ²)
  • Joint Position Error（权重 -1）、Joint Velocity Error（权重 -0.1）、Action Rate L2（权重 -0.001）
  • 早期终止惩罚（-100/-200）：torso 位置误差 > 0.5m 或方向误差过大

域随机化：地面摩擦 [0.3, 0.8]、关节默认位置偏移 ±0.05、关节 armature [0.2, 2.0]、随机推力 ±0.5 m/s（每 10-15 秒一次）。

2DAgger 蒸馏到学生

DAgger（Dataset Aggregation）是一种在线模仿学习算法：学生策略在实际环境中 roll-out，碰到的每个状态都向教师请求"正确动作"标签，持续聚合新的 (state, expert_action) 对。

优点：学生学到的是"从自己可能到达的状态如何行动"，而不是"从专家轨迹起点开始"——更健壮。

LeVERB 的学生（LeVERB-A）架构：2层 Transformer，4头注意力，隐层 128 维，dropout 0.3。注意：实验发现加入观测历史会降低性能——因为策略可能从历史推断出未来动作而忽视 latent 条件。

Synthetic Data Pipeline Data

合成数据生成流水线 · MoCap → 渲染 → 标注

📌 TL;DR

用人类运动捕捉（MoCap）数据重定向到人形机器人骨骼，在 IsaacSim 光线追踪渲染器中程序化随机化场景（纹理/颜色/相机角度），再用 VLM 自动生成语言标注。154 条核心轨迹 × 100 次随机化 = 17.1 小时数据，无需任何真实机器人采集。

1为什么用合成数据

真实人形机器人全身动作数据极难采集：需要穿戴动作捕捉设备、稳定的机器人平台、大量人工操控，且安全风险高。与操作任务（可用远程操作）不同，全身运动的遥操作基础设施更复杂。

合成数据的关键优势（论文 §3 中明确列出）： (1) 运动学回放不需要稳定的动力学控制（只播运动，不用真实物理控制）； (2) 运动学状态已足够 VLA 学到任务层面语义（不需要精确物理仿真来理解"走向椅子"）； (3) 未来可以直接用互联网视频重定向的人类动作数据扩展。

2流水线步骤

  1. MoCap 来源：AMASS 数据集（全身运动，坐下/伸手）+ LAFAN（行走/奔跑）+ 自采 RL 轨迹。
  2. 重定向（Retargeting）：将人类 MoCap 骨骼映射到目标人形机器人骨骼，保留运动语义。
  3. 场景程序化生成：随机化场景背景纹理、目标物体颜色/材质、摄像机视角（第一人称头部相机 + 2-3 个随机第三人称相机），以及任务物体的位置和属性。
  4. IsaacSim 光线追踪渲染：模拟真实光照/阴影，减小 sim-to-real 视觉差距。
  5. VLM 语言标注：用 VILA 模型自动为每条轨迹生成多样化文本描述（如 "go sit on the red sofa"，"walk toward the desk"）；部分任务手动标注。
  6. 镜像增强：水平翻转一半演示，免费翻倍数据多样性。

4局限性

⚠️ 注意

合成数据的运动学回放会产生轻微"幽灵效应"（动力学不一致的穿模或漂浮），但作者验证了配合高质量低层策略后 sim-to-real 仍然成功。真正的瓶颈是动作种类有限（主要是导航和坐下），更复杂的全身操作（双臂协调、搬运重物）的 MoCap 数据尚未纳入。

⚠️ 局限：benchmark 未开源

截至 2026-06，LeVERB-Bench 的评测工具包、环境配置和数据集未公开发布，第三方无法复现评测结果。这是一个重要的透明度缺陷。

GRL Discriminator Domain Align

Gradient Reversal Layer + Adversarial Discriminator · 梯度反转层 + 对抗判别器

📌 TL;DR

LeVERB-VL 混训了两类数据——有图像的"视觉语言轨迹"和无图像的"纯语言轨迹"（白噪声代替图像）。为防止这两类数据在 latent space 中分裂成两个孤立的簇，用梯度反转层训练一个判别器反过来迫使 latent 变得"判别不出来"——即两类数据共享同一 latent 词表。

1问题背景

LeVERB-Bench 有两类训练数据：(1) 视觉语言演示（有真实图像 + 语言），(2) 纯语言演示（无图像，输入白噪声图像）。

如果直接混训，VLA 的 encoder 会"看穿"图像是否为白噪声，导致 latent space 被模态区分（视觉 latent vs. 语言 latent 落在不同区域），低层策略无法把从纯语言数据学到的运动技能迁移到视觉语言任务上。

2梯度反转层（GRL）

GRL 是 Ganin & Lempitsky 2015（domain adaptation 经典工作）提出的技巧：

  • 正向传播：输入原样通过，不做任何变换。
  • 反向传播：梯度乘以 -λ（λ > 0），梯度符号翻转。

效果：判别器想最小化分类 loss（分辨出图像是否存在），但通过 GRL 反向传播到 encoder 时梯度翻转，encoder 被迫去最大化判别器的 loss——即 encoder 学会让 latent 无法区分两类输入（模态不变的表征）。

3在 LeVERB 中的效果

消融实验（ND variant，去掉 discriminator）：整体成功率从 58.5% 降至 33.0%，视觉导航任务降幅最大。这证明 GRL 对齐是视觉语言任务泛化的必要条件，而不只是锦上添花。

4类比

可以把 GRL discriminator 类比为"口音统一训练"：你有英语口音的演讲和中文口音的演讲，你想让识别器无法从声音中判断说话人是哪个背景——那么就反向惩罚识别器的任何成功识别，强迫模型提取与口音无关的通用特征。
