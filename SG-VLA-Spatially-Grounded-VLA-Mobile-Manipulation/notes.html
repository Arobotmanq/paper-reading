VLA Vision-Language-Action

Vision-Language-Action Model · 视觉-语言-动作模型

📌 TL;DR

VLA 是一类把「看图 + 理解语言 + 输出机器人动作」端到端合并在同一个模型里的架构。用预训练 VLM（视觉-语言模型）的表征直接驱动机器人控制，无需人工设计中间表示或手写规则。

1定义

标准 VLA 接受三类输入：

  • 视觉输入：相机图像（RGB，有时 + Depth）
  • 语言指令：自然语言任务描述（"Pick up the apple"）
  • （可选）机器人状态：关节角、末端位姿等

输出机器人动作：关节角增量、末端速度、底盘速度等，可以是离散 token（autoregressive）也可以是连续向量（flow matching / diffusion）。

2本文用法

SG-VLA 以 Prismatic VLM 为骨干，接受 Head + Hand 的 RGB + Depth 四路输入，输出 13D 动作向量（底盘 3D + 躯干 1D + 臂 7D + 夹爪 2D）。与标准 VLA 的区别是：SG-VLA 在 VLM backbone 上额外挂了 5 路辅助解码器，提供空间辅助监督。

3对比 · 出处

模型backbone动作输出特点

4类比

VLA 类似于「把人的眼睛（视觉）、耳朵（语言）和手（动作）接进同一个大脑里联合训练」。传统机器人需要分开的感知模块 + 规划模块 + 控制器，VLA 将它们端到端合并。

5具象例子

输入：Head camera 拍到厨房台面，Hand camera 拍到苹果，语言指令"Pick up the apple"。VLA 直接输出：底盘前进 0.1m、臂关节角 +5°、夹爪闭合——无需任何中间规划步骤。

Spatial Grounding 辅助解码器

Spatially-Grounded Representation Learning via Auxiliary Decoders · 空间感知表征学习

📌 TL;DR

在 VLM backbone 的 latent representation 上挂多个辅助解码器，让模型同时重建「机器人在哪、夹着什么、物体在哪、关节怎么摆、目标是什么」这些可解释的空间信号。辅助解码器只在训练时存在，推理时不增加代价，但已让 backbone 学到了更丰富的空间感知表征。

1为什么需要它

标准 VLA 的监督信号只有「动作正确」这一个目标。对于 mobile manipulation，正确的动作需要模型同时理解：

  • 全局场景布局（机器人在房间哪个位置）
  • 目标物体的精确位姿（放哪里）
  • 当前关节配置（怎么到达目标）
  • 抓取状态（夹着没有）
  • 目标物体的视觉边界（分割）

如果只用动作监督，模型无法被「强迫」学习这些信息——它可能走捷径，用浅层特征猜动作。辅助解码器提供了显式的空间监督压力。

2本文的 5 路解码器

解码器架构重建目标作用
Global Position DecoderMLP机器人 2D 坐标 (x, y)导航感知，对 open/close 有显著帮助
Grasp Success DecoderMLP (二分类)是否正在抓取物体抓取状态感知，对 pick 提升 pick+17%
Object Pose DecoderMLP目标物体 3D 位置 + 四元数精确放置所需，place 提升最大
Joint Pose DecoderTransformer12D 关节角配置运动规划感知，全任务最稳定提升
Segmentation DecoderCNN (转置卷积)目标物体 128×128 二值掩码精确目标定位，配合 Object Pose 效果倍增

3损失函数

\[\mathcal{L}_{auxiliary} = \lambda_{pos}\mathcal{L}_{pos} + \lambda_{grasp}\mathcal{L}_{grasp} + \lambda_{qpos}\mathcal{L}_{qpos} + \lambda_{obj}\mathcal{L}_{obj} + \lambda_{seg}\mathcal{L}_{seg}\]

权重设置：\(\lambda_{pos}=1.0,\ \lambda_{grasp}=5.0,\ \lambda_{qpos}=1.0,\ \lambda_{obj}=1.0,\ \lambda_{seg}=1.0\)。grasp loss 权重最高（5.0），因为二分类任务训练信号相对稀疏。

4类比

类似于训练一个会「考试」的学生：不只考最终答案（动作是否正确），还考中间步骤（机器人现在在哪、手里夹着什么、目标物体在哪）。学生为了通过所有考试，被迫学习所有这些知识——即使推理时只用到最终答案。

📜 相关工作对比

SpatialVLA（arXiv:2501.15830）也关注 VLA 的空间表征，但通过改变动作表示（spatial grid action）而非辅助解码器。SG-VLA 的思路是「不改变动作格式，而是加监督信号」——两条不同路线。

Multi-view + Depth 输入增强

Multi-view RGB + Depth Input Enhancement · 多视角 RGB + 深度图输入增强

📌 TL;DR

同时使用安装在头部（全局视野）和手部（近景）的 Head + Hand 相机，各提供 RGB + 归一化 Depth 图，共 4 路输入。Depth 提供显式的几何距离信息，Multi-view 提供互补的全局-局部视角。相比单视角 RGB，组合输入使成功率从 0.04 → 0.60（8×+）。

1为什么需要多视角

Mobile manipulation 的场景理解需要：

  • 全局视角（Head camera）：理解房间布局、目标物体大致位置、导航障碍
  • 局部视角（Hand camera）：精确抓取点、物体纹理细节、接触检测

单视角 RGB 无法同时提供这两类信息。

2Depth 的归一化方式

\[p_{obs} = 1 - \tanh\left(\frac{\text{depth value}}{1000}\right)\]

这个归一化将原始深度值（毫米）映射到 (0, 1)：近处物体值接近 1，远处接近 0。tanh 提供软截断，避免远处噪声过度影响。归一化后 depth 可以像 RGB 通道一样输入 visual encoder。

3消融结果

输入配置平均成功率相对提升
Single-view RGB (OpenVLA baseline)0.04基线
+ Multi-view (Head + Hand RGB)0.52+1200%
+ Multi-view + Depth0.60+15% vs 上行
+ Multi-view + Depth + History (4 帧)0.49-18% 有损

反直觉发现：加入 4 帧历史会降低性能（0.60 → 0.49）。论文认为这是因为 ManiSkill-HAB 的 subtask 设计本质上是 reactive 的（反应式控制），不需要追踪历史轨迹。

4Visual Encoder 如何处理 4 路输入

Prismatic VLM 的 dual encoder（DINOv2 + SigLIP）分别处理每个视角的输入（将 depth 视为灰度图或额外通道），通过 projector 映射到 LLM embedding space，拼接为统一的 token 序列输入 Qwen2.5 LLM。

5类比

类似于人眼的外周视觉（全局）+ 中央凹（精细），再加上触觉（depth）。只有单眼中央凹（single-view RGB）既无周边信息又无深度感——这正是 OpenVLA 7B 只有 4% 成功率的根本原因。

Prismatic VLM 双视觉编码器

Prismatic Vision-Language Model · 棱柱形视觉-语言模型

📌 TL;DR

Prismatic VLM 是 Stanford 提出的 VLM 设计框架，核心是「双编码器融合」——同时用 DINOv2（空间/结构理解强）和 SigLIP（语义/文本对齐强）处理同一张图像，将两组特征拼接后输入 LLM。SG-VLA 用 1.3B 版本（DINOv2 + SigLIP + Qwen2.5-0.5B）作为 backbone。

1为什么用双编码器

单一视觉编码器存在能力偏差：

  • CLIP/SigLIP：通过图文对比学习训练，语义对齐强，但空间感知（精确位置、深度关系）弱
  • DINOv2：自监督训练，局部结构特征和空间关系强，但语义联系弱

Prismatic 将两者特征拼接后输入 LLM，同时获得语义对齐和空间感知能力——这正好匹配 SG-VLA 对「空间理解 + 语义跟随」的需求。

2本文用法

SG-VLA 的 Prismatic backbone：

  • Visual encoder: DINOv2 + SigLIP（处理每路图像输入）
  • Projector: 可训练的线性投影层，将视觉特征映射到 LLM embedding 维度（896D）
  • LLM: Qwen2.5-0.5B（中心推理组件，处理多模态 token 序列）
  • 总参数：~1.3B（远小于 OpenVLA 的 7B）

3对比 OpenVLA

OpenVLA (原始)SG-VLA backbone
视觉编码器SigLIP 单编码器DINOv2 + SigLIP 双编码器
LLMLLaMA-2 7BQwen2.5-0.5B

成功率(SetTable)0.040.60 (仅输入增强)

更小的模型配合更好的视觉编码器设计，在专项任务上碾压更大的通用基线——说明 task-specific 的感知设计远比堆参数重要。

4出处

📜 来源

Karamcheti et al. 2024 — "Prismatic VLMs: Investigating the Design Space of Visually-Conditioned Language Models"（ICML 2024）。系统探索了 VLM 的 visual encoder 设计空间，发现双编码器在多个下游任务上优于单编码器。OpenVLA 基于 Prismatic 框架，SG-VLA 也沿用并扩展了它。

Progressive Training 渐进式训练

Three-Stage Progressive Training Strategy · 三阶段渐进式训练策略

📌 TL;DR

SG-VLA 用三阶段训练解决「随机初始化的辅助解码器会用噪声梯度破坏预训练 VLM」的问题。Stage 1 冻结 backbone 只训解码器 → Stage 2 全梯度联合精调 → Stage 3 冻结所有参数单独训 flow matching head。跳过这个顺序直接共训，平均成功率从 0.60 跌至 0.51。

1为什么需要渐进训练

核心矛盾：辅助解码器是随机初始化的，但 VLM backbone 是预训练的（已有良好的视觉-语言表征）。

  • 随机解码器输出与 GT 差距极大 → 产生大梯度
  • 大梯度通过 backbone 反传 → 破坏 backbone 已学到的表征
  • 结果：backbone 为了满足混乱的解码器要求，放弃了已有的好表征

2三阶段详细流程

阶段时长Backbone解码器Action Head目的
Stage 1
Decoder Adaptation		3 epoch部分训练（仅 action token 路径）训练，Stop gradient无解码器学会解读 backbone 的表征
Stage 2

3Stage 3 为什么要单独训 Action Head

Flow matching 的 denoising loss 在和其他 loss 同时优化时难以收敛。论文发现：即使阻止 flow matching 梯度回传到 backbone（stop gradient），denoising loss 仍然不能在并发训练中有效下降。完全冻结 backbone、只优化 action head，才能让 flow matching 正常收敛。

4类比

类似于医院里「先让新实习生（解码器）跟着经验丰富的主治医（backbone）学，再让他们合作共同进步，最后才让他们独立执行专科手术（action head）」——而不是第一天就让新实习生主刀。

✅ 工程启示

这是预训练模型 co-training 的通用技巧：「先让新组件适应老模型的输出，再一起精调」。适用于任何在预训练 VLM 上增加新模块的场景（如加 depth encoder、加 proprioception encoder 等）。

Flow Matching 连续动作生成

Flow Matching Action Expert · 流匹配动作专家

📌 TL;DR

Flow Matching 是一种生成模型，通过学习「从随机噪声到目标动作的连续变换路径」来生成高精度连续动作。比 diffusion 更快（通常 10 步去噪），比离散 token 精度更高。SG-VLA 将其作为可选 action head（100M 参数），与主 VLM backbone 分离训练（Stage 3）。

1核心原理

Flow Matching 学习一个向量场（vector field）\(\mathbf{v}_\theta(\mathbf{x}, t)\)，定义从噪声分布 \(p_0\) 到目标动作分布 \(p_1\) 的连续变换：

\[\frac{d\mathbf{x}}{dt} = \mathbf{v}_\theta(\mathbf{x}, t), \quad t \in [0, 1]\]

推理时：从随机噪声 \(\mathbf{x}_0 \sim p_0\) 出发，用 ODE 积分器沿向量场积分 \(T\) 步（通常 T=10），得到目标动作 \(\mathbf{x}_1\)。

2本文用法

SG-VLA 的 flow matching action head：

  • 参数量：100M
  • 输入：VLM backbone 的 frozen latent representation
  • 输出：action chunk（8 步动作序列），执行前 2 步后重新规划
  • 去噪步数：10 步
  • 目标：13D 连续动作（底盘 3D + 躯干 1D + 臂 7D + 夹爪 2D）

3离散 token vs Flow Matching：任务类型决定优劣

动作类型离散 tokenFlow Matching结果
Pick（精细抓取）0.130.27（+108%）FM 显著更好
Place（精确放置）0.700.80（+14%）FM 略好
Open Fridge（导航+开门）0.870.76（-13%）离散 token 更好
Open Drawer（导航+开抽屉）0.770.60（-22%）离散 token 显著更好

规律：精细末端执行器控制用 FM（连续精度更高），底盘导航 + 方向性动作用离散 token（决策性更强）。

📜 来源

Lipman et al. 2023 — "Flow Matching for Generative Modeling"（ICML 2023）。机器人控制领域的 flow matching action expert 设计来自 π0（Black et al. 2024），SG-VLA 直接沿用 π0 的 action expert 设计，但训练策略上做了修改（Stage 3 完全隔离训练）。

13-DoF Action Space 移动操作动作空间

13-Dimensional Coordinated Mobile Manipulation Action Space · 13 维协调移动操作动作空间

📌 TL;DR

SG-VLA 控制的动作空间是 13 维连续向量，同时覆盖底盘移动、躯干升降、机械臂 7 轴和夹爪。底盘运动和臂运动必须协调——走到目标附近的同时预置臂的姿态，才能高效完成操作。这远比桌面操作（通常 6-7D）复杂。

1动作向量分解

子空间符号维度含义
底盘\(\Delta X\)3D底盘位置变化（x, y）+ 朝向变化（yaw）
躯干\(\Delta z\)1D躯干高度变化（用于适应不同高度的操作目标）
机械臂\(\Delta q\)7D7 个关节角的增量
夹爪\(\Delta G\)2D夹爪状态变化（两指各一维，用于精细控制）

2协调控制的挑战

Mobile manipulation 的难点在于底盘和臂的协调运动：

  • 底盘移动时臂需要同时调整以保持末端稳定
  • 接近目标时需要底盘精确对准同时臂预置合适位姿
  • 全局导航需要底盘大幅运动，局部操作需要末端精细控制——两种模式的时间尺度和精度要求不同

3与桌面操作的对比

桌面操作（如 OpenVLA）移动操作（SG-VLA）
动作维度6-7D（臂 + 夹爪）13D（底盘 + 躯干 + 臂 + 夹爪）
观测范围单固定视角，桌面有界整个房间，部分可观测
移动能力无底盘全向移动
任务时长通常 <30s可达数分钟

4具象例子

完成「从冰箱取苹果」：① 底盘导航至冰箱前（\(\Delta X\) 变化大），② 调整躯干高度对准冰箱层（\(\Delta z\) 调整），③ 臂伸出开冰箱门（\(\Delta q\) 大幅变化），④ 底盘微调对准苹果，⑤ 臂精细抓取（\(\Delta q\) 小幅、\(\Delta G\) 闭合）。五个阶段动作风格完全不同，需要同一策略无缝切换。

ManiSkill-HAB 评测基准

ManiSkill Home Assistant Benchmark · ManiSkill 家庭助理基准测试

📌 TL;DR

ManiSkill-HAB 是 UC San Diego（本文作者之一 Arth Shukla 参与开发）发布的家务重排 benchmark，基于 Habitat 仿真环境，包含 TidyHouse / PrepareGroceries / SetTable 三类长程任务，分解为 Pick / Place / Open / Close 四类 subtask。SG-VLA 的所有实验都在这个仿真环境中进行。

1任务组成

任务Subtask难度特点
TidyHousePick, Place中-难不规则物体，复杂放置目标
PrepareGroceriesPick, Place难精细操作，小物体，密集排列
SetTablePick, Place, Open, Close易-中覆盖所有 4 类 subtask，用于主要消融

2数据规模

  • 总数据：44K episodes，1.4M transitions
  • 辅助任务数据分配：SetTable 8K/240K（用于 global pos / grasp / qpos），所有 Pick&Place 40K/1.2M（用于 seg / obj pose）
  • 评测：每个 subtask 30 episodes，平均成功率

3局限性

⚠️ 仿真 vs 真实

ManiSkill-HAB 是纯仿真环境（基于 Sapien/Habitat）。SG-VLA 的所有评测结果均在仿真中获得，没有 sim-to-real 验证。仿真中的 depth 是完美的，而真实场景的 depth sensor 存在噪声、遮挡、反光等问题——这是本文最大的局限之一。

4出处

Shukla, Tao, Su — "ManiSkill-HAB: A Benchmark for Low-Level Manipulation in Home Rearrangement Tasks"（ICLR 2025）。注意 Arth Shukla 和 Hao Su 同时是 ManiSkill-HAB 的作者和 SG-VLA 的作者——这意味着 SG-VLA 实质上是在自己开发的 benchmark 上测试，可能存在设计上的协同优化偏差。
