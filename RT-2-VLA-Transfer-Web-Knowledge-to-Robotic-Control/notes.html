VLA Vision-Language-Action

Vision-Language-Action Model · 视觉-语言-动作模型

📌 TL;DR

VLA 是一类同时接受图像+语言输入、输出机器人动作的端到端模型。RT-2 是首个大规模验证 VLA 范式的工作：把 VLM 改造成 VLA 的关键是让动作和语言共享同一 token 空间。

1定义

VLA（Vision-Language-Action）是在 VLM 基础上扩展的机器人控制模型：

类型输入输出
VLM图像 + 文本文本 token
VLA图像 + 文本（任务描述）动作 token（=机器人控制命令）

核心思想：动作可以被表示为特殊的 token，从而 VLM 的训练框架无需修改即可输出机器人动作。

2在 RT-2 里干什么

RT-2 定义并首次大规模验证了 VLA 范式。具体做法：将 PaLI-X 或 PaLM-E 等 VLM，通过 co-fine-tuning（在机器人数据+网页数据混合）直接变成可以闭环控制机器人的 VLA。输入格式是 VQA 风格的字符串 "Q: what action should the robot take to [task]? A:"，输出是动作 token 字符串。

3对比/出处

📊 VLA vs 其他机器人学习范式

vs RT-1：RT-1 是专门为机器人设计的 Transformer 策略，从头训练，无 VLM 预训练；VLA 直接继承 VLM 的网页知识。

vs LLM 高层规划（如 SayCan）：SayCan 用 LLM 做高层任务规划，低层动作由独立控制器执行——LLM 和控制器是两个模型，权重不共享。VLA 是单一端到端模型。

后续工作：OpenVLA（UC Berkeley，2024）、π0（Physical Intelligence，2024）均采用 VLA 范式，是 RT-2 思想的开源延续。

4类比

类比语音识别：把 ASR 的输出不再是文字而是「控制命令」——VLA 做的事本质上相同，只是把"输出空间"从自然语言 token 扩展到包含动作 token。

Action Tokenization 动作 token 化

Action-as-Text-Tokens · 将机器人动作表示为文本 token

📌 TL;DR

RT-2 把机器人的 8 维控制向量（6-DoF 末端执行器位移 + 抓手 + 终止）均匀离散为 256 bins，用整数 token 表示，拼接成像语言一样的字符串——动作从此与语言同质化，无需新架构。

1具体做法

机器人动作空间：6-DoF 末端执行器位置/旋转位移 + 抓手开合程度 + 终止信号 = 8 维向量。

  • 连续维度（7 维）：每个维度均匀离散为 256 个 bin（整数 0–255）。
  • 终止命令：独立的离散 token（0 或 1）。
  • 组合成字符串："1 128 91 241 5 101 127"（依次是 terminate, Δpos_x, Δpos_y, Δpos_z, Δrot_x, Δrot_y, Δrot_z, gripper）。
  • 对 PaLI-X：整数直接对应 tokenizer 中的数字 token（0–999 各有独立 token）。
  • 对 PaLM-E：覆盖 256 个最低频 token 作为动作词表（symbol tuning）。

2为什么这样做

好处：零架构改动——VLM 的训练、推理、损失函数全部不变，只是输出序列的"含义"改变了。代价：离散化有精度损失（256 bins 在连续空间里有量化误差），但对于机器人操作任务已足够。

3出处

📜 起源

离散化动作设计来自 RT-1（Brohan et al., 2022），RT-2 直接沿用了这套 action encoding，只是把 token 嵌入进 VLM 的文本 tokenizer 中。

4类比

类比 DALL-E 的图像 tokenization（VQ-VAE）：把连续图像像素离散成 codebook token，使图像生成可以用语言模型做自回归采样。RT-2 对动作做了类似的事——把连续控制信号离散成整数 token。

Co-Fine-Tuning 联合精调

Co-Fine-Tuning · 在网页数据与机器人数据上同时 fine-tune

📌 TL;DR

co-fine-tuning 是 RT-2 最关键的训练配方：在 fine-tuning 阶段，网页 VQA 数据和机器人轨迹数据同时混在训练 batch 中——防止遗忘 VLM 语义知识，同时学习控制。纯 fine-tuning（只用机器人数据）会让模型遗忘 VLM 能力，泛化崩塌。

1定义

标准 fine-tuning：在新数据集上继续训练，可能遗忘原始知识（灾难性遗忘）。

co-fine-tuning（RT-2 的叫法）= 在 fine-tuning 阶段同时保留原始预训练数据混合：

训练 batch =
  [机器人轨迹样本 × 采样权重_robot]
  + [网页 VQA / 图文 样本 × 采样权重_web]

RT-2-PaLI-X：机器人数据约占 batch 的 50%
RT-2-PaLM-E：机器人数据约占 batch 的 66%

2为什么有效

机器人数据（~100k 轨迹）远少于 VLM 预训练数据（~1B 图文对）。单独用机器人数据 fine-tune 会覆盖 VLM 权重，导致模型"忘记"语义概念。co-fine-tuning 通过同时喂网页数据，维持了 VLM 知识，使其在机器人任务上也能利用这些语义能力。

3消融证据

训练方式未见物体/场景均值说明
from scratch（5B）9%完全不用 VLM 预训练
fine-tuning（5B）42%仅机器人数据 fine-tune
co-fine-tuning（5B）46%混合训练
co-fine-tuning（55B）62%规模进一步提升

4类比

类比多任务学习（Multi-Task Learning）：把语言任务（VQA）和机器人任务同时训练，互相防止过拟合，并让两类任务共享底层特征表示。

PaLI-X VLM Backbone

Pathways Language and Image model, X-scale · 多语言视觉语言模型

📌 TL;DR

PaLI-X 是 Google 的大规模多语言视觉-语言模型，ViT-22B（图像）+ encoder-decoder LLM（32B）。RT-2 基于其 5B 和 55B 版本，是论文中泛化能力最强的 backbone。

1架构

  • 图像编码器：ViT-22B（Vision Transformer，220 亿参数）——极大的视觉骨干。
  • 语言模型：encoder-decoder 结构，32B 参数，类 UL2；图像 token + 文本 token 共同进 encoder，decoder 自回归输出。
  • 总参数：RT-2-PaLI-X-55B（55B = ViT-22B + LLM-33B 左右）；RT-2-PaLI-X-5B（5B 精简版）。

2在 RT-2 里的特殊性

PaLI-X 的 tokenizer 对整数 0–999 各有独立 token，非常方便直接用整数表示离散化的动作 bin（无需覆盖任何已有 token）。这使得 action tokenization 对 PaLI-X 尤为自然。

3与 PaLM-E 对比

📊 PaLI-X vs PaLM-E in RT-2

PaLI-X：encoder-decoder，视觉骨干更大（ViT-22B），整体泛化/符号理解更强，尤其 55B 版本在涌现能力上高达 60% 均值。

PaLM-E：decoder-only，12B，在数学推理上略占优（因预训练数据配比中含更多数学），且支持 chain-of-thought（RT-2 用 PaLM-E 版本做了 CoT 实验）。

PaLM-E Embodied VLM

PaLM-Embodied · 具身多模态语言模型

📌 TL;DR

PaLM-E 是 Google DeepMind 专门为具身 AI 设计的多模态 LLM，decoder-only 架构，支持交叉的图像-文本序列输入。RT-2-PaLM-E-12B 版本用于验证方法对 backbone 的通用性，并做了 chain-of-thought 实验。

1架构

  • 结构：decoder-only LLM（PaLM 基础）+ 视觉投影层（ViT-4B 投影图像到语言 embedding 空间）。
  • 多模态输入：将连续变量（图像、传感器读数等）投影进语言 token 空间，支持交叉的图文序列。
  • 尺寸：12B 参数。

2在 RT-2 里的特殊性

PaLM-E 的 tokenizer 没有为整数提供独立 token，因此 RT-2 使用 symbol tuning 方法：覆盖 256 个最低频 token作为动作词表。这验证了 action tokenization 在 tokenizer 不友好的情况下同样可行。

PaLM-E 版本用于实现 chain-of-thought 推理（因其 decoder-only 结构更自然支持先输出文本 plan 再输出动作的混合序列）。

3出处

📜 来源

PaLM-E 由 Driess et al. 2023 提出（Google）。论文原本就将 PaLM-E 设计为具身 AI 的基础模型，RT-2 是其在闭环机器人控制中的直接应用验证。

Emergent Capabilities 涌现能力

Emergent Capabilities · 从网页预训练迁移到控制的涌现能力

📌 TL;DR

RT-2 的"涌现能力"指的是：机器人训练数据中完全没有出现过的任务（如"将物体移到数字 3 旁"、多语言指令、识别具体名人），RT-2 能成功执行——这些能力纯粹从 VLM 的网页预训练中迁移而来。

1三类涌现能力

类别示例指令需要的知识
Symbol Understanding"move coke can near 3"
"push coke can on top of heart"		数字/符号的语义理解，机器人数据中未出现
Reasoning"move banana near the sum of two plus one"
"mueve la manzana al vaso verde"（西班牙语）
"pick a healthy drink"		数学计算、多语言、营养常识
Human Recognition"move coke can to Taylor Swift"
"move coke can to person with glasses"		名人识别、人脸属性理解

2为什么叫"涌现"

参考 LLM 领域的涌现能力定义（Wei et al., 2022）：能力在小模型上不存在，在大模型上突然出现。RT-2 的用法略有不同——指从 VLM 预训练迁移到控制任务的能力，而非严格意义上的规模涌现。这些能力对 RT-1（同等机器人数据，无 VLM）完全不出现，对 RT-2 则大幅出现（3×+）。

3重要限制

涌现的是语义理解和部署技能的方式，而不是新的物理动作技能。RT-2 不能"看视频学会用毛巾擦桌子"——运动技能仍受限于机器人训练数据。

4类比

类比人类：一个学会了"抓取"动作的婴儿，通过语言学习理解"把苹果给妈妈"——他不需要重新学习如何抓，只需要把语义理解和已有动作连接起来。RT-2 正是这种机制的机器学习实现。

Chain-of-Thought for Control CoT 控制

Chain-of-Thought Reasoning for Robotic Control · 机器人控制中的链式思维推理

📌 TL;DR

RT-2 通过少量数据增强（几百步 fine-tune），让模型先输出自然语言"Plan"再输出动作 token——用 LLM 的 chain-of-thought 能力桥接语义推理和低层控制，同一模型既是规划器又是执行器。

1实现方式

数据增强：在少量轨迹上增加 Plan 字段，格式为：

Instruction: I'm hungry.
Plan: pick rxbar chocolate.
Action: 1 128 124 136 121 158 111 255.

训练时让模型预测整个序列（Plan + Action）。推理时，模型先生成 Plan（自然语言），再生成 Action（动作 token）。

2意义

这是首次在单一 VLA 模型中实现"先推理再行动"——不需要独立的规划模型+控制器，也不需要多次 API 调用。Plan 文本充当了"工作记忆"，让模型解决更复杂的多步语义问题（如"需要锤子 → 找石头"）。

3与分层规划对比

📊 单模型 CoT vs 双模型规划

双模型方案（如 SayCan）：LLM 规划器 → 生成高层计划 → 低层策略执行。两个模型独立，权重不共享，接口复杂。

RT-2 CoT：单一模型，Plan 和 Action 都是同一序列的 token——高层推理和低层控制完全共享权重，信息流无损。

4出处

📜 来源

Chain-of-thought prompting 来自 Wei et al. (2022) "Chain of Thought Prompting Elicits Reasoning in Large Language Models"。RT-2 把这个思想从纯文本推理迁移到了机器人控制序列中。

RT-1 Predecessor

Robotics Transformer 1 · 机器人学习的 Transformer 前辈

📌 TL;DR

RT-1 是 RT-2 的前身，同样来自 Google 机器人团队（Brohan et al., 2022）。35M 参数的纯机器人 Transformer，没有 VLM 预训练。RT-2 使用与 RT-1 完全相同的机器人数据，但通过 VLM co-fine-tuning 在泛化和涌现能力上大幅超越它——对比 RT-1 是理解 RT-2 贡献的最直接方式。

1架构对比

RT-1RT-2
参数规模35M5B / 12B / 55B
预训练数据无（从头训练）网页图文 ~1B+ 样本
动作表示离散 bin，专用 action headaction-as-text-tokens，同语言 head
seen tasks 性能92%91%（持平）
泛化未见场景均值32%62%（~2×）
涌现能力均值17%60%（~3.5×）

2为什么 RT-1 做不到涌现能力

RT-1 仅在机器人轨迹数据上训练，从未见过"数字 3"、"泰勒·斯威夫特"或西班牙语的语义含义——它没有这些概念的内部表示，自然无法执行相关任务。RT-2 通过继承 VLM 的知识获得了这些表示。

3关系

RT-2 = "RT-1 的数据集" + "VLM 的知识" + "action-as-text-tokens 的统一表示" + "co-fine-tuning 的训练配方"。
