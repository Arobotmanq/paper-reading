Soft Prompt Core Innovation

Soft Prompt / Soft-Prompted Transformer · 软提示 / 软提示 Transformer

📌 TL;DR

Soft Prompt 是一组随机初始化、端到端学习的可学习嵌入向量，每个数据源（embodiment）独享一组，注入 Transformer 编码器的输入端。它让模型从编码早期就感知到"当前是哪个机器人硬件"，而无需修改任何预训练主干权重，也不需要人手工写硬件描述。X-VLA 里所有 embodiment-specific 参数（soft prompt + 输入/输出线性投影库）合计仅占模型总参数的 0.04%。

1定义

Soft Prompt 源自 NLP 中的 prompt tuning（Lester et al. 2021）。与"hard prompt"（固定文字模板）不同，soft prompt 是：

  • 连续向量：不是离散 token，是实数空间中的可学习参数 \(p_i \in \mathbb{R}^k\)
  • 随机初始化：不依赖人手工设计，从 \(\mathcal{N}(0, I)\) 初始化，靠端到端训练学出语义
  • 与模态无关：可直接 concat 到任意 Transformer 输入序列的开头/中间
\[ p_i \approx \Phi(h_i), \quad \Phi: \mathcal{H} \to \mathbb{R}^k \]

其中 \(h_i\) 是第 \(i\) 个数据源的硬件配置（相机、控制频率、机械臂型号等），\(\Phi\) 是隐式学到的从硬件配置到提示空间的映射。

2在 X-VLA 里干什么

X-VLA 建立了一个 Soft Prompt Library：每个数据源（dataset id）对应一组提示向量，推理时按 dataset id 查询。具体位置：

输入序列 = [Soft Prompt(p_i)] + [Multi-modal Tokens] + [Control Tokens]
                ↓
         N × Standard Self-Attention Transformer Block
                ↓
            Action Chunk

除了 soft prompt 本身，X-VLA 还配套了 Input Linear Projection Library（每个 embodiment 独立映射低维本体/动作 token 到高维特征空间）和 Output Linear Projection Library（每个 embodiment 独立映射动作预测输出回低维控制信号）。这三个库合计 0.04% 参数，但作用于整个前向传播。

3与其他异构性处理方案的对比

方案处理位置覆盖异构性维度缺点
Domain-specific action head（π0/GR00T）输出端仅动作空间视觉/语义层面的 embodiment 差异完全未处理
HPT-style 投影层中间观测处理视觉+动作破坏预训练 VLM 表征 → 训练不稳定
语言提示输入端（文字）视觉+动作+任务需要手工写硬件描述，无法量化扩展到新 embodiment
Soft Prompt（本文）输入端（向量）全维度需要新 embodiment 的少量 demo 来训练新 prompt；提示空间维度 k 是超参

4出处

📜 起源

Lester et al. 2021 — "The Power of Scale for Parameter-Efficient Prompt Tuning"（Google Brain）：在 GPT/T5 等大型语言模型上，用少量可学习 token 前缀替代全量 fine-tuning，参数量少 1000× 但性能相当。X-VLA 直接引用此工作，并将其从 NLP 迁移到跨本体机器人学习——用 dataset id 查询对应 prompt 的方式，与多任务 prompt tuning 完全对应。

5类比 + 具象例子

想象你是一个多语种翻译官，每次接待客人前先戴上不同颜色的"耳机"（soft prompt）——耳机会过滤和放大特定语言的频率，让你的大脑（VLM backbone）无需改造就能切换工作模式。新来一个你没听过的方言？只需用 10 分钟录几句话（少量 demo），新耳机就训好了（两步适配），大脑权重完全不动。

Cross-Embodiment Core Problem

Cross-Embodiment Learning / Training · 跨本体学习 / 训练

📌 TL;DR

跨本体训练是指用来自不同机器人硬件平台的数据联合训练一个通用策略模型。核心挑战是：不同 embodiment 的数据在动作空间、相机配置、控制频率、任务类型上全都不同，简单混训会导致性能下降而非提升。X-VLA 的 soft prompt 机制是目前解决这一问题最简洁的公开方案之一。

1为什么需要跨本体训练

单个机器人平台的数据量往往不足以训练强大的基础模型。多个机器人数据联合训练可以：

  • 增加数据多样性 → 更好的视觉/语义泛化
  • 共享低层操作技能（抓取、放置等）跨 embodiment 迁移
  • 减少每个新机器人的数据采集负担（迁移+适配 vs. 从零采集）

2具体异构性维度

维度X-VLA 预训练数据中的实例
机械臂类型Franka（单臂）· UR5（单臂）· Dual-Franka（双臂）· Agilex（双臂）· AGIBOT
控制频率15 Hz（Droid）· 30 Hz（AGIBOT/RoboMind）
相机配置Top View · Left/Right View · Front+Wrist · Head+Wrist
动作空间维度7-DOF EEF（单臂）· 14-DOF EEF+gripper（双臂）
任务分布抓取/放置（AGIBOT） · 桌面操作（RoboMind） · 多样场景（Droid）

3X-VLA 中的解决思路

X-VLA 同时用了三层机制：① Soft Prompt（感知 embodiment）② 解耦编码流水线（主视角/辅助视角分开处理）③ 对齐动作表示（统一转换到 EEF absolute pose + Rotate6D）。三层配合，实现"骨架不感知 embodiment，只有提示感知"。

4类比

如同培训一个能适应不同医院系统的护士：医疗知识（骨架）通用，但每家医院的电子病历格式、设备操作界面（embodiment-specific）不同。Soft prompt 相当于给护士配一本该医院专用的"快速手册"（轻量提示），而不是让她重新学医（重新训练骨架）。

Scalable VLA Architecture

Scalable Vision-Language-Action Model · 可扩展的视觉-语言-动作模型

📌 TL;DR

X-VLA 的"可扩展性"指：沿模型规模、数据多样性、数据量三个轴增长时，预训练验证误差单调下降且无饱和迹象（R²=0.925 强相关于下游成功率）。实现可扩展的关键是把 action decoder 从 DiT（扩散 Transformer）换成标准 self-attention Transformer 编码器栈，避免了 DiT 的 AdaLN 条件导致的训练不稳定。

1为什么 DiT 不够 scalable

早期实验（Table 1）显示：用标准 DiT-base 做 action decoder 时，在异构数据上预训练后适配成功率仅 25%（比无预训练的 39.6% 还低！）。原因是 DiT 的 Adaptive Layer Norm（AdaLN）用时间步 \(t\) 做条件，在跨 embodiment 混训时容易出现特征分布混乱。

2X-VLA 的架构选择

X-VLA 改用标准 self-attention Transformer 编码器做 action denoiser（替代 DiT），让 VLM 的多模态 token 和 noisy action token 一起输入 Transformer，通过标准注意力机制融合。这一改动使 Val Error 从 0.071 降至 0.053（+16.7% 适配成功率，见 Table 1）。

3三轴 Scaling 行为

Scaling 轴实验范围趋势
模型规模Hidden 256→1024, Layers 4→24单调下降，无饱和
数据多样性1→7 个数据源单调下降，无饱和
数据量20K→290K episodes单调下降，无饱和

验证误差与下游适配成功率的 Pearson 相关系数 R²=0.925——这意味着可以用廉价的验证误差替代昂贵的真机/仿真评测来做架构搜索。

4对比 π0/GR00T N1

π0 使用 PaliGemma(2B VLM) + 独立 flow matching action expert（300M），总 3B 参数。X-VLA 使用 Florence-2-Large + 共享 Transformer 编码器，总 0.9B 参数，以更小规模在大多数 benchmark 上超过 π0——说明架构效率更高。GR00T N1(3B) 也在 LIBERO 上被 X-VLA-0.9B 超越（98.1% vs. 93.9%）。

Flow Matching Action Head

Flow-Matching Policy / Action Head · 流匹配动作头

📌 TL;DR

Flow matching 是一种生成式建模方法：学习一个速度场（velocity field），将高斯噪声样本沿 ODE 轨迹"流动"到目标动作分布。相比扩散模型（DDPM），使用最优传输（OT）路径的 flow matching 路径更直接、去噪步数更少（通常 10 步），训练目标更简单。X-VLA 用 flow matching 生成 action chunk（未来 4 秒的 30 个锚点动作序列）。

1数学形式

X-VLA 采用最优传输（OT）路径的 flow matching：

\[ \mathcal{L}^{FM}_{BC}(\theta) = \mathbb{E}_{t \sim \mathcal{U}(0,1),\, (o, A) \sim \mathcal{D}} \left\| v_\theta(A_t, o, t) - (A - A_0) \right\|^2 \]

其中 \(A_t = (1-t)A_0 + tA\)，\(A_0 \sim \mathcal{N}(0, I)\) 是初始噪声，\(A\) 是专家动作 chunk，\(t \in [0,1]\) 是连续时间变量。推理时用 Euler-Maruyama ODE：

\[ A_{t+\Delta t} = A_t + v_\theta(A_t, o, t) \cdot \Delta t \]

2在 X-VLA 里的角色

不同于 π0 把 flow matching 放在独立的 action expert（300M 参数，与 VLM 分离），X-VLA 让噪声 action token 和 VLM 多模态 token 一起进入共享 Transformer 编码器：

[Soft Prompt] + [Multi-modal Tokens: image + lang + proprio] + [Control Tokens: noisy A_t + flow_t]
                            ↓ N × Standard Self-Attention Transformer
                       [Denoised Action Chunk]

Flow-time \(t\) 和 noisy action \(A_t\) 被 concat 后通过 Input Linear Projection 映射到高维，与其他 token 一起做 self-attention。输出端通过 Output Linear Projection 映射回低维动作空间。

3Action Representation

X-VLA 的动作空间统一为 End-Effector (EEF) pose：

  • Cartesian xyz position（3D）
  • Absolute EEF rotation — Rotate6D（6D 连续表示，避免 Euler 角的不连续性）
  • Gripper state（二值化，BCE loss）

Intention abstraction（意图抽象）：预训练时对动作轨迹做时序下采样，生成"未来 4 秒的 30 个锚点"（而非每步的精确动作），让模型学习高层意图而非低频噪声动作。

VLM Backbone Foundation

Vision-Language Model Backbone · 视觉-语言模型主干

📌 TL;DR

VLA 模型的"大脑"——预训练在海量图文数据上的视觉-语言编码器，提供开箱即用的场景理解和语言跟随能力。X-VLA 使用 Florence-2-Large（Microsoft, 0.77B 参数）作为主视角 + 语言的编码器，冻结 or 以极低学习率微调，以保留预训练表征。

1Florence-2 简介

Florence-2（Xiao et al. 2024, Microsoft）是一个统一视觉-语言基础模型，训练于大规模图像-文字数据集（FLD-5B，54 亿个标注），支持图像描述、物体检测、分割、OCR 等多种视觉任务。其编码器采用 ViT-Large 视觉backbone + 文字 token，输出多模态嵌入序列。

2在 X-VLA 中的角色

X-VLA 使用 Florence-2-Large 处理主视角图像 + 任务语言指令，输出多模态 token 序列，作为 action denoising Transformer 的条件信息。

设计选择：腕部相机等辅助视角 不经过 VLM（避免高频嘈杂图像污染语言推理），而是走 Shared ViT（共享视觉编码器），单独编码后与 VLM 输出 concat 进 Transformer。这一"解耦"设计对比 π0/GR00T 直接把所有视角喂给 VLM 的做法更加精细。

3为什么不用 Gemma/PaliGemma（π0 的选择）

π0 使用 PaliGemma（SigLIP-400M + Gemma-2B），总 VLM 参数约 2.6B。X-VLA 选用 Florence-2-Large（约 0.77B），在保持强视觉-语言表征的同时大幅缩小模型体积，配合更轻量的动作 Transformer，总参数 0.9B。这是工程效率与性能的权衡选择，实验结果证明在多数 benchmark 上 Florence-2 够用。
