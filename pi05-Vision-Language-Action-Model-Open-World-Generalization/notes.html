异构 co-training Heterogeneous Data

Co-training on heterogeneous data sources · 多源异构数据联合训练

📌 TL;DR

把「别的机器人采的动作」「网页图文问答」「人给的口头指令」「高层子任务标注」等完全不同类型的数据，全部铸成同一种 token 序列，丢进同一个模型一起训练。π0.5 的关键发现：直采移动操作数据只占 2.4%，泛化能力主要来自其余 97.6% 的异构迁移。

1为什么能混在一起训

VLA 的核心技巧是「万物皆 token 序列」：图像 patch、文本、bounding box、离散动作 token，都映射进同一个 transformer 的输入/输出序列。于是「机器人模仿学习」就退化成「下一 token 预测」——和训练一个语言模型没有本质区别。既然如此，任何能写成这种序列的数据都能拼进同一个 batch 联合优化。

2π0.5 的六类数据源

代号含义主要作用
MMMobile Manipulator：~400h 移动双臂机器人在 ~100 个真实家庭采的家务数据最贴近评测任务，但仅占 2.4%
MEMulti-Environment：非移动（固定）单/双臂在多样家庭采的数据跨「环境」迁移，便携故覆盖面更广
CECross-Embodiment：实验室桌面多种机型的多任务数据 + 公开 OXE跨「任务/本体」迁移
HLHigh-Level：把「打扫卧室」拆成「捡起枕头」等子任务的标注教模型做高层语义推理
WDWeb Data：captioning、VQA、物体定位等网页多模态数据带来对物体的广博语义（尤其 OOD）
VIVerbal Instruction：专家实时用语言「遥操作」给已训策略示范好的子任务（仅后训练）低成本高杠杆地校准高层策略

预训练用 MM+ME+CE+HL+WD（动作全离散化）；后训练聚焦移动操作，去掉 CE、加入 VI，并引入 flow matching action expert。

3为什么「97.6% 非目标数据」反而有用

核心论点：复杂长程任务（扫整个厨房）无法靠在目标平台蛮力采数据覆盖所有场景——成本不可行。于是改为「迁移」：从其它机器人学会怎么抓盘子、从网页学会认识没见过的物体、从口头指令学会怎么把大任务拆成步骤。论文 Figure 8 的反事实证明：只用 104 个场景的纯移动数据、不做完整 co-training 预训练，性能崩塌；而完整配方达到了「直接在测试家训练」的上界。

🎯 与 π0 / 一般 VLM co-training 的区别

过去 VLA 多是「用 VLM 的图文数据 co-train」改善物体/背景泛化。π0.5 把范围扩到机器人相关的更广监督源（其它机器人、高层子任务、口头指令），并实证：正是这套特定组合让移动机器人能在全新环境完成长程行为。详见 两级分层推理 与 FAST。

两级分层推理 High/Low-Level

Hierarchical high-level / low-level inference · 同一模型的语义子任务 → 低层动作

📌 TL;DR

推理时模型先回答「现在该干哪个子任务」（如 pick up the plate），再以这句话为条件生成低层动作 chunk。关键：高层和低层是同一套权重，不是两个模型——更像 chain-of-thought 把「思考」和「行动」串在一个模型里。

1形式化分解

模型联合分布按链式法则拆成两部分：

\[ \pi_\theta(a_{t:t+H}, \hat{\ell}\,|\,o_t, \ell) = \underbrace{\pi_\theta(a_{t:t+H}\,|\,o_t, \hat{\ell})}_{\text{low-level: 动作}} \cdot \underbrace{\pi_\theta(\hat{\ell}\,|\,o_t, \ell)}_{\text{high-level: 子任务}} \]
  • \(\ell\)：人给的总指令（如「把碗碟放进水槽」）；
  • \(\hat{\ell}\)：模型自己预测出的语义子任务（如「拿起杯子」）；
  • 注意：低层动作分布只依赖 \(\hat{\ell}\)，不直接依赖 \(\ell\)——总指令的语义被「蒸馏」进了子任务这个文本瓶颈。

2运行节奏

高层子任务以较低频率更新（一个子任务管一小段），低层动作 chunk 以 50 Hz 高频输出。这与「embodied chain-of-thought」每步都推理不同，省算力且更稳定。一次低层推理＝自回归解出子任务文本 → 10 步 flow matching 去噪出连续动作（见 flow matching action expert）。

3为什么不用两个模型

π0.5：单模型两级常见方案：VLM 规划 + 独立低层策略
权重同一套（高/低共享表征）两套，分别训练
知识来源高低两级都吃同一异构数据，互相塑造规划器靠 LLM 常识，低层靠机器人数据，割裂
类比chain-of-thought / test-time compute经典 task-and-motion 分层
🤯 最反直觉的发现

Figure 13：完整 π0.5 甚至超过人类 oracle 高层策略；而次优的是 implicit HL——推理时根本不显式分层，但训练里包含了子任务预测数据。这说明显式分层有增益，但大部分好处只靠「把高层数据放进 co-training 混合」就已获得——分层数据在悄悄塑造低层表征。用 GPT-4 零样本当高层策略反而最差，印证「高层也必须用机器人数据适配」。

FAST Action Tokenizer

压缩式离散动作分词（Pertsch et al. 2025，π0.5 预训练阶段使用）

📌 TL;DR

把连续动作 chunk 用类似「频域变换 + 压缩编码」的方式压成一串离散 token，于是动作就能像文字一样进自回归 transformer 训练。π0.5 预训练阶段全靠它把动作变成文本 token，训练又快又稳。

1解决什么痛点

动作是连续高维向量，还是 chunk（一段未来轨迹）。若朴素地每维离散成一个 token，序列又长又冗余，自回归训练慢。FAST 借鉴信号压缩思路（DCT 频域 + 熵编码类压缩），把一个动作 chunk 压成少量、信息密集的离散 token，大幅缩短序列、提高训练效率。

2在 π0.5 里的角色

π0.5 两阶段的动作表示对照
预训练 (alpha=0)：动作 --FAST--> 离散 token --> 和文本一样做下一token预测
                ↑ 训练快、与 VLM 自回归接口同构、语言跟随强

后训练 (alpha=10)：保留文本/FAST 的交叉熵 + 新增 action expert 走 flow matching
                ↑ 推理时改用连续动作，实时、细粒度

论文用注意力矩阵保证「FAST 离散表示」与「flow matching 连续表示」互不注意，两条动作通路共存于一个模型。

3为什么不直接全程用 FAST

离散 token 推理时要逐 token 自回归解码，对实时控制太慢；而且离散化损失细粒度。所以 π0.5 的设计是「离散训练 + 连续推理」：用 FAST 拿到快而稳的预训练，再用 flow matching 拿到实时连续控制。论文也指出（引 Pertsch et al.）：按计算量算，FAST token 训练比纯 diffusion 训练更高效，这也是 π0.5 优于 π0 的原因之一。

Flow Matching Action Expert Continuous Action

流匹配 + 专属 action-expert 权重 · 连续动作生成（继承自 π0）

📌 TL;DR

Flow matching 是一种类 diffusion 的连续生成方法：学一个「向量场」，从纯噪声出发迭代积分若干步，得到平滑连续动作 chunk。Action expert 是 transformer 里专门处理动作 token 的一套独立权重（300M，类 MoE），让动作生成用比主干小得多的模型实时跑。π0.5 后训练才加入它。

1Flow matching 怎么生成动作

构造从噪声到真动作的线性插值轨迹：

\[ a^{\tau,\omega}_{t:t+H} = \tau\, a_{t:t+H} + (1-\tau)\,\omega,\qquad \omega \sim \mathcal{N}(0, I),\ \tau \in [0,1] \]

模型被训练去预测流场（向量场）\(\omega - a_t\)；推理时从噪声 \(\omega\) 出发，沿预测的流场积分 10 步即得连续动作 chunk。相比逐 token 自回归，这是非自回归、并行的，适合实时控制。

2Action Expert：动作的「专科医生」

  • 沿用 π0 设计：动作 token 线性投影进 transformer 嵌入空间，用一套独立的专家权重处理（类似 mixture-of-experts）。
  • 它可以显著小于主干 LLM（这里 300M vs 主干 ~3B），因为只需专注「把条件 + 噪声映射成流场」。
  • 后训练时它随机初始化从头学，主干则继承预训练好的 VLA 权重。

3联合损失（Eq. 1）

\[ \mathbb{E}\Big[\, \underbrace{H\big(x_{1:M}, f^\ell_\theta(o_t,\ell)\big)}_{\text{文本/FAST 交叉熵}} + \alpha\, \underbrace{\big\| \omega - a_{t:t+H} - f^a_\theta(a^{\tau,\omega}_{t:t+H}, o_t, \ell)\big\|^2}_{\text{flow matching 回归}} \Big] \]
  • 预训练 \(\alpha=0\)：退化成纯 VLM 自回归训练（动作即 FAST 文本 token），稳、且语言跟随强；
  • 后训练 \(\alpha=10\)：开启 flow matching，让 action expert 学连续动作，同时保留交叉熵以不丢文本能力。
🔗 与 FAST 的分工

FAST＝训练期的离散动作表示（快、稳、利于预训练）；flow matching action expert＝推理期的连续动作生成（实时、细粒度）。两者在同一模型里靠注意力掩码隔离，是 π0.5「离散训练 + 连续推理」混合方案的两半。
