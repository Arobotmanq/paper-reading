VLA Vision-Language-Action

Vision-Language-Action Model · 视觉-语言-动作模型

📌 TL;DR

VLA 是把视觉编码器 + 语言模型 + 动作输出头组合成一个统一模型，直接从 RGB 图像 + 语言指令端到端输出机器人动作（通常是末端器 7-DoF 位姿）的基础模型。关键优势是在固定基座操作任务上的强泛化能力。

1定义

VLA 模型接受三类输入：视觉观测（RGB 图像）、语言指令（如 "pick up the red cup"）、机器人本体感知状态（关节角、末端器位姿等），直接输出机器人动作（通常为末端器 \([Δx, Δθ, ΔG]\) 的增量控制指令）。其核心是利用在互联网规模数据上预训练的视觉-语言模型（VLM）作为主干，注入操作任务数据微调后获得操作策略。

2本文用法

本文使用 OpenVLA-7B 作为 VLA 后端，通过 LoRA 在 200 条 OVMM 专家轨迹上微调 10K epochs（4× RTX 3090），使其适配 Hello Robot Stretch 底盘+臂的末端器动作空间。VLA 在 MoManipVLA 中只被当作路径点生成器（waypoint oracle）使用，仅输出末端器 waypoint，不输出底盘动作。

3对比 / 出处

模型架构特点
OpenVLA-7B（本文使用）Prismatic VLM + 动作解码头开源，Apache-2.0，跨任务泛化好
π0VLM + flow matching action expert连续动作，跨机器人本体
TinyVLA小型 VLM + 扩散策略解码器数据效率高，视角鲁棒性强
RDT-1B扩散基础模型双臂操作，统一动作空间

4类比

VLA 之于机器人操作，类似 GPT 之于文本生成——用大规模预训练获取跨场景通用能力，再用少量任务数据微调。本文的洞察是：VLA 的"通用能力"体现在末端器运动路径（waypoint）上，这个能力可以"拆出来"跨本体迁移。

5具象例子

输入："Pick up the sponge" + 当前 RGB 帧 + 末端器当前位姿 → VLA 输出：末端器应往哪移动 \(Δx = [0.05m, 0, -0.02m]\)，旋转 \(Δθ\)，夹爪开合度 \(ΔG\)。这就是"路径点"，但此时 VLA 并不知道底盘应该在哪里。

Mobile Manipulation 移动操作

Mobile Manipulation · 移动操作

📌 TL;DR

移动操作 = 底盘（可移动）+ 机械臂（可操作）的整体控制问题。难点在于需要协同规划底盘和臂的动作，搜索空间是二者自由度之和（本文 3+7=10 DoF）。

1定义

移动操作机器人通常由一个可移动的底盘（如轮式平台，3 DoF：x, y, 旋转）和安装在其上的机械臂（如 7-DoF 臂）组成。与固定基座操作的区别在于：底盘本身可以移动，因此可以在大工作范围内执行任务——但同时也带来了底盘+臂协同控制的难题。

2本文场景

OVMM（Open Vocabulary Mobile Manipulation）benchmark：60 个模拟房间场景，18k+ 3D 物体模型；任务定义为"将物体从容器 A 移到容器 B"，含导航、注视、抓取、导航、放置五个子阶段。真实世界平台：hexman echo plus 底盘 + RM65 七轴机械臂。

3与固定基座操作的核心差异

维度固定基座操作移动操作
工作范围臂的工作空间（~1m³）整个场景（可达任意位置）
DoF7（臂）10（3底盘+7臂）
训练数据VLA 有大量数据（OXE 等）数据稀缺，成本高
可达性固定，静态工作空间动态，底盘位置影响臂的可达性

4类比

类比人类：固定基座操作 = 坐在椅子上够桌上的东西；移动操作 = 站起来走到不同房间去拿东西并放回来——需要走路（底盘）和伸手（臂）的协调。

Waypoint Generation 路径点生成

End-Effector Waypoint Generation · 末端器路径点生成

📌 TL;DR

VLA 不直接输出底盘速度或关节角，而是输出末端器在机器人基座坐标系中的增量位姿路径点 \([Δx, Δθ, ΔG]\)。这些路径点具有语义意义（"移到物体上方""下压夹取"），是 MoManipVLA 整个迁移框架的起点。

1定义

路径点（waypoint）是机器人末端器（end-effector，即夹爪或手腕）在空间中应经过的关键位姿序列。在本文中，每个路径点是一个 7 维向量：

\[ Q_i = [\Delta x, \Delta\theta, \Delta G] \]

其中 \(\Delta x \in \mathbb{R}^3\) 为位置增量，\(\Delta\theta \in \mathbb{R}^3\) 为旋转增量（欧拉角），\(\Delta G \in \mathbb{R}^1\) 为夹爪开合度。

2本文用法

VLA 模型基于当前 RGB 图像 + 本体感知状态 + 语言指令，按照固定基座 VLA pipeline 预测 \(Q_i\)。然后通过变换函数 \(\Gamma\) 将其从基座坐标系转换到世界坐标系：\(\hat{Q}_i = \Gamma(P^0_i, Q_i)\)，再由双层优化框架生成可执行的底盘+臂轨迹。

3为什么路径点可以跨本体迁移

路径点捕获的是操作的语义意图（"末端器应到哪里"），而非特定机器人的关节角配置。因此，在固定基座机器人上训练出来的 VLA 预测的路径点，在语义上对移动机器人同样有效——只要移动机器人能够物理上到达这些路径点即可（这正是双层优化要解决的问题）。

4具象例子

VLA 看到"桌上有个海绵"，预测：waypoint 1 = 末端器移到海绵正上方 5cm；waypoint 2 = 下降 5cm；waypoint 3 = 夹爪闭合。这三个路径点对固定臂和移动臂都有同样的语义意义，但固定臂可能工作空间内直接可达，而移动臂则需要先调整底盘位置。

Bi-Level Trajectory Optimization 双层轨迹优化

Bi-Level Trajectory Optimization · 双层轨迹优化

📌 TL;DR

把底盘（3 DoF）和臂（7 DoF）的联合 10-DoF 轨迹搜索分解为两层：上层先优化底盘位姿以扩大臂的可行搜索空间，下层再在确定底盘后优化臂轨迹。贪心分解降低了搜索复杂度，同时避免了直接联合搜索在非凸大空间里的局部最优问题。

1为什么需要双层

底盘和臂的联合动作空间是 10-DoF，且不可证明为凸（即存在多个局部最优），直接搜索会非常慢且易陷入局部最优。类比：人形机器人控制中把 39-DoF 分解为 27-DoF 上身 + 12-DoF 下身的做法——分解贪心求解是移动机器人控制的通用策略。

2上层优化（底盘）

目标：找到使机械臂搜索空间质量最高的底盘位姿序列。评估一个底盘候选位姿的质量用以下目标：

\[ J_{up} = \sum_{x_e \in M} O(x^j_b, x_e) + \alpha \sum_{x_e \in N} O(x^j_b, x_e) \]

其中 M 为采样的候选臂位姿集合，N 为 top-k 最优臂位姿子集。低期望目标（臂整体质量好）+ 低 top-k 目标（最优臂候选的上界高）= 好底盘位姿。用 Dual Annealing 搜索。

3下层优化（臂）

底盘确定后，从已构建的搜索空间中用目标函数 (6) 评估各候选臂位姿，选取整体目标最低的作为最终轨迹。本质是在有限候选集上的穷举最优选择。

4对比：直接联合搜索

消融实验（Table 2）显示：直接联合搜索（Base Method w/ Direct）Overall SR 48.5%，延迟 743ms；双层方法 49.4%，延迟 693ms——不仅成功率更高，推理速度也更快（7% 提升）。证明分解不仅可行，而且更优。

5类比

类比国际象棋 AI：不把每一步所有棋子的所有移动全部展开搜索（组合爆炸），而是先选"开局策略"（底盘位姿决定大局），再在该策略下精选最佳落子（臂轨迹）——层次化减少了搜索空间。

Dual Annealing 双退火算法

Dual Annealing Search Algorithm · 双退火搜索算法

📌 TL;DR

Dual Annealing 是结合模拟退火（全局随机搜索）与梯度局部优化（SLSQP）的混合优化算法，能在非凸目标函数上找到接近全局最优的解。本文用它搜索使物理可行性目标最小的底盘和臂轨迹。

1原理

标准模拟退火（Simulated Annealing）模拟金属退火过程：以高"温度"开始大范围随机跳跃（避开局部最优），随温度降低逐渐收敛到局部最优解。Dual Annealing（scipy 实现）在此基础上额外运行一个梯度局部优化器（默认 LBFGSB，本文用 SLSQP）来精炼每次跳跃后的候选解。

2本文用法

用于对目标函数 \(O = \lambda_1 F_r + \lambda_2 F_s + \lambda_3 F_c\) 进行双层轨迹优化搜索。SLSQP（Sequential Least Squares Programming）作为局部精炼器，能处理目标函数中的约束（起止点与路径点一致）。步长由路径点之间的位置+旋转变化量决定，步数 = 变化量 / 0.05 步长。

3与其他优化方法的对比

方法全局性梯度需求适合场景
Dual Annealing（本文）强（随机跳跃避局部最优）可选（局部用梯度精炼）非凸、中等维度
梯度下降 / Adam弱（局部最优）必须平滑可微目标
粒子群优化 (PSO)较强无需黑箱目标，高维
CMA-ES强无需连续优化基准测试常用

4类比

类比爬山：普通梯度下降是"往上爬，遇到山顶就停"（局部最优）；Dual Annealing 是"先坐热气球从高空俯瞰整个山脉找到高峰区域（全局），再下来精细攀登（局部 SLSQP 精炼）"。

Reachability Cost 可达性代价

End-Effector Reachability Cost · 末端器可达性代价

📌 TL;DR

用 IK 求解器迭代次数作为可达性的代理指标：迭代次数越多 = 关节角越接近极限 = 可达性越低；超出最大迭代预算 = 不可达，代价赋极大值。这是三项目标中权重最高（λ1=10.0）、对成功率影响最大的一项。

1定义与公式

\[ F_r = \begin{cases} N_{IK}/N_{max} & \text{if } N_{IK} \leq N_{max} \\ C_0 & \text{if } N_{IK} > N_{max} \end{cases} \]

其中 \(N_{IK}\) 为 IK 求解实际迭代次数，\(N_{max}\) 为最大允许迭代次数，\(C_0\) 为极大惩罚常数。使用 Pinocchio IK solver 迭代求关节角。

2为何用迭代次数而非直接判断"可达/不可达"

IK 求解的迭代次数提供了一个软信号：少量迭代 = 关节角远离极限 = 机械臂姿态舒适；大量迭代 = 关节角已接近极限 = 在执行时容易受噪声干扰失败。这比二元判断更能区分"勉强可达"和"轻松可达"，为优化器提供更好的梯度信号。

3本文重要性

消融实验（Table 2）：去掉可达性代价后 Overall SR 从 49.4% 降至 48.2%（-1.2 pp），是三项代价中对成功率影响最大的一项。直觉：移动操作的核心问题就是底盘不在合适位置时末端器不可达，可达性代价直接解决这个核心矛盾。

4类比

类比开车停车位：不是简单判断"能不能停进去"，而是衡量"停进去需要倒几次"——需要倒多次说明角度很别扭（接近可达边界），代价高；一把停进去说明空间充裕，代价低。

Smoothness Cost 平滑性代价

Trajectory Smoothness Cost · 轨迹平滑性代价

📌 TL;DR

平滑性代价惩罚相邻轨迹点之间的关节角跳变和底盘位移跳变，避免急动（jerk）保护电机安全。注意：用 IK 求出的关节角做平滑性约束，而非末端器位姿——因为末端器小移动不等于关节角小变化。

1定义与公式

\[ F_s = \sum_{t=0}^{T_i} \left\| \theta^{t+1} - \theta^t \right\|^2 + \left\| x^{t+1}_b - x^t_b \right\|^2 \]

其中 \(\theta^t\) 为第 t 步的臂关节角向量，\(x^t_b\) 为底盘位姿（位置+旋转）。两项分别约束臂的关节角变化和底盘的运动平滑性，权重 λ2=1.0。

2用关节角而非末端器位姿的原因

论文明确指出："small changes in the arm pose cannot guarantee slight difference in joint angles"——末端器在奇异点附近微小移动可能引起关节角的巨大跳变，对电机造成冲击。因此平滑性应在关节空间而非末端器笛卡尔空间中定义。

3类比

类比开车：好的驾驶员方向盘转动和油门变化都是渐进的（平滑），不会突然猛打方向或急踩刹车——这就是平滑性代价要保证的。对机器人来说，关节角的急变会损伤关节减速器。

Collision Cost & ESDF 碰撞代价

Collision Avoidance Cost + Euclidean Signed Distance Field · 碰撞代价 + 欧氏有符号距离场

📌 TL;DR

碰撞代价用 nvblox 从 RGB-D 图像实时构建 ESDF，在机器人表面随机采样查询点，惩罚查询点到最近障碍物表面距离小于安全阈值 ε0 的情况。ESDF 让碰撞检测从"离散接触判断"变成"连续距离场"，可微分、适合优化。

1ESDF 定义

Euclidean Signed Distance Field（欧氏有符号距离场）：对环境中每个体素赋一个值 \(D(p, \Omega)\)，表示该点 p 到最近障碍物表面 \(\Omega\) 的有符号距离（障碍物内部为负，外部为正）。本文用 nvblox（GPU 加速 SDF 建图工具，ICRA 2024）从 RGB-D 流实时更新 ESDF。

2碰撞代价公式

\[ F_c = \sum_{t=0}^{T} \sum_{j=1}^{N_q} \max(0, \varepsilon_0 - D(q^t_j, \Omega)) \]

其中 \(q^t_j\) 为第 t 步在机器人表面随机采样的第 j 个查询点，\(\varepsilon_0=0.1\) 为安全裕度。当查询点到障碍物距离小于安全裕度时产生代价，完全安全（距离 ≥ ε0）时代价为 0。权重 λ3=0.6（三项中最低）。

3nvblox 的作用

nvblox 实时从 RGB-D 图像融合计算 GPU 上的 ESDF，比传统离散碰撞检测（逐个三角形测试）快得多，且输出连续可微的距离场，便于优化器利用梯度信息。真实世界中用 ORB-SLAM + Intel RealSense T265 获取实时相机位姿后，nvblox 增量更新 ESDF。

4类比

类比汽车倒车雷达：不只显示"有障碍物/没有"，而是持续报告"距离 X 厘米"——ESDF 对整个空间做了同样的事，给优化器提供了连续的"距离障碍物有多远"信号，使轨迹可以自动调整到安全距离以外。
