Flow Matching 连续动作生成

条件流匹配 (Conditional Flow Matching) · 连续正规化流的高效训练方法

📌 TL;DR

Flow matching 是 diffusion 的「直线简化版」：不学复杂的噪声预测曲线，而是直接学一个从噪声到数据的直线速度场，让积分步数从 100+ 步减到 10 步，同时保持同等的分布建模能力。π0 用它在 10 步内从高斯噪声生成 50 步动作 chunk。

1概念定义

Flow matching 是一种训练连续正规化流 (CNF) 的方法，由 Lipman et al. 2022 和 Liu 2022（Rectified Flow）独立提出。它的目标是学一个时变向量场 \(v_\theta(x^\tau, \tau)\)，使得从纯噪声 \(x^0 \sim \mathcal{N}(0, I)\) 开始，按向量场积分，最终到达数据分布 \(p_{\text{data}}\)。

与 DDPM/DDIM 的区别在于概率路径的选择：

方法概率路径需要积分步数训练目标
DDPM方差保持 SDE（弯曲路径）100–1000 步预测噪声 \(\epsilon\)
Flow Matching (OT 路径)线性最优传输 \(x^\tau = \tau x^1 + (1-\tau)\epsilon\)10–50 步预测速度场 \(u = x^1 - \epsilon\)

训练 loss：

\[ \mathcal{L}(\theta) = \mathbb{E}_{p(x^1|o), q(x^\tau|x^1)} \| v_\theta(x^\tau, o) - u(x^\tau|x^1) \|^2 \]

其中 \(u(x^\tau|x^1) = x^1 - \epsilon\)（目标速度），\(q(x^\tau|x^1) = \mathcal{N}(\tau x^1, (1-\tau)^2 I)\)（noisy 中间状态）。

2在 π0 里干什么

π0 用 flow matching 建模条件动作分布 \(p(A_t | o_t)\)：

  • 输入 \(o_t\)（图像 + 语言 + 关节状态）由 VLM backbone 编码成前缀
  • 动作 chunk \(A_t = [a_t, a_{t+1}, ..., a_{t+H-1}]\)（H=50）是预测目标
  • 训练时：对随机 \(\tau \sim \text{Beta}(\alpha, \beta)\)（强调低 \(\tau$，即更噪声的阶段），计算 \(v_\theta\) 和 \(u\) 的 MSE
  • 推理时：从 \(A^0_t \sim \mathcal{N}(0,I)\) 出发，10 步 Euler 积分到 \(A^1_t\)——即最终动作 chunk
🎯 类比：气象预报 vs. 弹道导弹

DDPM 像用复杂气象模型预测未来天气（精确但慢）；flow matching 像直接预测弹道轨迹（假设直线，10 步精确到终点）。对于动作生成这种「目标明确」的任务，直线假设非常合理，计算代价大幅降低。

3出处与关键引用

📜 起源

Lipman et al. 2022「Flow Matching for Generative Modeling」(arXiv:2210.02747) 提出条件流匹配框架；Liu 2022「Rectified Flow」(arXiv:2209.14577) 独立提出最优传输直线路径。π0 同时引用了两篇（文中 [28][32]）。Stable Diffusion 3 和 FLUX 等图像生成模型也采用了 flow matching。

Action Expert 动作专家

动作专家 · π0 中处理机器人专属输入输出的独立权重块

📌 TL;DR

动作专家是 π0 中与 VLM backbone 并存、但权重独立的第二套 transformer 权重，专门处理机器人状态和动作 token。它使 flow matching 的训练信号（含连续噪声动作）不污染 VLM 的语义表征，同时两套权重在同一次 forward pass 中共享注意力计算。

1为什么需要独立权重

直接把动作 token 加入 VLM（Transfusion 风格）的问题：

  • Flow matching 的训练 loss 是连续 MSE，作用在「噪声动作 token」上——这些 token 在各 \(\tau\) 下包含不同程度的噪声，梯度信号与语言建模的 cross-entropy 性质完全不同。
  • 若共享权重，动作训练会影响 VLM 的语言语义表征，损害语言跟随能力。
  • 独立权重方案：动作 token 走自己的 Q/K/V/FFN，仅通过 cross-attention 到 VLM 输出（注意力的 key/value 来自 VLM）来获取语义上下文。

2架构细节

π0 使用块状因果注意力遮罩（blockwise causal attention mask）分为 3 块：

Block 1: [I¹_t, ..., Iⁿ_t, ℓ_t]   ← 图像 + 语言，VLM backbone 权重
Block 2: [q_t]                       ← 机器人关节状态，action expert 权重
Block 3: [a^τ_t, ..., a^τ_{t+H-1}] ← noisy action chunk，action expert 权重

规则：块内全双向注意力；块间只能看之前的块（causal）
      Block 1 不能看 Block 2/3；Block 3 可以看 Block 1/2

动作 expert 块内使用全双向注意力（action token 互相看），确保 chunk 内的动作能相互协调。

3大小与初始化

  • VLM backbone（SigLIP 400M + Gemma 2.6B）：从 PaliGemma 开源权重初始化
  • Action expert（300M）：从头随机初始化，靠与 VLM 共享训练来对齐
  • 总参数：3.3B
🎯 类比：大脑皮层 + 小脑

VLM backbone 像大脑皮层（负责理解语言和场景），action expert 像小脑（负责精细运动协调）。两者通过神经连接共享信息，但各自有独立的处理回路，互不覆盖对方的「专业功能」。

4对比：Transfusion 与 π0

Transfusion [59]π0 Action Expert
动作/图像 token 权重与文本 token 共享 FFN独立 300M 参数块
动作类型连续图像 patch连续机器人动作 chunk
注意力遮罩图像内双向，语言 causal三块分层 causal + 块内双向
VLM 语义影响有（共享权重受动作梯度影响）小（独立权重隔离梯度）

VLA 视觉语言动作模型

Vision-Language-Action Model · 视觉-语言-动作模型

📌 TL;DR

VLA 是把 VLM（视觉语言模型）扩展到输出机器人动作的框架：把图像+语言理解能力与运动控制结合在同一模型内。核心思路是「用 Internet 预训练的语义知识去引导机器人行为」，避免从零训练。

1概念演变

模型动作表示特点
RT-2 (Google 2023)离散 token（discretized actions）首个大规模 VLA，动作变成文本 token
OpenVLA (Kim et al. 2024)离散 token开源 7B VLA，基于 LLaVA；不支持 action chunking
π0 (Black et al. 2024)连续 flow matching动作专家 + flow matching，支持 50 Hz chunk

2离散 vs. 连续动作表示的核心权衡

这是 VLA 领域最核心的设计决策之一：

  • 离散化（token 化）：动作离散成 256/512 个 bin → 可直接用 LLM 的 cross-entropy 训练 → 但丢失精度，且 tokenizer 逐个 token 自回归，无法 chunk
  • 连续（diffusion/flow）：直接建模连续分布 → 精细、多模态 → 需要专用 loss 和推理流程，不能直接接入标准 LLM 接口

π0 选择连续方案，并通过 action expert 实现了与 VLM 的融合。

3在 π0 中的用法

π0 是一个 VLA：VLM backbone（PaliGemma）提供视觉和语言理解能力，action expert 增加机器人控制输出，整体作为一个跨本体的通用策略使用。

PaliGemma VLM Backbone

PaliGemma · Google 开源 3B 视觉语言模型

📌 TL;DR

PaliGemma 是 Google 于 2024 年开源的 3B 参数 VLM，由 SigLIP（视觉编码器，400M）+ Gemma（语言模型，2.6B）组合而成。π0 选它作 backbone 的原因：小巧（实时控制可行）+ 开源（可微调）+ 性能足够强。

1架构组成

  • SigLIP（Sigmoid Loss for Language-Image Pre-Training）：视觉编码器，替代 CLIP 的 softmax 对比损失，用 sigmoid 二分类损失训练，效果更稳定
  • Gemma：Google 开源的轻量级语言模型系列（2.6B 参数版本），基于 Gemini 研究
  • Late fusion 架构：图像经 ViT 编码后投影为与语言 token 同维的嵌入，直接拼入 Gemma 序列

2π0 如何使用它

PaliGemma 权重不冻结——π0 在 PaliGemma 上做继续训练（包括机器人数据微调），同时增加 action expert 权重。这使 VLM 的视觉语言表征能被调整以更好服务于机器人控制任务。

π0 论文明确说明选 PaliGemma 是出于便利性和大小考虑，框架兼容任何 VLM：

"Note that we use PaliGemma for convenience and because of its comparatively small size (which is useful for real-time control), but our framework is compatible with any base pre-trained VLM."

Action Chunking 动作 Chunk

动作分块 · 一次预测多步动作序列

📌 TL;DR

Action chunking 是「每次推理输出未来 H 步动作序列（chunk），然后执行前几步再重新推理」的策略，而非每步推理一次。它让策略具备前瞻规划能力、减少抖动、允许复杂动作的连贯表达——对灵巧操作至关重要。

1为什么要 chunk

逐步推理的问题：

  • 高频控制（50 Hz）意味着每 20ms 要推理一次——VLM 的推理延迟无法达到
  • 单步策略容易产生「compounding error」（每步小误差累积导致轨迹偏离）
  • 复杂动作（如折叠衬衫的一连串动作）需要连贯性——逐步采样产生抖动

Chunking 的解决方案：一次预测 H=50 步动作，以较低频率更新 chunk（如每 10 步更新一次），执行期间按序列播放。

2π0 的具体使用

  • H = 50（chunk 长度），对应 1 秒动作序列（50 Hz 控制）
  • 推理：10 步 flow matching 采样生成整个 chunk → 执行 → 到达 chunk 中段再次采样（overlap 设计）
  • action expert 中 chunk 内的 token 使用全双向注意力，使 chunk 内的动作可以相互协调（第 t+5 步可以参考第 t+10 步）

3出处

📜 起源

ACT（Action Chunking with Transformers，Zhao et al. 2023）是 action chunking 在灵巧操作中的关键应用。π0 直接引用了 ACT（文中 [57]），并将 chunking 与 flow matching 结合，形成「生成高质量 chunk 分布」的能力。

Cross-Embodiment Training 跨本体训练

跨本体训练 · 多机器人平台混合训练

📌 TL;DR

跨本体训练是把来自不同机器人构型（不同自由度、不同运动学）的数据混进同一个模型训练，让模型学到跨硬件泛化的操作知识——而非针对单一机器人的专属策略。

1技术挑战

不同机器人的动作空间维度不同（6–18 DoF）。π0 的解决方案：

  • 以最大机器人的维度（18 维，容纳双臂 + 移动底盘 + 躯干）为统一动作维度
  • 较小维度的机器人用零填充（zero-padding）对齐
  • 缺少的图像槽位用遮罩（mask）处理

2π0 的 7 个机器人平台

  • UR5e（单臂）、Bimanual UR5e
  • Franka（单臂）
  • Bimanual Trossen、Bimanual ARX/AgileX
  • Mobile Trossen/ARX（非完整运动底盘）
  • Mobile Fibocom（完整运动底盘）

3为什么有效

共享 VLM 预训练知识：不同机器人在相似场景（桌面操作）中看到的视觉和语言模式高度重叠，VLM backbone 可以共享视觉表征。动作空间的异构性由 action expert 的条件化处理——proprioceptive state \(q_t\) 包含机器人构型信息，专家可以此区分不同本体。

📜 出处

跨本体训练的大规模实践由 Open X-Embodiment (OXE，2023) 数据集推动，该数据集包含 22 个机器人平台的数据。π0 使用了 OXE 的子集（称为 OXE Magic Soup），并以自采的 7 平台高质量数据为主。

MoE 混合专家

Mixture of Experts · 混合专家模型

📌 TL;DR

MoE 是一种模型架构：用多个「专家」子网络，每次推理时让路由器只激活其中的少数几个，从而以较低计算成本实现大参数量。π0 的动作专家是一种硬路由二元 MoE——图像/文本 token 固定走 VLM backbone，动作/状态 token 固定走 action expert，不经过路由网络。

1标准 MoE 与 π0 的类比

标准 Sparse MoE (e.g. Switch Transformer)π0 Action Expert
路由方式可学习路由网络（top-k 门控）硬规则：token 类型决定路径
专家数8–64 个2 个（VLM / action）
每次激活top-k（通常 k=2）每个 token 固定走一个
共享部分注意力的 key/value注意力的 key/value（从 VLM 到 action expert 的 cross-attention）

2为什么这样设计比较好

动作 token 和语言 token 的「性质差异」非常大——语言是离散的语义单元，动作是连续噪声信号。强制共享 FFN 权重（标准 Transfusion）意味着同一组权重同时要处理两种完全不同性质的信号，容易出现梯度冲突。π0 的解法是直接分离权重，把问题从「两种任务共享参数」变成「两套参数共享上下文」。

📜 出处

MoE 的当代形式由 Shazeer et al. 2017（Outrageously Large Neural Networks）规范化，Switch Transformer（Fedus et al. 2022）大规模应用于语言模型。π0 在论文中明确引用了这些工作（[45][25][12][16]）来定位其设计选择。
