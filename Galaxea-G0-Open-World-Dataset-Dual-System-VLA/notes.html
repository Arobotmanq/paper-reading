双系统 Dual-System VLA

Kahneman's System 1 & System 2 in Robotics · 机器人双系统框架

📌 TL;DR

把 Kahneman「快慢思维」理论搬进机器人：System 2（VLM）做慢速、深度的任务规划，System 1（VLA）做高频、低延迟的动作执行——两者异步运行，频率解耦。

1理论来源

出自 Daniel Kahneman《Thinking, Fast and Slow》（2011）。System 1：快速、自动、直觉；System 2：慢速、费力、逻辑推理。机器人领域借用此框架：

层次机制频率负责的问题
System 2（慢）VLM（G0-VLM / Qwen2.5-VL 微调）~2 Hz"下一步该做什么子任务？" 任务分解、场景理解
System 1（快）VLA（G0-VLA / flow matching）200 Hz"具体怎么动？" 闭环低层控制

2在 G0 里的具体实现

G0-VLM 接收用户的高层自然语言指令（如"帮我整理床"），理解当前场景，将任务分解为一系列原子子任务（如"向床移动"→"抓住被子"→"展平被子"），以约 2 Hz 的频率下发给 G0-VLA。G0-VLA 接收子任务指令后，以 200 Hz 闭环执行动作 chunk，直到收到下一条子任务指令。

🔑 G0 vs π0.5 的关键区别

π0.5：同一套权重同时做高层推理和低层动作（类 chain-of-thought，权重共享），两级推理耦合。
G0：两个完全独立的模型，不共享权重，训练分开，推理频率解耦。工程上更灵活，也更容易替换其中一个组件；但高层规划质量对系统整体影响更明显（G0-VLM 的指令精度直接决定 VLA 能否执行成功）。

3历史脉络

早期机器人规划已有 TAMP（任务与运动规划）的层次解耦，但依赖手工符号规划。VLM 兴起后，SayCan（2022, Google）首先用 LLM 做零样本高层规划 + 独立低层执行。G0 沿袭这条思路，并系统研究了 VLM 需要经过领域适配才能胜任机器人原子动作规划。

G0-VLM VLM Planner

Vision-Language Model as Task Planner · VLM 高层任务规划器

📌 TL;DR

基于 Qwen2.5-VL，用 Galaxea 数据集的子任务标注 + DeepSeek-R1 合成的高层对话数据进行微调，使通用 VLM 学会输出 VLA 可直接执行的原子动作指令（而非自然语言描述）。

1为什么要微调，直接用通用 VLM 不行？

通用 VLM（Gemini-2.5-pro、Qwen2.5-VL-72B 等）对自然语言的理解能力强，但输出的是自由文本，而 VLA 需要的是精确对齐的原子动作原语（如 "open the microwave door" 而不是 "你可以先开微波炉的门"）。实验表明，直接用 Gemini-2.5-pro 的指令精度仅 16–54%，而微调后 G0-VLM 达 74–83%。

2训练数据构建（含 DeepSeek-R1 合成）

论文设计了一个两步数据构建流程：

  1. 采样关键帧 + 子任务标注：从 Galaxea 数据集中采样 episode，提取子任务切换时的关键帧（gripper state 变化、子任务接近结束时），赋予更高采样权重。
  2. 用 DeepSeek-R1 合成高层指令：将 episode 的任务名和子任务序列喂给 DeepSeek-R1，让 LLM 推理实际场景，生成人类风格的高层口头指令（如"我要坐下，能帮我把椅子拉出来吗？"）以及机器人的语言回应。关键：不给 LLM 看图像，仅凭原子标注即可推理，说明高质量标注可替代视觉输入用于数据合成。

3评测指标

使用「指令精度」（instruction accuracy）：给定场景和可选原子动作列表，测模型能否选出正确的下一步动作。标准化 prompt 设计（约束模型只能从提供的选项中选），消除 prompt engineering 变量。

4历史脉络

类似思路：SayCan（用 LLM + affordance 模型规划）、Inner Monologue（用 LLM 进行闭环对话式规划）、HiRobot。G0 的贡献在于首次系统比较了微调 vs 直接使用大 VLM 在机器人原子动作规划上的差距，并给出了基于推理 LLM 合成数据的廉价适配方案。

G0-VLA VLA Executor

Vision-Language-Action Model as Low-Level Executor · VLA 低层动作执行器

📌 TL;DR

G0-VLA = PaLiGemma（SigLIP 视觉编码器 + Gemma Transformer）+ Action Transformer（flow matching head），以 VLM 的 KV cache 为条件生成 200 Hz 连续动作 chunk，控制 23-DoF 移动双臂机器人。

1模型结构

组件具体实现作用
视觉编码器SigLIP（来自 PaLiGemma）三路摄像头图像（头部 RGB + 双腕 RGB）→ 图像 token 序列
语言主干Gemma Transformer（PaLiGemma 初始化）融合图像 token + 语言指令 + 本体感知 → KV cache
动作专家Action Transformer（flow matching head）以 VLM KV cache 为条件，从高斯噪声去噪出连续动作 chunk

2输入输出

  • 输入：三路视觉观察 \(o_t\)（头部 + 左腕 + 右腕 RGB）+ 语言指令 \(l_t\)（高层或子任务级）+ 本体感知状态 \(s_t\)
  • 输出：动作 chunk \(A_t = a_{t:t+k}\)，horizon \(k\)，包括双臂关节位置、夹爪指令、底盘速度、躯干控制
  • 频率：200 Hz 闭环控制，15 Hz VLM 后台规划更新

3与 π0/π0.5 的对比

G0-VLAπ0 / π0.5

action headflow matching（Action Transformer）flow matching（action expert）

Flow Matching Action Head

Continuous Normalizing Flow / Rectified Flow · 流匹配动作头

📌 TL;DR

Flow matching 是扩散模型的「快速升级版」：直接学习从高斯噪声到目标动作的直线轨迹速度场，只需少量去噪步即可生成高质量连续动作，速度远快于 DDPM。G0-VLA 用它作为 Stage-2/Post-train 的动作生成头。

1直觉理解

想象从随机噪声 \(\epsilon\) 到目标动作 \(A\) 的「搬运」过程：

  • 扩散模型（DDPM）：学习弯弯曲曲的随机游走路径，需要 1000 步去噪。
  • Flow matching（Rectified Flow）：学习直线插值 \(A^\tau = \tau A + (1-\tau)\epsilon\) 对应的速度场 \(v_\theta\)，路径短，10 步以内即可。

2G0-VLA 中的 loss

\[ \mathcal{L}_\text{flow}(\theta) = \mathbb{E}_{p(A^\tau_t | o_t, l_t, s_t)} \left[ \left\| v_\theta(A^\tau_t, \tau, o_t, l_t, s_t) - u(A^\tau_t | A_t) \right\|^2 \right] \]

其中 \(A^\tau_t = \tau A_t + (1-\tau)\epsilon\) 是在时间步 \(\tau \in [0,1]\) 的插值噪声动作，\(u(\cdot)\) 是从轨迹推导的目标速度，\(v_\theta\) 是模型预测的速度场。

3为什么 Stage-1 不用 flow matching？

论文明确解释：Stage-1 跨本体数据标注质量参差不齐，如果过早引入扩散/flow matching loss，在模型表征未收敛时会损害学习过程（diffusion loss may harm the learning process before convergence）。Stage-1 用 FAST 离散 token + cross-entropy 更稳定，Stage-2 数据质量高且本体一致，再切换到 flow matching。

4与 Diffusion Policy 对比

Diffusion Policy (DDPM)Flow Matching
去噪步数~100–1000~10 步
路径随机游走（SDE）直线（ODE）
速度场训练间接（score matching）直接（速度场回归）
实时控制困难（延迟高）可行（G0-VLA 实现 200 Hz）
📜 出处

Flow matching 由 Lipman et al. 2022（Meta）和 Liu et al. 2022（Rectified Flow）独立提出。机器人领域的大规模应用来自 π0（Black et al. 2024, Physical Intelligence），G0 沿用了类似设计。

开放世界数据集 Open-World Dataset

Galaxea Open-World Dataset · 大规模真实场景机器人行为数据集

📌 TL;DR

500h、100K 轨迹、150 任务类别、50 真实场景（住宅/餐饮/零售/办公），全部使用同一款机器人（R1 Lite）采集，提供子任务级中英双语标注，是当前公开的最大规模真实开放世界移动操作数据集之一。

1为什么「单一本体」是关键设计选择

多本体数据集（如 Open-X Embodiment）混合来自不同机器人的数据，导致三个问题：（1）动作空间不一致；（2）相机视角和标注标准不一致；（3）训练时模型需要同时学习「在 Franka 上该怎么动」「在 UR5 上该怎么动」，导致干扰。单一本体数据集消除这些噪声，让模型专注于感知-语义-动作的对齐。实验证明，这是 G0 Stage-2 有效的根本原因。

2采集平台：Galaxea R1 Lite

  • 23-DoF：2×6-DoF 手臂 + 3-DoF 躯干（俯仰+升降）+ 6-DoF 向量驱动全向底盘（最高 1.5 m/s）+ 1-DoF 夹爪×2
  • 感知：头部立体 RGB（场景级）+ 双腕 Intel RealSense D405 RGB-D（近距离精细操作）
  • 遥操作：同构遥操作（直接映射人体运动到机器人运动学），避免 IK 失败和人机重定向噪声

3与先驱数据集对比

数据集规模本体场景标注粒度

DROID~76K traj单一（Franka）多实验室任务级

跨本体迁移 Cross-Embodiment

Cross-Embodiment Pre-training & Transfer · 跨机器人本体预训练与迁移

📌 TL;DR

用多种机器人数据（OXE 等）联合预训练 VLA，期望学到可迁移的通用感知-动作先验。G0 的实验揭示了关键边界：本体差异小时有益（通用 pick-and-place 先验），本体差异大时有害（底盘/躯干控制被错误先验干扰）。

1什么是「本体差异」（Embodiment Gap）

本体差异指不同机器人在以下维度上的差距：

  • DoF 结构：固定臂（6-DoF）vs 移动底盘+躯干+双臂（23-DoF）
  • 动作空间：关节角 vs 末端位姿 vs 底盘速度
  • 感知视角：固定外部相机 vs 移动头部+腕部相机
  • 任务类型：桌面操作 vs 全身移动操作

2G0 的核心发现（与主流认知的差异）

主流认知（RT-X、π0 等）认为跨本体预训练总是有益的（"数据越多越好"）。G0 实验给出了更细腻的结论：

动作类型Stage-1 跨本体预训练效果原因
Pick-and-place（通用）有益 ✅OXE 有大量此类数据，先验可迁移
底盘移动（本体专属）有害 ❌OXE 数据无底盘，引入错误偏置
躯干俯仰/升降（本体专属）有害 ❌同上，本体结构差异过大

3实践建议（从本文提炼）

  1. 跨本体预训练用于初始化 VLM 的视觉-语言表征，效果最稳定（Stage-1 只训 VLM，不训 Action Transformer）。
  2. 目标机器人有独特自由度时，必须有足够的单一本体数据做 Stage-2 专项化，才能覆写错误的跨本体先验。
  3. 仅用 Stage-1（无 Stage-2）时，少样本（20 条）场景下表现甚至不如从头训练，说明跨本体先验在低数据制下可能是负迁移。
✅ 对 Track A 用户的直接建议

若你的目标机器人有移动底盘或特殊躯干自由度，在使用 OXE / Open-X 等跨本体数据预训练时，应将其限制在纯 VLM 预训练阶段（只用高层任务描述，不训 action head），然后用目标平台的专属数据做 Stage-2 action 微调——这正是 G0 三阶段课程的核心逻辑。
