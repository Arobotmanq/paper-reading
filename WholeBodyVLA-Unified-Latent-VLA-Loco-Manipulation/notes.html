LAM Latent Action Model

Latent Action Model · 潜动作模型

📌 TL;DR

LAM 是一个「看两帧画面，反推它们之间发生了什么动作」的神经网络——但输出的不是真实物理动作，而是 VQ-VAE 量化的离散潜动作码字。作用是给没有 action 标签的视频自动补上伪标签，让人类 egocentric 视频也能进 VLA 预训练。

1定义与原理

LAM 的核心问题：给定相邻两帧 \((o_t, o_{t+k})\)，世界从状态 A 变到状态 B，中间发生了什么？LAM 用逆动力学的思路，用 VQ-VAE 将这个「变化」编码为离散码字：

\[ z_t = E_i(o_t, o_{t+k}), \quad c_t = \arg\min_{c \in \mathcal{C}_i} \|z_t - c\|_2 \] \[ \hat{o}_{t+k} = D_i(o_t, c_t), \quad \mathcal{L}_{\text{LAM}} = \|o_{t+k} - \hat{o}_{t+k}\|^2 + \|\text{sg}[c_t] - z_t\|^2 + \beta\|c_t - \text{sg}[z_t]\|^2 \]

编码器 \(E_i\) 基于 DINOv2 特征构建（spatio-temporal transformer）；解码器 \(D_i\) 接收当前帧和量化后的潜动作码字，重建未来帧。整个训练是自监督的——只需要视频，不需要 action 标注。

2本文具体用法

WholeBodyVLA 训练两个 LAM，分别对应两类数据：

LAM训练数据捕获的动作语义典型视觉变化模式
Manipulation LAMAgiBot World 双臂操作数据集（真实机器人）手臂关节运动、末端轨迹、物体交互摄像机静止，手臂区域变化主导
Locomotion LAM约 300h 单人头戴摄像机 egocentric 视频（向操作目标移动）前进方向、横向移动、转向、蹲下摄像机持续运动，整个场景随视角变化

LAM 预训练完成后，VLA 骨干（Prismatic-7B）被训练来联合预测两类潜动作码字：

\[ \min_\theta \left[-\log \pi_\theta(c_t^{\text{mani}}, c_t^{\text{loco}} \mid o_t, \ell)\right] \]

这个联合预测强迫 VLA 学习 locomotion 和 manipulation 在统一潜空间中的交互关系。

3为什么要两个 LAM 而非一个

论文发现单一 LAM 在混合数据上性能次优，原因是两类数据的视觉模态根本不同：

  • 静止摄像机（manipulation）：图像变化由手臂运动主导，LAM 倾向于关注手臂区域
  • 移动摄像机（locomotion）：整个场景相对摄像机运动，LAM 需要关注全局场景流
  • 冲突：在 loco-manipulation 任务中，手臂始终在镜头内——单一 LAM 可能把摄像机运动引起的手臂位置变化误解为手臂自身运动（即把 locomotion 误认为 manipulation）

实验验证：shared LAM（66.0%）确实优于 manipulation-only LAM（63.3%），但仍显著低于双 LAM（78.0%）。

4对比 / 出处

📜 起源与相关工作

LAM 思路来自 Genie (Bruce et al., 2024, ICML)——游戏世界模型，用 VQ-VAE 从视频中学习「潜动作」使世界可交互控制。LAPA (Ye et al., 2025, ICLR) 将其扩展到机器人操作。UniVLA (Bu et al., 2025, RSS)——同为 OpenDriveLab 团队，把 LAM 用于跨本体操作 VLA 预训练。WholeBodyVLA 的创新是将 LAM 扩展到 locomotion 域，并提出双 LAM 联合训练以处理两类数据模态的根本差异。

5与 IDM（Inverse Dynamics Model）的区别

方法输出空间训练监督适用场景
LAM（本文）VQ-VAE 学出的抽象潜动作码字自监督（重建未来帧）无 action 标注的视频，跨本体/域
IDM（如 VPT）真实物理动作空间（关节角等）监督（需要有 action 标注的数据）有大量真机 action 数据

类比：LAM 是「学会用抽象手势描述动作」，IDM 是「学会用精确数字描述动作」。前者更灵活、不依赖真机 action 数据，但输出是抽象码字而非可直接执行的指令。

统一潜学习 Unified Latent Learning

Unified Latent Learning · 统一潜动作学习框架

📌 TL;DR

「统一」的核心是：让同一个 VLA 在同一次前向传播中同时预测 manipulation 潜动作和 locomotion 潜动作——强迫模型学到两者交互的内聚表征，而不是两个分开的子系统。

1定义

统一潜学习是 WholeBodyVLA 的预训练框架，分三个阶段：

  • Stage I（LAM 预训练）：分别训练 manipulation LAM 和 locomotion LAM，在各自数据上学习离散潜动作码表
  • Stage II（VLA 联合预训练）：固定 LAM，用 LAM 码字作为伪标签，训练 VLA 同时预测 \(c_t^{\text{mani}}\) 和 \(c_t^{\text{loco}}\)
  • Stage III（finetune）：在少量真机遥操轨迹上用 LoRA 微调，将 VLA 输出的潜动作解码为真实控制信号

2本文用法

「统一」体现在 Stage II：VLA 接收图像 \(o_t\) 和语言指令 \(\ell\)，在同一前向传播中输出两类潜动作码字。训练目标是联合 MLE：

\[ \min_\theta \left[-\log \pi_\theta(c_t^{\text{mani}}, c_t^{\text{loco}} \mid o_t, \ell)\right] \]

这个联合目标强迫模型学到「当我要向前走时，我的手应该怎么配合」这类 loco-manipulation 交互先验。

3与模块化方法的对比

维度统一潜学习（本文）模块化方案（如 Being-0）
表征单一 VLA 内聚表征，locomotion+manipulation 共享权重独立的 locomotion 规划器 + manipulation 策略
交互建模隐式学习于联合训练目标中显式定义「先导航，再操作」的接口
错误传播端到端，无显式接口脆弱点模块间接口处误差累积
数据需求预训练用廉价视频，finetune 用少量遥操通常各模块需要独立大量数据

4具象例子

任务：「拿起箱子，转身，放到推车上」。在统一潜学习框架里：

  1. VLA 看到当前画面（面朝箱子）和语言指令
  2. 同时输出 manipulation 潜动作（如码字 #3 = 双臂张开准备抓取）和 locomotion 潜动作（如码字 #0 = 保持原位）
  3. 抓取后，VLA 输出 locomotion 潜动作（如码字 #2 = 左转 90°）同时 manipulation 潜动作（如码字 #7 = 双臂持重稳定）
  4. 因为是联合训练的，模型知道「转身时手臂要保持特定姿态以维持稳定」——这是模块化方案无法隐式捕获的

LMO Loco-Manipulation-Oriented RL

Loco–Manipulation–Oriented RL Policy · 面向 loco-manipulation 的 RL 控制器

📌 TL;DR

LMO 是一个专为 loco-manipulation 设计的下肢 RL 控制器，核心创新是用离散三值指令（前/停/后，左/停/右，转/停/反转，蹲高度）替代连续速度追踪目标，配合两阶段 curriculum 和真实手臂扰动训练，实现精准启停和转向。

1定义

LMO 将下肢控制建模为目标条件调节（goal-conditioned regulation），接收离散指令：

\[ u_t = [s_x, s_y, s_\psi, h^\star] \in \{-1, 0, 1\}^3 \times \mathbb{R} \]

其中 \(s_x\)（前进/停止/后退）、\(s_y\)（左移/停止/右移）、\(s_\psi\)（左转/停止/右转）为三值离散标志，\(h^\star\) 为目标站立高度（处理蹲下动作）。

2为什么要离散指令而非速度追踪

连续速度追踪的问题：

  • 启停不明确：速度从 0.5 m/s 减速到 0 时，没有明确的「停止」语义——RL 无法精确学习何时完全停止
  • 速度跨度训练不稳定：在 0~1.5 m/s 全速度范围采样训练，导致不同速度下步态碎片化
  • 偏航漂移：在前进途中无 yaw 指令时，仍可能累积偏航误差
  • VLA 输出困难：VLA 预测连续速度值比预测离散码字难度更高（连续回归 vs. 分类）

离散指令的收益：每种指令只有有限状态，RL 可以精确优化「有指令时开始、无指令时停止」的启停行为；VLA 只需要预测 3 个三值变量。

3两阶段 curriculum

阶段目标关键设计
Stage I 基础步态习得学会不摔倒、响应离散指令随机采样速度幅值 \(v^{\text{goal}}_k \sim U([0, v^{\max}_k])\)；上身跟随随机 pose 目标；关节限制逐步放宽（curriculum）
Stage II 精准稳定在操作扰动下保持精确轨迹固定巡航速度消除碎片步态；终端偏航误差惩罚 \(J_{\text{dir}} = |\text{wrap}(\psi_{\text{end}} - \psi_{\text{start}})|\)；replay AgiBot-World 手臂动作片段作为结构化扰动；静止惩罚 \(J_{\text{stand}} = \|a^{\text{leg}}_i\|_2^2\)

4结构化扰动的关键作用

Stage II 中，从 AgiBot-World 数据集采样真实手臂动作片段，插值后以不同速率 replay 给 RL 控制器作为上身扰动。这比随机力扰动更接近真实 loco-manipulation 中的惯性耦合：

  • 抓取重物时手臂向下用力 → 腿部需要向上补偿
  • 双臂展开到侧面 → 整体重心横移 → 腿部需要快速调整支撑
  • 快速将物体举高 → 惯性冲击向下传递 → 腿部需要吸收

用真实手臂数据片段训练，比随机力扰动更有针对性，能让腿部策略适应真实操作场景的惯性耦合模式。

5性能量化

  • LMO 转向精度（终端偏差）：0.05±0.01 rad，速度追踪 RL：0.26±0.01 rad（5倍差距）
  • LMO 前后走位置误差：0.21±0.01 m，速度追踪：0.24±0.04 m（且标准差更大）
  • 在三任务评测中：LMO（78.0%）vs. 速度追踪 RL（54.0%），差距的 91.7% 来自有 locomotion 的子任务阶段

VQ-VAE Vector Quantized VAE

Vector Quantized Variational AutoEncoder · 向量量化变分自编码器

📌 TL;DR

VQ-VAE 把普通 VAE 的连续 latent 替换为「从一个可学习码本（codebook）里选最近邻 token」的自编码器。在 WholeBodyVLA 里，它是 LAM 的量化模块——把连续的帧间变化向量强制离散化为有限的码字集合，使得 VLA 可以用分类（预测码字索引）而非回归来学习动作。

1核心结构

由 van den Oord et al. 2017 (DeepMind) 提出。三个组件：

连续编码器输出 z_e (连续向量)
       ↓ argmin 找码本最近邻
     码本 C = {e_1, e_2, ..., e_K}  ←  K 个可学习原型向量
       ↓ 量化
     z_q = e_{k*}  (离散码字，可用索引 k* 表示)
       ↓
     解码器重建目标

2训练 loss

\[ \mathcal{L} = \underbrace{\|x - \hat{x}\|^2}_{\text{重建}} + \underbrace{\|\text{sg}[z_e] - e_{k^*}\|^2}_{\text{codebook loss}} + \beta \cdot \underbrace{\|z_e - \text{sg}[e_{k^*}]\|^2}_{\text{commitment loss}} \]

argmin 操作不可导，用 straight-through estimator 将梯度从 \(z_q\) 直接复制给 \(z_e\)。

3在本文中的角色

WholeBodyVLA 用 VQ-VAE 量化 LAM 的连续帧间变化向量：

  • 编码器（基于 DINOv2）输出连续向量 \(z_t\)，描述帧间的「运动语义」
  • VQ-VAE 将 \(z_t\) 量化为码本中最近的码字 \(c_t\)（离散索引）
  • VLA 预训练时只需预测这个离散索引——变成分类问题，比预测连续动作向量容易
  • finetune 时，一个轻量 decoder 将 VLA 预测的码字索引转换为真实控制信号

4为什么离散化有帮助

对于 loco-manipulation 的潜学习，离散化特别有价值：

  • 有限动作词汇：locomotion 的基元（前进、后退、左转、右转、蹲）本就是离散的；强制离散化与这个先验对齐
  • 跨域对应：人类「向前走」和机器人「向前走」映射到同一个码字索引，实现跨域语义对齐
  • VLA 接口友好：预训练的 VLM 本就用 token 预测，潜动作码字 = 额外的 token 词表，无需修改架构

Loco-Manipulation Locomotion + Manipulation

Loco-Manipulation · 移动-操作协同控制

📌 TL;DR

Loco-manipulation 指机器人在移动的同时（或移动后立即）执行操作，两者需要协调而非顺序执行。关键挑战是「manipulation-aware locomotion」：移动的方向、速度、停止位置必须主动为后续操作创造前提条件。

1与普通 manipulation 的区别

维度Tabletop Manipulation（传统操作）Loco-Manipulation（本文）
机器人基座固定（桌面臂）双足行走，动态平衡
工作空间有限（臂长范围内）大范围（本文 5m+）
任务结构直接操作对象先移动到合适位置/姿态，再操作
主要挑战精准末端控制移动和操作的协调 + 大范围导航
数据可用性OXE、AgiBot World 等大规模数据集几乎没有大规模全身 loco-manipulation 数据集

2本文的三个测试任务

  • Bag Packing（袋子打包）：双手抓纸袋 → 侧步到纸箱 → 蹲下 → 放入。评测：侧向精确移步 + 蹲起精度 + 双臂协调
  • Box Loading（箱子装载）：蹲下抓箱子 → 站起来 → 转身 → 放到推车。评测：转向精度 + 蹲起 + 重物持稳
  • Cart Pushing（推车）：抓住 50kg 推车把手 → 向前推进。评测：重载下的前进稳定性和方向保持

3为什么「manipulation-aware」是关键

朴素的「先走到，再操作」方案的失败模式：机器人走到目标附近时停下，但此时的朝向、与目标的距离可能并不适合后续抓取。例如：

  • 侧步到纸箱旁，但距离偏了 5cm → 蹲下后手臂够不到
  • 转身到面对推车，但偏了 10° → 双手无法对称抓住把手

manipulation-aware locomotion 要求移动本身就以「为操作创造最优初始条件」为目标，而不只是「到达目标位置附近」。

VLA Backbone Prismatic-7B

Vision-Language-Action Backbone · 视觉语言动作主干网络

📌 TL;DR

VLA = VLM（视觉语言模型）+ 动作输出头。WholeBodyVLA 用 Prismatic-7B 作为 VLA 主干——这是一个开源的 7B 参数视觉语言模型，通过 LoRA 高效微调适配机器人控制任务。

1在本文的角色

VLA 主干完成以下几件事：

  • 接收 egocentric 图像序列和语言任务指令
  • 预训练阶段：输出两类潜动作码字（分类 head，对应 manipulation codebook 和 locomotion codebook）
  • finetune 阶段：输出连接到 action decoder 的特征，decoder 产生上身关节角和离散 locomotion 指令

2训练设置

  • 预训练：8 × H100 GPU，LAM 训练 30,000 步（batch 256），VLA 预训练 20,000 步（batch 1024）
  • 微调：LoRA（低秩适配），batch 64，10,000 步；一个模型同时覆盖三个任务（非任务专用微调）
  • 推理：RTX 4090 工作站，VLA 约 10 Hz；LMO RL 在 NanoPi 板载计算机上 50 Hz

3对比其他 VLA 主干

VLA骨干动作输出适用
WholeBodyVLA（本文）Prismatic-7B + LoRA潜动作码字 → decoder → 关节角 + 离散指令全身 loco-manipulation
GR00T N1.5Eagle2 VLM + flow matching连续关节位置双臂操作（无locomotion）
OpenVLA-OFTLlama 系列离散 token 动作单臂/双臂操作
pi0PaliGemma + flow matching连续动作（flow matching）通用操作

Action Decoder Lightweight Grounding

Action Decoder · 轻量动作解码器

📌 TL;DR

Action Decoder 是 VLA 和真实机器人之间的「翻译器」：将 VLA 预测的抽象潜动作码字，结合当前机器人状态，解码为可执行的上身关节角（manipulation）和离散 locomotion 指令（locomotion），实现「接地」（grounding）。

1定义

Action Decoder 是一个轻量网络 \(f\)，执行以下映射：

\[ a_t = f(\hat{c}_t^{\text{mani}}, \hat{c}_t^{\text{loco}}, s_t) \]

其中 \(\hat{c}_t^{\text{mani}}, \hat{c}_t^{\text{loco}}\) 是 VLA 预测的潜动作码字，\(s_t\) 是当前机器人状态（关节位置、速度等），\(a_t\) 分为两部分：

  • 上身关节角：直接发给双臂关节执行器（7-DoF × 2 臂）
  • 离散 locomotion 指令：\([s_x, s_y, s_\psi, h^\star]\)，发给 LMO RL 控制器在 50 Hz 执行

2为什么需要 Decoder 而非直接输出

LAM 学到的是跨域共享的抽象潜动作（人类视频和机器人数据共用一套码本），而不同机器人平台的关节配置不同。Decoder 提供一个轻量的「本体适配层」：

  • 预训练阶段：VLA 学通用的 loco-manipulation 先验（与具体机器人无关）
  • finetune 阶段：Decoder 用少量真机遥操数据（每任务 50 次）学习具体机器人的映射
  • 这种设计使得理论上同一 VLA 只需更换 Decoder 就能迁移到不同人形机器人平台

3与 flow matching head 的对比

输出方式代表优点缺点
轻量 Decoder（本文）WholeBodyVLA简单高效，可扩展到离散指令输出不如 diffusion/flow 处理多模态动作分布
Flow Matching Headpi0、GR00T N1强大的连续动作分布建模，处理多模态推理时需要多步去噪，计算量大
离散 Token 输出OpenVLA、RT-2与 VLM 接口一致，无需额外 head离散化损失连续动作精度
