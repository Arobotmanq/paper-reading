痛点
人形 loco-manip 普遍把上下身拆成两个控制器（上身 IK、下身 RL），下身只当移动底盘——退化成「轮式双臂」，用不上腿/脚/下蹲来做操作。
做法
不发明新模型，而是一条受控实验路线图：遥操接口 → VLA 适配 → 异构 co-training，每次只改一个变量、用真机 task progress 验味，敲定可复刻配方 OpenHLM。
结果
8 训练任务 89%；held-out 任务靠廉价数据 33%→87%；长 horizon 87.5% 反超 GR00T N1.6（57.5%）/ Ψ0（48.8%），且只用不到一半演示时间。
读法
Important 做人形 VLA / 遥操数据 → 精读 洞察2（遥操对比）+ 方法6（co-training 数据策略）；关注训练诊断 → 洞察5（action MSE 是坏代理）。⚡ 最短精读路径：Fig.1（路线图）→ Table 1（遥操对比）→ Table 2（系统级反超）。

🧭 背景 · 为什么重要（不熟此子方向先读我）

人类做 loco-manipulation 靠整条运动链协同（脚踩踏板、下蹲够低处）。人形机器人运动链相似，本应同样。但主流系统（AMO / Ψ0 / GR00T）把上下身解耦控制——arm 走 IK、leg 走 RL，再用导航命令+根高拼接，下身只是移动底盘，行为像轮式双臂。「全身原生」stack 刚被认作前进方向，但它的设计空间几乎还是空白。

💡 一句话思想
不造新模型，而是给「怎么造全身人形 VLA」写一本受控实验菜谱：像调一锅汤，一次只改一味料（遥操接口 / 预训练来源 / 动作生成 / 数据源），每改一味都上真机尝味（task progress），尝过再决定留哪味——最后端出可复刻的配方 OpenHLM。
🔗 逻辑链 · 为什么成立
  1. 先定标准：一个策略直接命令人形全部关节（whole-body native），真机 task progress 评分，立下 G1 全身原生 / G2 语言可驱动·数据高效 / G3 廉价数据可扩展 三个目标。
  2. 阶段 I「数据从哪来」：对比 3 种遥操，发现 joint-based 全身遥操是唯一能完成全部任务（踩踏板、下蹲）的接口 → 敲定它 + 在线 retarget 到机器人关节空间 + 0.2s 预览延迟。
  3. 阶段 II「策略怎么搭」：把操作 VLA（π0.5）适配到人形 34 维动作；惊人发现——非人形（静止 / 轮式）预训练迁移极好，而接口适配细节（投影初始化 / 排序 / 相对动作）都不是瓶颈。
  4. 一记警示：action MSE 在 held-out 上几乎一样、真机表现却天差地别——MSE 是坏代理（单步动作生成也栽在这个坑里）。
  5. 阶段 III「怎么省钱扩展」：廉价数据 co-training——静止遥操给「运动+语义」，HuMI（机器人无关）只给「语义」、给不了运动。
  6. 验证：长 horizon 任务 87.5% 反超两个把 G1 数据塞进预训练的 SOTA，且只用不到一半遥操时间。
🧩 方法骨架 · 管线一句话 + 三个设计决策

图像+语言指令 → 高层 VLA（π0.5 初始化，10Hz）输出 32+2 维全身参考命令 → 低层 motion-tracking 控制器（SONIC，50Hz）→ PD 跟踪到关节

  • ① 数据接口：joint-based 全身遥操（PICO 动捕 → GMR 重定向到每个关节）采数据，在线 retarget 到机器人关节空间，预览延迟 0.2s。
  • ② VLA 适配：π0.5 骨干内部原样不动，只改接口——weight surgery 扩动作投影、保留双臂排序、绝对关节目标、本体感觉输入、多步 flow matching。
  • ③ 异构 co-training：混入静止遥操 / HuMI 廉价数据，把策略扩到新物体与新指令，无需额外全身遥操。
✓ 效果 = 验证（不是重点）

问题与洞察

1问题

解耦控制让人形退化成「轮式双臂」

上下身分开控制时，下身只当移动底盘，凡是要用腿、脚、下蹲参与的操作都落在控制器表达不出的空间外。

解读
这是全文动机：要让一个策略把臂/腰/膝/脚都当成潜在「执行器」，而非把腿锁死成轮子。
📍 p.2 · §1 — 查逐字证据
Functionally, the lower body serves only as a mobile base, rather than an active participant in manipulation, leaving the humanoid closely akin to a wheeled dual-arm platform.
译
功能上，下身只充当移动底盘、而非操作的主动参与者，使人形机器人非常接近一个轮式双臂平台。
2洞察

joint-based 全身遥操是唯一全通的接口

遥操接口决定能采到什么演示：只暴露部分自由度的 VR 3-point / 解耦控制有些任务「结构上做不到」，而 joint-based 全身遥操是唯一三任务全完成的。

三种接口动作维度：解耦 21 维 / VR 3-point 24 维 / joint-based 32 维；每任务各采 40 条匹配演示再各训一个 VLA 对比。

✗ 不是「分低」而是「结构上无法表达」：取杯要下蹲、丢瓶要脚踩踏板，部分自由度接口根本碰不到这些动作。

Joint-based whole-body teleoperation is the only interface that completes all three tasks, reaching 80%–87% task progress at 10–12 footsteps per rollout.

方法（三阶段配方）

3方法

数据管线：在线 joint-space retarget + 0.2s 预览

两级层级控制（高层 10Hz、低层 50Hz）下，把全身演示在线重定向到机器人关节空间、预览延迟设 0.2s，是后续所有阶段共用的采数据底座。

两个对照敲定：(a) joint-space 在线 retarget 击败原生 SMPL 记录（88% vs 75%，81 维冗余更难学）；(b) 预览延迟扫 {0,0.2,0.4,0.6}s，0.2s 兼顾响应与平滑、0.6s 直接崩到 13%。
解读
这是「菜谱」最底层的两味料——先把高质量数据怎么采定死，后面 VLA 实验才有干净输入。
📍 p.6 · §3.1 — 查逐字证据
The data-collection pipeline that all later phases build on is now settled: joint-based whole-body teleoperation, retargeted to robot joint space online, with 0.2 s preview latency.
译
后续所有阶段所依赖的数据采集管线就此定型：joint-based 全身遥操、在线重定向到机器人关节空间、0.2s 预览延迟。
4洞察

非人形预训练惊人迁移；适配细节都不是瓶颈

用 π0.5（静止/轮式双臂预训练）初始化，迁移到人形全身效果远超无机器人预训练；而接口适配的四个选择（投影初始化/排序/相对动作/本体感觉）单改任何一个都只小幅波动。

三种初始化对比：π0.5 91% / PaliGemma（同架构无机器人数据）60% / 随机 42%。embodiment gap 真实存在，但远小于「有无任何机器人预训练」的差距。
π0.5 91% PaliGemma 60% 随机初始化 42%
解读
实操含义：做人形 VLA 不必非要人形预训练数据，直接拿成熟操作 VLA 当底座即可——π0.5 隐含的「抓失败→重试」闭环行为也一并迁移过来。
📍 p.6 · §3.2 — 查逐字证据
The gap is stark: π0.5 reaches 91% average task progress, PaliGemma drops to 60%, and random initialization collapses to 42%.
译
差距非常明显：π0.5 达到 91% 平均 task progress，PaliGemma 掉到 60%，随机初始化崩到 42%。
5洞察

action MSE 是真机表现的坏代理

两处都踩到同一坑：held-out 验证 action MSE 几乎相同甚至更低的模型，上真机却可能差一大截——离线 MSE 不能预测真机 task progress。

证据 A：π0.5 与 PaliGemma 初始化的模型 MSE 整个微调过程几乎无法区分，真机却分道扬镳。证据 B：单步动作生成 MSE 更低（≈0.007 vs 10 步 ≈0.009），真机却低约 20 个 progress 点。
解读
给做训练诊断的人的警告：别用离线 action MSE 选 checkpoint/配置，它会系统性误导——必须上真机评。
📍 p.7 · §3.2 — 查逐字证据
action MSE is a poor proxy for the value of robot pretraining: two models with matching action MSE can behave very differently on-robot.
译
action MSE 是衡量机器人预训练价值的糟糕代理：两个 action MSE 相当的模型在真机上可能表现迥异。
解读
单步动作生成本想把推理从 ~90ms 降到 ~60ms，但牺牲真机表现——作者保留多步 flow matching。
📍 p.8 · §3.2 — 查逐字证据
validation action MSE is lower for both one-step alternatives (∼0.007) than for the 10-step baseline (∼0.009). However, on the robot ... both underperform the baseline by roughly 20 task-progress points.
译
两种单步方案的验证 action MSE（≈0.007）都低于 10 步基线（≈0.009）；但在真机上二者都比基线低约 20 个 task-progress 点。
6方法

异构 co-training：静止遥操给运动+语义，HuMI 只给语义

用更便宜的数据 co-training 把策略扩到全身遥操从未覆盖的新物体/指令：静止遥操（脚不动只操作）同时供运动与语义，HuMI（机器人无关）只供语义、给不了新运动。

held-out 4 任务：8-任务 baseline 33% → 静止 co-train 87%（接近 12-任务 oracle 94%）。采集成本：每任务 40 条，静止 13min / HuMI 7min / 全身遥操 21min。HuMI 因相机/夹爪/「本质是人体动作」的域差，当前数据量下只当语义监督。
解读
给数据团队的配方：扩新物体/指令可大量用便宜的静止或 HuMI 数据；但要教全新运动（如 Pouring 倒水的倾杯），HuMI 还不行、得靠同体静止遥操或全身遥操。
📍 p.9 · §3.3 — 查逐字证据
HuMI co-training delivers new semantic understanding but not new motions, at roughly half the operator-time cost of stationary same-embodiment teleop.
译
HuMI co-training 带来新的语义理解、但带不来新运动，其操作员时间成本约为同体静止遥操的一半。
解读
33%→87% 是「廉价数据可扩展」(G3) 的承重证据：不再为每个新物体做昂贵全身遥操。
📍 p.9 · §3.3 — 查逐字证据
on the 4 held-out tasks, it lifts average progress from 33% (8-task baseline) to 87%, closing much of the gap to the oracle.
译
在 4 个 held-out 任务上，它把平均 progress 从 33%（8-任务 baseline）提到 87%，补上了与 oracle 的大部分差距。

结果与边界

7结果

长 horizon 反超 SOTA，且证明「光塞人形数据不够」

跨大垂直工作空间的长 horizon 语言任务上 87.5%，远超 GR00T N1.6（57.5%）与 Ψ0（48.8%）、逼近 oracle（97.5%），用不到一半操作时间；而两个 baseline 预训练里都含 G1 数据、OpenHLM 的 π0.5 没有。

解读
这是全文最有力的论点闭环：人形 VLA 的瓶颈不在「有没有把人形数据塞进预训练」，而在遥操格式、骨干适配、廉价数据混法这些设计细节——正是 §3 逐一敲定的东西。
📍 p.10 · §4 — 查逐字证据
GR00T N1.6 and Ψ0 both include Unitree G1 demonstrations in their pretraining data, while OpenHLM’s π0.5 initialization does not. This suggests that mixing humanoid data into pretraining is not enough on its own; building a strong humanoid VLA is a question of design details
译
GR00T N1.6 与 Ψ0 的预训练数据都含 Unitree G1 演示，而 OpenHLM 的 π0.5 初始化没有。这说明把人形数据混进预训练本身并不够；造出强人形 VLA 是个设计细节问题。
8局限

单平台 G1、HuMI 传不了运动、机制未解释

结论只在 Unitree G1 单一平台上验证，换机器人/更杂场景是否成立未知；廉价机器人无关数据（HuMI）当前还教不了新运动；且若干现象（MSE 不能预测真机、单步生成更差）仍缺机制解释。

解读
作者自陈三条短板，正好对应未来工作方向：跨平台泛化、更好用廉价数据、为 loco-manip 专门设计 VLA 架构。
📍 p.11 · §6 — 查逐字证据
it is still confined to our limited experiment resources (e.g., we only use Unitree G1 as the humanoid platform). Whether our conclusions hold in more general settings ... remains uncertain.
译
它仍受限于我们有限的实验资源（例如只用 Unitree G1 作为人形平台）。结论在更一般的设置下是否成立仍不确定。

⚖批判 · 评级

✅ 值得读的理由

少见的「全身人形 VLA 怎么造」系统性受控研究，每个设计选择配真机对照；三个反直觉发现（遥操接口决定上限、非人形预训练惊人迁移、action MSE 是坏代理）对做人形 VLA 的团队有直接工程价值；数据+权重开源、可复刻。

🔍 主要保留

结论只在 Unitree G1 单平台验证（见局限8），跨平台未知；评测多为 5 rollouts、规模有限；几个关键现象（MSE 不预测真机、单步更差）仅给假设、缺机制解释；代码尚未释出。

评级：Important  非 Breakthrough（重度建立在 π0.5 / SONIC / HuMI 之上，方法上是「配方」而非新模型），但作为开源、可复刻、反超两个 SOTA 的人形全身 VLA 实证菜谱，社区杠杆很大。
